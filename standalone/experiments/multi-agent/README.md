# Strands Agents 1.0　の検証

## 概要
Strands Agents 1.0は、エージェント間の通信と協調を可能にするためのプロトコルが追加。このフォルダでは、Strands Agents 1.0の基本的な機能とその使用方法について説明します。  

## law-search-agent （法令検索マルチエージェント）
### Strands AgentsとAWS Bedrockを活用した専門的な法令検索システム。検索・解析・法務の3つの専門ツールを統合し、包括的な法的情報を提供

### 特徴
#### 3つの専門領域
> 🔍 検索専門: 賢い法令検索、関連法令発見、高度な条文検索
> 📖 解析専門: 条文解説、法令要約、構造分析
> 💼 法務専門: 適用判定、罰則確認、手続き検索、リスク評価
#### 高度な検索機能
> ワイルドカード検索（第*条, 第?条）
> 論理演算子検索（AND, OR, NOT）
> 関連法令の自動発見
> 条文パターン検索
#### 実務的な機能
> 法的適用可能性の判定
> 罰則・制裁措置の詳細検索
> 手続き・届出要件の確認
> 法的リスクの評価

## distribute-multi-agent-system（ 不具合 ）
> 分散型のマルチエージェント</br>
> PMマネージャーがタスクをすべてのエージェントに振り分けて、統括された回答をPMマネージャーが回答
### 🤖 エージェントの種類
#### 1. ResearchAgent（リサーチエージェント）
    役割: データ・統計・事実に基づいた客観的分析
    機能:
    Knowledge Base検索の実行と評価
    検索結果の十分性を自律的に判定
    arXiv論文検索の必要性を判断
    学術的・技術的質問への最新研究動向の提供
    特徴: エージェント自身が検索結果の質を評価し、追加情報の必要性を判断
#### 2. CreativeAgent（創造エージェント）
    役割: 革新的なアイデアと未来シナリオの提案
    機能:
    既存の枠組みを超えた発想の提供
    創造的な視点と可能性の探求
    具体的な応用アイデアの提示
    設定: 高温度（0.8）で創造性を最大化
#### 3. CriticalAgent（批判的エージェント）
    役割: 客観的視点からの問題点・リスク分析
    機能:
    論理的検証と批判的思考の提供
    潜在的な課題や制約の特定
    具体的な改善提案の提示
    設定: 低温度（0.2）で論理的思考を重視
#### 4. PMAgent（プロジェクトマネージャーエージェント）
    役割: 複数エージェントの分析結果統合
    機能:
    包括的で実用的な最終回答の作成
    異なる観点のバランス調整
    実行可能なアクションプランの提示
    優先順位付けと重要ポイントの整理
## multi-agent-system
> マネージャー型のマルチエージェント。</br>
> ルーティングエージェントがタスクから適したエージェントに渡し、そのエージェントが答える
###  起動方法

```bash
python main.py
```
### 🤖 エージェントの種類

#### Router Agent
- **役割**: ユーザーの質問を分析し、適切なエージェントに振り分け
- **モデル**: Claude 3.5 Sonnet
- **判断基準**: FAQ、Expert、Generalの3カテゴリ

#### FAQ Agent
- **役割**: よくある質問、基本的な情報、製品・サービスに関する質問
- **モデル**: Claude 3.5 Sonnet
- **特徴**: 正確で簡潔な回答

#### Expert Agent
- **役割**: 専門的な技術質問、詳細な分析が必要な質問
- **モデル**: Claude 3.5 Sonnet
- **特徴**: 深い技術的洞察と詳細な説明

#### General Agent
- **役割**: 雑談、挨拶、一般的な会話
- **モデル**: Claude 3 Haiku
- **特徴**: 親しみやすく自然な会話


## swarm-research.py
#### 主要コンポーネント
AWS Strands Agents SDKを使って複数のAIエージェントを協調させる「Swarm」システム（ HTTPリクエスト ）を実装</br>
モデル設定:
Amazon Bedrockの Claude 3 Sonnetモデルを使用</br>
AWS認証情報は環境変数から取得
#### 4種類の専門エージェント:

- リサーチエージェント: 事実・データに基づく客観的分析を担当
- クリエイティブエージェント: 革新的なアイデアや未来シナリオを提案
- 批判的エージェント: 問題点やリスクを分析
- PMエージェント: 全体調整と最終回答の統合を担当
#### Swarm設定:

- 最大6回のエージェント間ハンドオフ
- 最大8回のイテレーション
- タイムアウト設定（全体10分、各ノード3分）
- 繰り返しハンドオフの検出機能
#### 動作プロセス
1. ユーザーの質問がスワームに入力される
2. PMエージェントが各専門エージェントにタスクを割り当て
3. 各エージェントが専門分野に基づいて分析を実行
4. PMエージェントが結果を統合して最終回答を作成
5. 結果とエージェント間のやり取りを表示
