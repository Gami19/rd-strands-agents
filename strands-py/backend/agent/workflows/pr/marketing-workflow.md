# マーケティングオペレーション ワークフロー

> marketing-context を土台に、ビジュアル戦略からグロース最適化まで、マーケティング施策を一気通貫で推進する。

## 反復サイクル

Phase 3（コンテンツ制作）と Phase 4（グロース最適化）は反復する。
グロース施策の結果が新たなコンテンツ要件を生み、Phase 3 にフィードバックされる。

```
Phase 0 → Phase 1 → Phase 2 → ┌→ Phase 3 → Phase 4 ─┐ → Phase 5
                                │                      │
                                └── 新コンテンツ要件 ◀──┘
```

- **Phase 3 → 4**: コンテンツの効果測定結果に基づきグロース施策を設計
- **Phase 4 → 3**: グロース施策から発見された改善点を新コンテンツに反映
- **Phase 5 → 3-4**: レビューで Critical/Major が残る場合は該当 Phase に差し戻し
- 反復は月次サイクルで回すことを推奨（Week 1-2: コンテンツ制作、Week 3-4: 効果測定・最適化）

## 品質ゲート（Phase 間の通過条件）

| ゲート | 条件 | 不合格時のアクション |
|:---|:---|:---|
| Phase 0 → 1 | marketing-context.md の全 7 セクション完成、ペルソナ定義済み、KPI 設定済み | Phase 0 に戻り不足セクションを補完 |
| Phase 1 → 2 | ビジュアル戦略承認済み、カラー/タイポ/レイアウト定義済み、WCAG AA 準拠 | Phase 1 に戻りデザイン修正 |
| Phase 2 → 3 | 技術 SEO 監査完了、GA4/GTM 設計完了、コンテンツ戦略確定 | Phase 2 に戻りインフラ整備 |
| Phase 3 → 4 | 全コピーが 7 Sweeps 通過、ブランドボイス準拠、プラットフォーム仕様準拠 | Phase 3 に戻りコンテンツ修正 |
| Phase 4 → 5 | ファネル分析完了、実験設計済み、リテンション計画策定済み | Phase 4 に戻り分析・設計を補完 |
| Phase 5（完了） | review で Critical/Major = 0、全数値検証済み、改善サイクル定義済み | 指摘修正後に再レビュー |

## Phase 0: マーケティング基盤構築

### 概要

全てのマーケティング施策の土台となる marketing-context.md を構築する。ペルソナ定義、ブランドボイス、競合分析、カスタマージャーニー、KPI フレームワークを一つのドキュメントにまとめ、後続 Phase の共通参照源とする。

市場調査資料が PDF で提供される場合は、pdf-convert で Markdown 化した後、distill で知見を蒸留してから marketing-context に反映する。

### 使用スキル

| スキル | 役割 | トリガー例 |
|:---|:---|:---|
| **marketing-context** | 基盤コンテキスト構築（ペルソナ・ブランドボイス・競合・KPI） | 「マーケティングの前提を整理して」「ペルソナを定義して」「ブランドボイスを決めて」 |
| **distill**（補助） | 市場調査資料の蒸留（project/10_inbox に資料がある場合） | 「project/10_inbox の資料を project/20_notes に蒸留して」 |
| **pdf-convert**（補助） | 市場調査 PDF の Markdown 変換 | 「PDF を変換して project/10_inbox に入れて」 |

### 成果物

| 成果物 | 形式 | 配置先 |
|:---|:---|:---|
| marketing-context.md（全 7 セクション） | `.md` | プロジェクトルート |
| ペルソナシート（ICP + 3 層モデル） | marketing-context.md 内 | — |
| ブランドボイスガイド | marketing-context.md 内 | — |
| 競合ポジショニングマップ | marketing-context.md 内 | — |
| カスタマージャーニーマップ | marketing-context.md 内 | — |
| KPI フレームワーク（NORTH STAR + ファネル） | marketing-context.md 内 | — |

## Phase 1: ビジュアル戦略・サイト設計

### 概要

marketing-context のブランドボイスとペルソナを、具体的なビジュアル戦略に落とし込む。カラーシステム、タイポグラフィ、レイアウトパターン、UX 心理学に基づくインターフェース設計を行い、Phase 3 のコンテンツ制作で使用するデザインアセットを生成する。

### 使用スキル

| スキル | 役割 | トリガー例 |
|:---|:---|:---|
| **front-design** | ビジュアル戦略策定、カラー/タイポ/レイアウト/CSS/画像リソース | 「LP のデザインを準備して」「ビジュアル戦略を策定して」 |
| **ux-psychology** | UX 心理学 10 法則に基づくインターフェース評価 | 「UX 心理学で設計を評価して」「認知負荷を減らしたい」 |
| **ui-design** | コンポーネントレベルの UI 設計・改善 | 「UI を改善して」「レイアウトを設計して」 |
| **diagram**（補助） | サイトマップ、ワイヤーフレーム、ユーザーフロー図 | 「サイトマップを描いて」「ユーザーフローを図にして」 |
| **review**（補助） | デザインアセットの品質レビュー（WCAG、ブランド整合性） | 「デザインをレビューして」 |

### 成果物

## Phase 2: SEO・計測基盤構築

### 概要

コンテンツ制作の前に、技術 SEO の基盤と計測インフラを整備する。既存サイトの監査、AI 検索への最適化、コンテンツ戦略の策定、構造化データの設計、GA4/GTM の計測設計を行う。

コンテンツなき計測は無意味であり、SEO 戦略なきコンテンツは非効率。Phase 2 で基盤を固めてから Phase 3 のコンテンツ制作に入る順序が重要。

### 使用スキル

| スキル | 役割 | トリガー例 |
|:---|:---|:---|
| **seo-analytics** | 技術 SEO 監査、AI 検索最適化、コンテンツ戦略、JSON-LD、GA4/GTM 設計 | 「SEO 監査をして」「GA4 のイベント設計をして」「AI 検索に最適化したい」 |
| **diagram**（補助） | コンテンツクラスター図、計測アーキテクチャ図 | 「コンテンツクラスター図を描いて」 |
| **data-validation**（補助） | トラッキング実装のデータ整合性検証 | 「トラッキングデータを検証して」 |

### 成果物

| 成果物 | 形式 | 配置先 |
|:---|:---|:---|
| 技術 SEO 監査レポート | `.md` | `seo/` |
| AI 検索最適化戦略 | `.md` | `seo/` |
| コンテンツ戦略（トピッククラスター + カレンダー） | `.md` | `seo/` |
| JSON-LD スキーマテンプレート | `.json` + `.md` | `seo/schemas/` |
| GA4/GTM 計測設計書 | `.md` | `analytics/` |
| UTM パラメータ命名規則 | `.md` | `analytics/` |

## Phase 3: コンテンツ制作

### 概要

マーケティング施策の実行フェーズ。LP コピー、メールシーケンス、SNS 投稿、ブログ記事、広告クリエイティブを一気に制作する。最もアウトプット量が多い Phase であり、marketing-copy と ad-creative を並行で実行可能。

全てのコンテンツは Phase 0 の marketing-context.md のブランドボイスに準拠し、Phase 2 の SEO キーワード戦略に基づいて最適化する。

### 使用スキル

| スキル | 役割 | トリガー例 |
|:---|:---|:---|
| **marketing-copy** | LP/メール/SNS/ブログのコピー執筆（7 Sweeps 編集付き） | 「LP のコピーを書いて」「メールシーケンスを作って」「SNS 投稿を作って」 |
| **ad-creative** | プラットフォーム別広告クリエイティブ生成（Google/Meta/LinkedIn） | 「広告コピーを作って」「Google Ads のクリエイティブを生成して」 |
| **seo-analytics**（補助） | ブログ記事への SEO キーワード提供 | 「SEO キーワードを確認して」 |
| **front-design**（補助） | LP 実装用ビジュアルアセット提供 | — |
| **data-validation**（補助） | コピー内の統計・数値の検証 | 「引用データを検証して」 |
| **slide-gen** / **kc-slide-gen**（補助） | マーケティング資料の PowerPoint 化 | 「マーケティング資料を PPTX にして」 |
| **md-to-pdf**（補助） | マーケティング資料の PDF 化 | 「レポートを PDF にして」 |

### Agent Teams（並行実行パターン）

marketing-copy と ad-creative は独立して並行実行可能:

```
Agent A: marketing-copy → LP コピー → メールシーケンス → SNS 投稿
Agent B: ad-creative    → Google RSA → Meta Ads → LinkedIn Ads
Agent C: marketing-copy → ブログ記事（SEO キーワード準拠）
```

### 成果物

| 成果物 | 形式 | 配置先 |

| A/B テスト計画（コピーバリアント） | `.md` | `content/` |

## Phase 4: グロース最適化

### 概要

ファネル全体のパフォーマンスを分析し、CRO 実験の設計、リテンション戦略の構築、クリエイティブ量産オペレーション、広告プラットフォーム横断分析を行う。Phase 3 で制作したコンテンツの効果を最大化するフェーズ。

Phase 3 と Phase 4 は反復的に実行される。グロース最適化の結果が新しいコンテンツ要件を生み出し、Phase 3 に戻ることがある。

### 使用スキル

| スキル | 役割 | トリガー例 |
|:---|:---|:---|
| **growth-ops** | ファネル分析、CRO、実験設計（ICE/RICE）、リテンション、クリエイティブ量産 | 「CVR を改善したい」「ファネルを分析して」「グロース実験を設計して」 |
| **seo-analytics**（補助） | A/B テスト設計手法、統計的厳密性 | 「A/B テストを設計して」 |
| **ad-creative**（補助） | パフォーマンス分析に基づく新バリアント生成 | 「新しい広告バリアントを作って」 |
| **marketing-copy**（補助） | テスト結果に基づくコピー改善 | 「LP のコピーを改善して」 |
| **diagram**（補助） | ファネル図、ダッシュボードアーキテクチャ図 | 「ファネル図を描いて」 |
| **data-validation**（補助） | ファネルメトリクスと実験結果の検証 | 「実験結果の数値を検証して」 |

### 成果物

## Phase 5: 品質検証・継続改善

### 概要

全マーケティング成果物の品質を総合検証する。コンテキスト文書から広告クリエイティブまで、ブランドボイス整合性・データ整合性・計測実装の正確性を横断的に確認し、継続的改善サイクル（PDCA）を定義する。

### 使用スキル

| スキル | 役割 | トリガー例 |
|:---|:---|:---|
| **review** | マーケティング成果物の品質レビュー（コンテキスト・コピー・戦略・レポート） | 「マーケティング成果物をレビューして」「LP コピーをレビューして」 |
| **data-validation** | 全数値・KPI・ファネルメトリクス・ROI 計算の検証 | 「数値を検証して」「KPI の整合性を確認して」 |
| **growth-ops**（補助） | 継続的改善 PDCA サイクル管理 | — |
| **seo-analytics**（補助） | 月次 SEO パフォーマンスレポーティング | 「SEO の月次レポートを作って」 |
| **slide-gen** / **kc-slide-gen**（補助） | マーケティング成果報告プレゼンテーション | 「マーケティング報告を PPTX にして」 |
| **md-to-pdf**（補助） | ステークホルダー向け PDF レポート | 「レポートを PDF にして」 |

### 成果物

| 成果物 | 形式 | 配置先 |
|:---|:---|:---|
| マーケティング成果物レビューレポート | `.md` | `reviews/` |
| データ整合性検証レポート | `.md` | `reviews/` |
| 月次パフォーマンスレポートテンプレート | `.md` | `reports/` |
| 改善サイクル記録 | `.md` | `project/50_decision_log/` |
| ADR ドキュメント（最終） | `.md` | `project/50_decision_log/` |

## クイックスタートパターン

### パターン A: 新規 LP 立ち上げ（フルパイプライン）

```
1. 「マーケティングの前提を整理して」           → marketing-context
2. 「LP のビジュアル戦略を策定して」            → front-design
3. 「UX 心理学で設計を評価して」               → ux-psychology
4. 「SEO 監査と GA4 イベント設計をして」       → seo-analytics
5. 「LP のコピーを書いて」                     → marketing-copy
6. 「Google と Meta の広告コピーを作って」     → ad-creative
7. 「CVR を改善する実験を設計して」            → growth-ops
8. 「LP と広告コピーをレビューして」           → review + data-validation
```

### パターン B: 既存サイト SEO 改善

```
1. 「マーケティングコンテキストを確認して」     → marketing-context（既存確認）
2. 「SEO 監査をして」                         → seo-analytics（Step 2-3）
3. 「コンテンツ戦略とカレンダーを作って」       → seo-analytics（Step 4）

5. 「検索流入のファネルを分析して」            → growth-ops（Step 2）
6. 「レビューして」                           → review
```

### パターン C: 広告キャンペーン最適化

```
1. 「マーケティングコンテキストを確認して」     → marketing-context（既存確認）
2. 「広告データ（CSV）を分析して」            → ad-creative（Step 2）

5. 「広告分析レポートを作って」               → growth-ops（Step 7）
6. 「数値を検証して」                         → data-validation
```

### パターン D: コンテンツマーケティング構築

```
1. 「ペルソナとブランドボイスを定義して」       → marketing-context
```

## トラブルシューティング

| 問題 | 原因 | 解決策 |
|:---|:---|:---|
| marketing-context.md が曖昧で後続スキルの精度が低い | Step 2-3 の情報具体性不足 | Phase 0 に戻り「顧客自身の言葉」アプローチで具体的な数値・競合名を追加 |
| ビジュアル戦略がブランドボイスと乖離する | Phase 0 と Phase 1 の接続不足 | marketing-context.md のブランドボイスを再確認し、front-design のコンセプトキーワードをブランド性格から導出 |
| SEO 戦略がブランドボイスと矛盾する | ブランドボイスが NG とするキーワードが SEO に必要 | ADR 記録し、ブランドボイス調整か代替キーワード戦略かを判断 |
| 広告クリエイティブが文字数制限を超過（日本語） | 日本語は概念あたりの文字数が英語より長い | 体言止め、略語、助詞省略の圧縮テクニックを適用 |
| LP コピーの CVR がベンチマーク以下 | 7 Sweeps 編集が不十分、またはフレームワーク不適合 | Phase 3 に戻り Specificity と Power の Sweep を重点実行、コピーフレームワーク変更（PAS → BAB）を検討 |
| GA4 イベントが発火しない | GTM トリガー設定ミスまたは同意モードのブロック | DebugView で検証、トリガー条件確認、同意モード設定を見直し |
| A/B テストが有意差に到達しない | サンプルサイズ不足またはエフェクトサイズが小さい | テスト期間延長、またはより大胆な変更をテスト。低トラフィックサイトではベイズ法を検討 |
| ファネル分析データが不完全 | フェーズ間の計測ギャップ | Phase 2 に戻り全ファネルステージに計測ポイントが定義されていることを確認 |
| 広告プラットフォーム間のメトリクスが比較不能 | アトリビューションウィンドウの不統一 | growth-ops Step 7b に従い全プラットフォームでアトリビューションウィンドウを統一（例: 7 日クリック） |
| コンテンツカレンダーの実行が停滞する | 計画過多・リソース不足 | 週 1 本に絞り、最もインパクトの高いキーワードから優先 |
| クリエイティブ疲労（2-3 週間で CTR 低下） | 同一クリエイティブの長期運用 | ad-creative のトラブルシューティングガイドに従い 2-3 週間でバリアントをローテーション |
| 複数スキルが矛盾する推奨を出す | 共通参照源の不在 | marketing-context.md を常に正とする。まず marketing-context.md を更新し、変更を各 Phase にカスケード |
