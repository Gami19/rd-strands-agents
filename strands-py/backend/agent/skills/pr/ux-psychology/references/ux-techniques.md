# UX リサーチ手法ガイド

> Laws of UX の各章で紹介される UX リサーチ手法の実践ガイド。

---

## 手法一覧

| 手法 | 関連法則 | 目的 |
|:---|:---|:---|
| ユーザーペルソナ | Jakob's Law | ターゲットユーザーの共有メンタルモデル構築 |
| コンテキスチュアルインクワイアリ | Fitts's Law | 実環境でのユーザー行動観察 |
| カードソーティング | Hick's Law | 情報アーキテクチャのユーザー中心設計 |
| ユーザーインタビュー | Postel's Law | ユーザーの行動・ニーズの深層理解 |
| ジャーニーマッピング | Peak-End Rule | 体験の感情的ピークと課題の可視化 |
| ユーザビリティテスト | Aesthetic-Usability | デザインの実効性検証 |
| アイトラッキング | Von Restorff Effect | 視覚的注意パターンの客観的計測 |

---

## ユーザーペルソナ

ターゲットユーザーの特徴・ニーズ・行動をまとめた架空の人物像。

### 構成要素

| セクション | 内容 |
|:---|:---|
| **基本情報** | 写真、名前、年齢、職業、キャッチフレーズ |
| **詳細** | 経歴、行動特性、フラストレーション、目標・動機、タスク |
| **インサイト** | 態度、マインドセット、ユーザーリサーチからの直接引用 |

### 注意点
- 実際のユーザーリサーチデータに基づくこと（架空のみは方向を誤るリスク）
- チーム全体で共有しセルフリファレンシャルな思考を防ぐ

---

## コンテキスチュアルインクワイアリ

ユーザーの自然な環境で観察とインタビューを行うエスノグラフィ手法。

### 実施手順

| フェーズ | 内容 |
|:---|:---|
| **プライマー** | 自己紹介、目的説明、機密性の保証、参加者をリラックスさせる |
| **コンテキスチュアルインタビュー** | 作業を観察しながら、興味深い行動があれば中断して議論。オープンエンドの質問で深掘り |
| **ラップアップ** | 観察したプロセスの理解を要約し、参加者に確認・修正してもらう |

### 強み
- ユーザーインタビューだけでは得られない「隠れたインサイト」を発見
- 実際の作業プロセスをリアルタイムで観察し、正確な理解を得る

---

## カードソーティング

ユーザーにトピックをグループ化・命名させる情報アーキテクチャ設計手法。

### 手順（モデレート・オープンソーティング）

| ステップ | 内容 |
|:---|:---|
| **1. トピック特定** | 整理対象の主要コンテンツを個別カードに記載 |
| **2. トピック整理** | 参加者に自分にとって意味のあるグループに整理させる。考えを声に出してもらう |
| **3. カテゴリ命名** | 参加者が各グループに名前をつける。メンタルモデルの理解に重要 |
| **4. デブリーフ** | グループ化の根拠を説明してもらい、困難だった点を収集 |

### バリエーション
- **オープン**: 参加者が自由にグループ化・命名（最も一般的）
- **クローズド**: 研究者がグループを事前定義、参加者はそこにカードを分類

## ユーザーインタビュー

1 対 1 セッションで行動・習慣・態度についてのインサイトを収集する手法。

### 実施手順

| ステップ | 内容 |
|:---|:---|
| **目的の定義** | 具体的で焦点を絞った目標を設定（「ユーザーについて学ぶ」は広すぎ） |
| **ディスカッションガイド作成** | テーマに沿った質問を準備。自然な会話の流れを許容 |
| **ラポール構築** | 参加者をリラックスさせる。正解/不正解はないと伝える |
| **インタビュー実施** | オープンクエスチョン（what/how/when/tell me about）を使用 |

### 避けるべきこと
- クローズドクエスチョン（はい/いいえ）
- 誘導質問（回答を方向づける質問）
- 曖昧すぎる質問

## ジャーニーマッピング

ユーザーが目標達成までに辿る体験を時系列で可視化する手法。

### 構成要素

| セクション | 内容 |
|:---|:---|
| **レンズ** | ペルソナ、シナリオ、期待値の定義 |
| **体験** | フェーズ → アクション → マインドセット → 感情曲線（時系列） |
| **インサイト** | 改善機会、担当メトリクス、内部オーナーシップ |

### Peak-End Rule との接続
- **感情曲線**: 体験中の感情的ピーク（ポジティブ/ネガティブ）を可視化
- **ピークの設計**: ポジティブピークを意図的に強化、ネガティブピークに軽減策を設計
- **終了体験**: 最後のタッチポイントが肯定的な印象を残すか確認

## ユーザビリティテスト

代表ユーザーにタスクを実行させ、デザインの問題を発見する観察手法。

### 実施手順

| ステップ | 内容 |
|:---|:---|
| **テスト計画** | 目的定義、テストスクリプト作成、代表ユーザーの招集 |
| **テスト実施** | プロトタイプまたは実製品でリアルなタスクを実行させる。Think Aloud 法を使用 |
| **データ統合** | グループでの振り返り。引用と観察を抽出 → テーマ・パターン特定 → インサイト導出 |

### Aesthetic-Usability Effect への対策
- 主観評価（「使いやすいと感じた」）と客観データ（タスク完了率・時間）を併用
- 美的要素を超えた質問で深掘り（「どこで迷いましたか」等）
- 観察者バイアス（ホーソン効果）の軽減: テストはデザインのテストでありユーザーのテストではないと明示

## アイトラッキング

ユーザーの視線の動きを計測・分析する調査手法。

### 研究タイプ

| タイプ | 目的 |
|:---|:---|
| ヒートマップ | 注目領域の視覚化 |
| ゲイズプロット | 視線の移動パターンの追跡 |
| 注視時間 | 特定要素への注目度の定量化 |
| 注意バイアス | 刺激に対する注意の偏り測定 |
| 比較研究 | デザインバリエーション間の視線パターン比較 |

### 実施手順
1. リサーチクエスチョンの定義
2. 代表的な参加者の選定
3. 機器のセットアップとキャリブレーション
4. 刺激の準備（UI、ウェブサイト等）
5. テスト実施・データ収集
6. データ分析
7. 結果の解釈と結論導出

### 注意点
- 他の手法と組み合わせて使用する（アイトラッキング単体では「なぜ」が分からない）
- 文化差、技術的制約、サンプルサイズの限界を考慮
