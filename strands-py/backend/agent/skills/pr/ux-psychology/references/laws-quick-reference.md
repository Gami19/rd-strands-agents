## 法則ごとの Key Takeaways と基準値

### 1. Jakob's Law

| Key Takeaway | 実践ガイドライン |
|:---|:---|
| 馴染みのある製品から期待を転移する | 業界標準のナビ配置・検索位置・フォーム構造に従う |
| 既存メンタルモデルでタスクに集中できる | デザインシステムの共通コンポーネントを活用 |
| 変更時はメンタルモデル不一致を最小化 | リデザイン時はオプトイン方式で段階的移行を提供 |

### 2. Fitts's Law

| Key Takeaway | 基準値 |
|:---|:---|
| 十分なターゲットサイズ | Apple 44×44pt / Google 48×48dp / WCAG 44×44px |
| ターゲット間の十分なスペース | 8dp 以上（Material Design） |
| 到達しやすい位置に配置 | 画面中央〜下部（スマートフォン）、画面端（デスクトップ） |

### 3. Miller's Law

| Key Takeaway | 実践ガイドライン |
|:---|:---|
| 小さなチャンクに整理 | 関連コンテンツを視覚的にグループ化 |
| 個人差を考慮 | 記憶容量は前知識と文脈で変動（最近の研究では 4 チャンク説も） |
| 「7」を制限の根拠にしない | ナビ項目の数はチャンキングとデザインの質で決まる |

### 4. Hick's Law

| Key Takeaway | 実践ガイドライン |
|:---|:---|
| 応答時間が重要なら選択肢を最小化 | 段階的開示で必要時のみ表示 |
| 複雑なタスクを小ステップに分解 | プログレッシブオンボーディング |
| 推奨オプションをハイライト | ソーシャルプルーフ、デフォルト選択 |
| 過度な単純化を避ける | アイコン+テキストラベルで意味を明確化 |

### 5. Postel's Law

| Key Takeaway | 実践ガイドライン |
|:---|:---|
| 多様な入力に柔軟に対応 | 名前・住所・電話番号の国際差を許容 |
| 信頼性の高いアクセシブルなUIを出力 | レスポンシブデザイン、プログレッシブエンハンスメント |
| 想定・計画するほどレジリエントに | テキスト拡張（最大 300%）、フォントサイズ変更に対応 |

### 6. Peak-End Rule

| Key Takeaway | 実践ガイドライン |
|:---|:---|
| ピーク瞬間と最終瞬間に注力 | ジャーニーマッピングで感情曲線を可視化 |
| 最も役立つ・楽しい瞬間を設計 | パーソナライゼーション、ゲーミフィケーション |
| ネガティブ体験はより鮮明に記憶される | 待ち時間軽減、リアルタイムバリデーション |

### 7. Aesthetic-Usability Effect

| Key Takeaway | 実践ガイドライン |
|:---|:---|
| 美しさがユーザビリティの知覚を向上 | 第一印象は 50ms で形成される |
| 軽微な問題が許容される | 美的品質は信頼と認知能力にも影響 |
| テスト時に問題が隠蔽される | 行動観察と主観評価の乖離を確認 |

### 8. Von Restorff Effect

| Key Takeaway | 実践ガイドライン |
|:---|:---|
| 重要要素を視覚的に際立たせる | 色・サイズ・形状・位置・モーションで差別化 |
| 抑制的に使用する | 1 画面の強調は 1-2 箇所に限定 |
| 広告と誤認させない | バナーブラインドネスに注意 |
| アクセシビリティ配慮 | WCAG コントラスト比 4.5:1、色以外の手がかり |

### 9. Tesler's Law

| Key Takeaway | 実践ガイドライン |
|:---|:---|
| 核心的複雑性はシステムかユーザーが負担 | 可能な限りシステム側で吸収 |
| 設計・開発側が複雑性を引き受ける | オートコンプリート、AI アシスト、プログレッシブディスクロージャー |
| 抽象化しすぎに注意 | 操作可能な手がかりを十分に残す |

### 10. Doherty Threshold

| Key Takeaway | 基準値 |
|:---|:---|
| 400ms 以内にフィードバック | 100ms = 即座、100-400ms = 知覚可能だが快適 |
| 知覚パフォーマンスを活用 | スケルトンスクリーン、オプティミスティック UI |
| プログレスバーで待ち時間を許容可能に | 10 秒超は残り時間表示を追加 |
| 意図的な遅延で信頼感を醸成 | 速すぎる応答は不信感を生むことがある |
