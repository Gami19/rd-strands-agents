# カテゴリ別クイズ生成パターンと出題テンプレート

> クイズの生成ルール、形式別テンプレート、カテゴリ別サンプル問題集

## クイズ生成ルール

### 出題原則

| 原則 | 説明 |
|:---|:---|
| **ドキュメント準拠** | 出題内容は必ず社内ドキュメントに根拠があること |
| **実務志向** | 暗記ではなく、実務で使える理解を問うこと |
| **難易度明示** | 各問題に難易度（基礎/応用/実践）を明示すること |
| **解説必須** | 正誤判定後、必ず根拠ドキュメントを引用した解説を付けること |
| **励ましトーン** | 不正解でも否定せず、学びの機会として前向きにフィードバック |

### 難易度別の形式バランス

| 難易度 | 4択（MC） | ○×（TF） | 記述式（SA） | 問題数 | 合格基準 |
|:---|:---|:---|:---|:---|:---|
| **基礎** | 50% | 30% | 20% | 5問 | 80%（4/5） |
| **応用** | 30% | 20% | 50% | 5問 | 70%（3.5/5） |
| **実践** | 20% | 10% | 70% | 5問 | 60%（3/5） |

### 記述式の採点基準

| 評価軸 | 配点 | 基準 |
|:---|:---|:---|
| **キーワード** | 40% | 必須キーワードが含まれているか |
| **論理構成** | 30% | 回答が論理的に構成されているか |
| **根拠引用** | 30% | ドキュメントの内容を踏まえた回答か |

## 形式別テンプレート

### 4択問題（Multiple Choice）テンプレート

```
【4択問題】[難易度: 基礎/応用/実践]
Q: [質問文]

A) [選択肢A]
B) [選択肢B]
C) [選択肢C]
D) [選択肢D]

あなたの回答を教えてください（A/B/C/D）。
```

フィードバックテンプレート:
```
正解: [X]
判定: [正解/不正解]

【解説】
[正答の理由と根拠ドキュメントの引用]

【補足】
[関連する知識や実務での活用ポイント]
```

### ○×問題（True/False）テンプレート

```
【○×問題】[難易度: 基礎/応用/実践]
Q: [命題文]

○（正しい）か ×（誤り）かで答えてください。
```

### 記述式問題（Short Answer）テンプレート

```
【記述式問題】[難易度: 基礎/応用/実践]
Q: [質問文]

※ 採点基準: キーワード（40%）+ 論理構成（30%）+ 根拠引用（30%）
2-3文で回答してください。
```

## カテゴリ別サンプル問題

### Culture カテゴリ（5問）

**Q1.** 【4択問題】[基礎]
ナレコムの3つのコアバリューに含まれないものはどれですか？

A) お客様の期待を超える
B) 業界トップの技術力
C) 主力選手
D) チームワークを超えた絆

> 正解: B（「業界トップの技術力」はコアバリューではない）
> 根拠: コアバリュー文書

**Q2.** 【○×問題】[基礎]
「主力選手」とは、全員がリーダーシップを発揮し、自ら考え行動することを意味する。

> 正解: ○
> 根拠: コアバリュー文書 — 主力選手の定義

**Q3.** 【4択問題】[応用]
チームメンバーと意見が対立した時、コアバリュー「チームワークを超えた絆」に基づく最も適切な行動はどれですか？

A) 上司に判断を委ねる
B) 自分の意見を押し通す
C) 建設的な対話で互いの視点を理解し、共に最善策を見つける
D) 対立を避けるため自分の意見を取り下げる

> 正解: C
> 根拠: 行動指針 — 対話と協力の項目

**Q4.** 【記述式問題】[応用]
「お客様の期待を超える」をあなたの業務で実践するとしたら、具体的にどのような行動をとりますか？

> 採点キーワード: 期待の理解、+αの価値提供、先回り
> 根拠: コアバリュー文書 + 行動指針

**Q5.** 【○×問題】[実践]
社内のコミュニケーションツールで、お客様に関する機密情報を含むメッセージを送信する際は、パブリックチャンネルでも問題ない。

> 正解: ×
> 根拠: コミュニケーション文書 — 情報の取り扱いルール

### Biz カテゴリ（5問）

**Q1.** 【4択問題】[基礎]
ナレコムの主力サービス領域として最も適切なものはどれですか？

A) ハードウェア製造
B) データ・クラウド・AI 領域のコンサルティングと開発
C) 人材派遣
D) パッケージソフトウェア販売

> 正解: B
> 根拠: 事業内容文書

**Q2.** 【○×問題】[基礎]
ナレコムのサービスは特定の業種に限定されている。

> 正解: ×
> 根拠: サービス概要文書 — 対象業種

**Q3.** 【4択問題】[応用]
お客様が「データ活用を推進したいが、何から始めればいいかわからない」と相談してきた場合、最初に提案すべきアプローチはどれですか？

A) 最新のAIモデルの導入
B) 現状のデータ活用状況のアセスメント
C) 大規模なデータ基盤の構築
D) 全社員向けのデータリテラシー研修

> 正解: B
> 根拠: サービス概要文書 — コンサルティングアプローチ

**Q4.** 【記述式問題】[応用]
ナレコムのサービスが競合他社と差別化できるポイントを2つ挙げて説明してください。

> 採点キーワード: 技術力、伴走型支援、End-to-End、アセット活用
> 根拠: 事業内容文書 + サービス概要文書

**Q5.** 【記述式問題】[実践]
製造業のお客様に対して、ナレコムのサービスがどのような価値を提供できるか、具体例を1つ挙げて説明してください。

> 採点キーワード: 品質管理、データ基盤、予知保全、コスト削減
> 根拠: 顧客理解文書 — 業種別事例

### Tech カテゴリ（5問）

**Q1.** 【4択問題】[基礎]
ナレコムの開発フローでプルリクエスト（PR）を作成する際、必ず必要なものはどれですか？

A) 全テストの通過とレビュアーの承認
B) マネージャーの口頭承認
C) 全メンバーへのメール通知
D) 顧客への報告

> 正解: A
> 根拠: 開発フロー文書 — PR ルール

**Q2.** 【○×問題】[基礎]
Databricks は主にデータ分析・AI/ML のための統合プラットフォームである。

> 正解: ○
> 根拠: データ基盤文書 — Databricks 概要

**Q3.** 【4択問題】[応用]
Medallion Architecture における「Silver層」の主な役割はどれですか？

A) 生データをそのまま保存する
B) データのクレンジングと標準化を行う
C) ビジネスレベルの集計テーブルを作成する
D) 外部システムへのデータ配信を行う

> 正解: B
> 根拠: データ基盤文書 — Medallion Architecture

**Q4.** 【記述式問題】[応用]
社内のセキュリティポリシーに違反するコードを発見した場合、どのような手順で対応すべきですか？

> 採点キーワード: 即時報告、修正、インシデント対応フロー、再発防止
> 根拠: セキュリティ基礎文書 — インシデント対応

**Q5.** 【記述式問題】[実践]
新しいプロジェクトで Git リポジトリを初期設定する際、最低限必要な設定項目を3つ挙げてください。

> 採点キーワード: ブランチ戦略、.gitignore、CI/CD設定、レビュールール
> 根拠: 開発フロー文書 + 開発ツール文書

### Ops カテゴリ（5問）

**Q1.** 【4択問題】[基礎]
経費精算の申請で最も重要なルールはどれですか？

A) 金額の正確な記載と領収書の添付
B) メールでの事前承認
C) 月末にまとめて申請
D) 上司の口頭承認

> 正解: A

**Q2.** 【○×問題】[基礎]
NDA（秘密保持契約）は、お客様との契約だけでなく、社内の機密情報にも適用される。

> 正解: ○
> 根拠: NDA・秘密保持文書 — 適用範囲

**Q3.** 【4択問題】[応用]
お客様先で業務中に、お客様の機密情報が含まれるファイルを誤って社外に送信してしまった場合、最初にすべき行動はどれですか？

A) 送信先にファイル削除を依頼する

> 採点キーワード: 事前申請・承認、経費管理、出張報告書、精算
> 根拠: 各種申請手続き文書 + 経費精算文書

## クイズ生成時の注意事項

| 注意点 | 説明 |
|:---|:---|
| **同一問題の繰り返し回避** | 前回と同じ問題を出さない。同じトピックでも表現を変える |
| **選択肢のバランス** | 4択の正解位置をランダムに分散させる |
| **記述式の柔軟採点** | キーワードの表現揺れを許容する（「即座に報告」≒「すぐに上司に報告」） |
| **実践問題の現実性** | 実際の業務で遭遇する場面を想定する |
| **コアバリューとの接続** | 可能な限り、回答にコアバリューとの関連を含める |
