# メンタリングシナリオ集とレスポンスパターン

> カテゴリ別のメンタリングシナリオ、緊急度マトリクス、回答テンプレート

## カテゴリ × 緊急度マトリクス

| カテゴリ | 高（即時対応） | 中（丁寧に回答） | 低（自習促進） |
|:---|:---|:---|:---|
| **Culture** | コアバリューの解釈で業務が止まっている | 行動指針の具体的な実践方法 | 社内イベントの参加方法 |
| **Biz** | 顧客対応で判断に迷っている | サービス内容の詳細理解 | 業界動向への興味 |
| **Tech** | 開発環境が動かない | Git の操作方法 | 新技術への興味 |
| **Ops** | セキュリティインシデント発生 | 経費精算の手順 | 福利厚生の詳細 |
| **キャリア** | — | キャリアパスの相談 | スキルアップの方法 |
| **メンタル** | 強い不安・孤立感 | 漠然とした不安 | 成長実感の確認 |

## シナリオ別詳細レスポンス例

### B-1: お客様から無理な要望を受けた

**状況**: 納期や仕様で実現困難な要望をお客様から直接受けた。

**OK例**:
1. まず要望を正確に理解する（復唱・確認）
2. 「社内で検討し、最善の方法をご提案します」と回答
3. 上司に報告し、対応可能な範囲と代替案を整理
4. お客様に代替案とともに回答（期待を超える＝最適解を共に見つけること）

**NG例**:
- 「できます」と安請け合い → チームに過負荷、品質低下
- 「できません」と即座に断る → 信頼関係の毀損

**判断フレームワーク**: 受け止める → 持ち帰る → 代替案を含めて回答する

### M-1: 技術力が周囲に比べて足りないと感じる

**状況**: 周りのエンジニアのスキルと比較して自信を失っている。

**共感**: 「入社直後は誰もが感じる不安です。あなただけではありません」

**正常化**: 「今のチームメンバーも、最初は同じスタートラインでした。大切なのは今の自分と比べて成長していること」

**コアバリュー接続**:
- 「主力選手」: 成長のスピードは人それぞれ。自分のペースで着実に前進することが主力選手
- 「絆」: チームは互いに支え合う。先輩への質問は迷惑ではなく、歓迎される文化

**アクション**:
1. Tech カテゴリの基礎クイズで今の理解度を客観的に確認
2. 1週間前の自分と比較して成長を可視化
3. メンターとの 1on1 で具体的な学習計画を相談

### O-2: 締切に間に合いそうにない

**状況**: タスクの進捗が遅れ、期限内に完了できる見込みがない。

**OK例**:
1. 遅延が見えた時点で即座に上司に報告
2. 現在の進捗と残作業を具体的に説明
3. 代替案を提示（スコープ縮小、期限延長、追加リソース）

**NG例**:
- 期限ギリギリまで報告しない → 対策の選択肢が狭まる
- 一人で抱え込む → 品質低下と燃え尽き

**判断フレームワーク**: 「報告は早ければ早いほど価値がある」。遅延報告は失敗ではなく、プロフェッショナルな行動。

### T-3: コードレビューで厳しい指摘を受けた

**状況**: PR のレビューで多くの指摘を受け、落ち込んでいる。

**共感**: 「厳しい指摘を受けると辛いですよね。その気持ちは自然です」

**正常化**: 「コードレビューの目的は人を批判することではなく、コードの品質を上げること。指摘が多い＝学びの機会が多いということ」

**コアバリュー接続**:
- 「絆」: レビュアーは敵ではなくチームメイト。良いコードを一緒に作ろうとしている
- 「主力選手」: 指摘を吸収して次に活かす力こそが成長

**アクション**:
1. 指摘を「すぐ直せる」「理解が必要」「質問したい」に分類
2. わからない指摘は遠慮なくレビュアーに質問
3. 同じ指摘を受けないようメモに残す

## エスカレーション判断基準

| 状況 | エスカレーション先 | タイミング |
|:---|:---|:---|
| ドキュメントに回答がない質問 | 人間のメンター | 即時 |
| セキュリティインシデント | 上司 + セキュリティ担当 | 即時 |
| メンタルヘルスの深刻な相談 | 人間のメンター + 人事 | 即時 |
| キャリアの重要な判断 | 人間のメンター | 次回 1on1 |
| 技術的な高度な質問 | テックリード / シニアエンジニア | 適宜 |
| 人間関係のトラブル | 人間のメンター + 人事 | 即時 |

**原則**: AI メンターの限界を認識し、人間のメンターへの接続を躊躇しない。「わからない」と正直に伝えることは信頼の証。
