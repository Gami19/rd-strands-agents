# チャネル別コピーガイドライン

> LP・メール・SNS・ブログそれぞれの特性に合わせたコピー設計ルール。
> チャネルが変わればルールも変わる。同じコピーを使い回さない。

## 1. LP（ランディングページ）

### セクション構成

| セクション | 内容 | 文字数目安 |
|:---|:---|:---|
| ヒーロー | 見出し + サブ見出し + CTA + ビジュアル | 見出し 10〜15 語、サブ見出し 20〜30 語 |
| ソーシャルプルーフバー | ロゴ / 導入社数 / 推薦の声 | ロゴ 3〜6 社、数値 1〜3 個 |
| 課題セクション | 読者の痛みを代弁 | 50〜100 語 |
| ソリューション | ベネフィット 3〜5 個。機能ではなく成果で語る | 各ベネフィット 20〜40 語 |
| 使い方 | 3〜4 ステップの簡潔なプロセス | 各ステップ 15〜25 語 |
| 詳細な社会的証明 | 推薦の声・事例（具体的な数字付き） | 各事例 50〜100 語 |
| FAQ | 反論への先回り回答 | 5〜8 問 |
| 最終 CTA | リスクリバーサル + CTA ボタン | 30〜50 語 |

### CTA 配置ルール

| ルール | 説明 |
|:---|:---|
| ファーストビューに CTA | スクロールせずに CTA が見える位置に配置 |
| CTA は最低 3 箇所 | ヒーロー、中間、最終セクションの 3 箇所 |
| ボタンテキストは具体的に | 「送信」ではなく「14 日間無料で試す」 |
| CTA 周辺にリスクリバーサル | 「クレジットカード不要」「いつでも解約可能」をボタンの直下に |
| 色のコントラスト | CTA ボタンは背景と明確に区別される色を使用 |

### LP のチェックリスト

- [ ] ヒーローセクションだけで「何のサービスか」「誰向けか」「何が得られるか」が分かる
- [ ] ページ全体で CTA が 3 箇所以上ある
- [ ] スクロールの深い位置にも社会的証明がある
- [ ] FAQ で主要な反論を 5 つ以上処理している
- [ ] モバイルで見た時に文字が小さすぎない・ボタンが押しやすい

## 2. メール

### 件名のルール

| ルール | 例（OK） | 例（NG） |
|:---|:---|:---|
| 2〜6 語で短く | 「新機能のお知らせ」 | 「【重要】新機能リリースのお知らせと今後のアップデート予定について」 |
| 内部メール風のカジュアルさ | 「来週の件」 | 「【株式会社〇〇】ご提案の件」 |
| 売り込み語を避ける | 「事例をまとめました」 | 「★限定★今だけ無料！お見逃しなく！」 |
| パーソナライゼーション | 「{名前}さん、1 つ質問です」 | 「お客様各位」 |

### プレビューテキスト

| ポイント | 説明 |
|:---|:---|
| 文字数 | 90〜140 文字（モバイルで表示される範囲） |
| 役割 | 件名を補完する。件名と同じ内容の繰り返しは NG |
| 書き方 | 件名が「問いかけ」ならプレビューは「答えのヒント」 |

### 本文構成

```
Hook（フック）    — 最初の 1〜2 行で興味を引く
Context（文脈）   — なぜこのメールを送っているか
Value（価値）     — 読者が得られるもの
CTA（行動喚起）   — 1 つだけ、明確に
Sign-off（署名）  — 個人名で。チーム名や部署名だけは NG
```

### 本文の長さガイド

| メールの種類 | 語数目安 | 特徴 |
|:---|:---|:---|
| トランザクション（確認・通知） | 50〜125 語 | 用件のみ。余計な装飾なし |
| 教育・ナーチャー | 150〜300 語 | 1 つの Tips / インサイトを提供 |
| ストーリー型 | 300〜500 語 | 事例やストーリーで価値を示す |
| コールドメール | 50〜100 語 | 短ければ短いほど良い |

### モバイル最適化

- [ ] 件名が 30 文字以内でモバイルの 1 行に収まる
- [ ] プレビューテキストが設定されている（未設定だと「このメールを表示するには〜」が表示される）
- [ ] 本文の段落が 2〜3 行以内（モバイルでは長い段落が壁のように見える）
- [ ] CTA ボタンが指で押せるサイズ（最低 44x44 px）
- [ ] 画像に alt テキストが設定されている（画像非表示環境への対応）

## 3. SNS

### LinkedIn

| 項目 | ガイドライン |
|:---|:---|
| 文字数 | 1,200〜1,500 文字（「もっと見る」の前に 210 文字表示） |
| トーン | プロフェッショナルだが人間味がある。実体験ベース |
| フック技法 | 最初の 1 行で「もっと見る」を押させる。意外な事実・逆説・質問を使う |
| 構成 | フック → ストーリー or インサイト → 学び → CTA or 質問 |
| リンク | コメント欄に配置（本文中のリンクはリーチが下がる） |
| ハッシュタグ | 3〜5 個。投稿の最後にまとめて配置 |
| 投稿時間 | 火〜木の 8:00-10:00 または 17:00-18:00 |

**フック例**:
```
「3 年前、私は最悪のプレゼンをしました。」（ストーリー型）
「マーケティング予算を 50% カットしたら、リードが 2 倍になりました。」（逆説型）
「あなたの LP、最初の 3 秒で離脱されています。理由を知っていますか?」（質問型）
```

### X（旧 Twitter）

| 項目 | ガイドライン |
|:---|:---|
| 文字数 | 単発: 100 文字以下が最適。スレッド: 1 ツイート 200 文字以内 |
| トーン | 簡潔で鋭い。意見を明確に。曖昧さは NG |
| フック技法 | 数字で始める / 常識を否定する / 「〜する方法」リスト形式 |
| スレッド構成 | 1 ツイート目がフック → 2〜8 本文 → 最終ツイートで CTA + リツイート依頼 |
| ビジュアル | 画像・動画付きはエンゲージメント 2〜3 倍 |
| 投稿時間 | 平日 12:00-13:00 または 18:00-20:00 |

**フック例**:
```
「LP の CVR を 3 倍にした見出しの書き方（スレッド）」
「99% のコールドメールが無視される理由はたった 1 つ。」
「マーケティングの最大の嘘: 『コンテンツは王様』」
```

### Instagram

| 項目 | ガイドライン |
|:---|:---|
| キャプション文字数 | 125〜200 文字（ファーストビューで表示される範囲に収める） |
| トーン | ビジュアルファースト。テキストは補助。親しみやすさ重視 |
| フォーマット | リール（最優先）> カルーセル > 静止画 > ストーリーズ |
| カルーセル | 教育コンテンツに最適。1 枚目がフック、2〜9 枚目が本文、最終枚が CTA |
| ストーリーズ | 舞台裏・日常・アンケートで親密さを構築 |
| ハッシュタグ | 5〜15 個。ニッチなタグを中心に。コメント欄に配置も可 |
| 投稿時間 | 平日 11:00-13:00 または 19:00-21:00 |

**カルーセル構成例**:
```
1 枚目: 「LP の CVR を 3 倍にする 5 つのコツ」（フック）
2〜6 枚目: 各コツの解説（1 枚 1 メッセージ）
7 枚目: まとめ
8 枚目: CTA（「保存して後で見返してね」「詳細はプロフィールのリンクから」）
```

## 4. ブログ

### SEO 構成

| 要素 | ルール |
|:---|:---|
| タイトル | ターゲットキーワードを含む。60 文字以内。数字・具体性を入れる |
| メタディスクリプション | 120〜155 文字。キーワードを自然に含む。読者のメリットを明示 |
| H1 | ページに 1 つだけ。タイトルと一致させる |
| H2 / H3 | キーワードのバリエーションを自然に含める。3〜6 個の H2 で構成 |
| 冒頭段落 | 最初の 100 語以内にターゲットキーワードを自然に配置 |
| 内部リンク | 関連記事へのリンクを 3〜5 個。アンカーテキストは自然な表現で |
| 画像 | alt テキストにキーワードを含める。1 記事に 3〜5 枚 |

### スキャン性の確保

ブログ読者の 79% はスキャン（流し読み）する。読まれる記事にするためのルール:

| ルール | 具体的な基準 |
|:---|:---|
| 段落の長さ | 2〜4 文。5 文以上の段落は分割する |
| 箇条書き | 3 項目以上の列挙は箇条書きにする |
| 太字 | 各セクションの要点を太字にする（段落あたり 1〜2 箇所） |
| 視覚的な区切り | H2 の間隔は 200〜400 語。長すぎるセクションは H3 で分割 |
| 冒頭文の強さ | 各セクションの最初の 1 文で結論を述べる（逆ピラミッド構造） |
| 画像・図表 | 300〜500 語ごとにビジュアル要素を挿入 |

### CTA 配置

| 配置位置 | CTA の種類 | 例 |
|:---|:---|:---|
| 記事冒頭（導入直後） | コンテンツアップグレード | 「このチェックリストをダウンロード」 |
| 記事中盤 | 関連リソース | 「詳しくはこちらのガイドで解説しています」 |
| 記事末尾 | メインの CTA | 「無料トライアルを始める」 |
| サイドバー | メルマガ登録 | 「週 1 回のマーケティング Tips を受け取る」 |

### ブログ記事の構成テンプレート

**タイトル**: `{数字} + {ターゲットキーワード} + {ベネフィット}`（例: 「LP の CVR を 3 倍にする 7 つの見出しテクニック」）

| セクション | 語数目安 | 内容 |
|:---|:---|:---|
| 導入 | 100〜200 語 | 読者の課題を提示 → この記事で得られるものを明示 → キーワード配置 |
| 本文 | 1,200〜2,500 語 | H2 で 3〜6 分割。各セクション: 結論→理由→具体例→実践方法 |
| まとめ | 100〜200 語 | 要点を箇条書きで再掲 → 次のアクション 1 つ → メイン CTA |

## チャネル横断の共通ルール

| ルール | 説明 |
|:---|:---|
| 1 チャネル 1 最適化 | 同じコピーを複数チャネルで使い回さない。各チャネル向けに書き直す |
| モバイルファースト | すべてのチャネルでモバイル表示を最優先で確認する |
| CTA は 1 つ | 1 つのコンテンツで求めるアクションは 1 つだけ |
| トーンの一貫性 | チャネルが変わってもブランドの人格は統一する |
| データで検証 | 勘ではなく A/B テストと KPI で判断する |
