# Skill: Marketing Copy（マーケティングコピー執筆・編集）

> **人を動かす言葉は「うまい文章」ではなく「正確な共感」から生まれる**

## Instructions

### ワークフロー内の位置

```
marketing-context → [marketing-copy] → review
                          ↓
                     LP / メール / SNS / ブログ記事
                          ↓
                     A/Bテスト案 + KPI測定ポイント
```

### 入力

| 入力 | 説明 | 例 |
|:---|:---|:---|
| コピーの目的 | 何のためのコピーか | 「SaaSのLP」「ウェルカムメール」 |
| ターゲット情報 | 理想的な読者像 | 「中小企業のマーケ責任者」 |
| ブランドボイス | トーン・人格の方向性 | 「プロフェッショナルだが親しみやすい」 |
| 既存コピー（編集時） | 改善対象のテキスト | LP全文、メール文面など |

### 出力

| 出力 | 形式 | 説明 |
|:---|:---|:---|
| コピー本文 | Markdown | チャネル別に最適化されたコピー |
| 見出し候補 | リスト | 3案以上のバリエーション + 根拠 |
| 編集レポート | Markdown | 7 Sweeps による改善点と修正提案 |
| テスト計画 | テーブル | A/Bテスト案と測定KPI |

## Step 1: マーケティングコンテキスト確認

ブランドボイスとターゲット情報を取得し、コピー全体の基盤を固める。

### 1a. コンテキスト情報の確認

marketing-context スキルで作成された `.claude/marketing-context.md` が存在する場合、先に読み込む。
存在しない場合、以下を確認する:

- **ブランドボイス**: トーンの方向性（カジュアル / プロフェッショナル / 権威的）
- **ターゲットペルソナ**: 理想顧客のプロフィール、課題、言語
- **バリュープロポジション**: 競合との差別化要因
- **社会的証明**: 導入実績、数値、推薦の声

### 1b. 顧客の言葉の収集

ターゲットが実際に使う言葉を収集する:

- レビューサイトの表現
- サポートチケットのフレーズ
- SNS上の不満・要望
- インタビュー・アンケートの生の声

**チェックリスト**:
### 品質ガードレール

**必須**: ブランドボイスガイドラインとの整合性を必ずチェックすること

- 誇大広告に該当する表現（景品表示法違反リスク）を使用してはならない
- 他社の著作物を出典明記なく引用しないこと

品質基準:
- メタディスクリプション 最大 160 文字
- CTA 数 ≥1 個/ページ

- [ ] ブランドボイスの方向性が明確である
- [ ] ターゲットの課題を「顧客の言葉」で記述できる
- [ ] 競合との差別化ポイントが特定されている
- [ ] 使用可能な社会的証明がリストアップされている

---

## Step 2: コピー目的の明確化

コンバージョン目標と読者の旅路を定義する。

### 2a. 目的の定義

- **コンバージョンゴール**: 読者に最終的にとってほしいアクション（1つだけ）
- **読者の現在地**: コピーに触れる時点で読者が知っていること・信じていること
- **読者の到達地点**: コピーを読み終えた後に読者がとる行動・持つ感情
- **チャネル**: LP / メール / SNS / ブログのどれか

### 2b. チャネル別の制約確認

チャネルによって文字数・構造・トーンが異なる。
詳細は [channel-guidelines.md](references/channel-guidelines.md) を参照。

| チャネル | 文字数目安 | 主な制約 |
|:---|:---|:---|
| LP | 制限なし | スクロール構造、CTA配置 |
| メール | 50〜500語 | 件名・プレビュー・モバイル最適化 |
| SNS | プラットフォーム依存 | フック重視、文字数制限 |
| ブログ | 1,500〜3,000語 | SEO、スキャン性、内部リンク |

**チェックリスト**:
- [ ] コンバージョンゴールが1つに絞られている
- [ ] 読者の「現在地」と「到達地点」のギャップが明確
- [ ] チャネル別の制約を把握している
- [ ] メールの場合、シーケンス全体か単発かを確認した

---

## Step 3: リサーチとインサイト

ターゲットの「声」と競合のコピーを分析し、差別化のためのインサイトを導出する。

### 3a. ターゲットの声（VOC）分析

顧客の言葉を以下の4カテゴリに整理する:

| カテゴリ | 収集する情報 | 活用方法 |
|:---|:---|:---|
| 不満・苦痛 | 「〜が面倒」「〜にうんざり」 | Problem セクション、共感フック |
| 願望・期待 | 「〜したい」「〜だったらいいのに」 | ベネフィット、見出し |
| 反論・不安 | 「でも〜が心配」「本当に効果ある?」 | FAQ、リスクリバーサル |
| 成功体験 | 「〜が解決した」「〜時間削減できた」 | 社会的証明、事例 |

### 3b. 競合コピー分析

競合の LP / メール / SNS を調査し、差別化ポイントを特定する:

- 競合が使っている言葉・フレーム
- 競合が言及していない角度
- 競合の弱いポイント（曖昧な約束、証拠不足）

**チェックリスト**:
- [ ] VOC を 4 カテゴリに整理した
- [ ] 顧客の生の言葉を 5 個以上収集した
- [ ] 競合のコピーを 2 社以上分析した
- [ ] 差別化できる切り口を 1 つ以上特定した

## Step 4: コピーフレームワーク選定

メッセージの構造を決定する最適なフレームワークを選択する。
詳細は [copy-frameworks.md](references/copy-frameworks.md) を参照。

### 4a. フレームワークの選択基準

| 状況 | 推奨フレームワーク | 理由 |
|:---|:---|:---|
| 読者が課題を認識している | PAS（Problem-Agitate-Solution） | 痛みの増幅→解決で行動を促す |
| ビフォー・アフターが明確 | BAB（Before-After-Bridge） | 変化の落差で感情を動かす |
| データ・証拠が豊富 | AIDA（Attention-Interest-Desire-Action） | 論理的に説得を積み上げる |
| ブランドストーリーが強い | StoryBrand | 顧客をヒーローに据えた物語構造 |
| 短いコピーが必要 | 4Ps（Promise-Picture-Proof-Push） | 最小限の要素で説得を完結 |
| コールドメール | QVC / Observation-Problem-Proof-Ask | 簡潔さとパーソナライゼーション |

### 4b. 見出しフォーミュラの選択

見出しは最も重要な一文。以下のパターンから選定する:

- **成果型**: 「{成果} を {痛みなしに} 実現する」
- **課題型**: 「もう {嫌なこと} に悩まない」
- **対象型**: 「{対象} のための {カテゴリ}」
- **差別化型**: 「{従来と正反対の方法} で {成果} を」
- **証拠型**: 「{数字} の {人々} が {製品} で {成果}」

**チェックリスト**:
- [ ] チャネルと読者の認知段階に合ったフレームワークを選択した
- [ ] 見出しフォーミュラを 2〜3 パターン選定した
- [ ] フレームワークの各要素に対応するコンテンツ素材がある
- [ ] メールの場合、件名戦略も併せて選定した

## Step 5: ドラフト執筆

見出し・リード・本文・CTA の各パーツを執筆する。

### 5a. 見出しとサブ見出し

- **見出し**: バリュープロポジションの核心を 1 文で。3 案以上作成する
- **サブ見出し**: 見出しを補足し、具体性を加える。1〜2 文
- 各見出しに選定根拠（どのフォーミュラ、なぜこの角度）を添える

### 5b. リード（冒頭）

読者を引き込む最初の数行。以下のアプローチから選択:

- **共感型**: 読者の痛みを代弁する（「〜で困っていませんか?」）
- **驚き型**: 意外な事実やデータで注意を引く
- **質問型**: 読者に自分の状況を考えさせる
- **ストーリー型**: 具体的なシーンから始める

### 5c. 本文

フレームワークに沿って本文を構成する:

- **1 セクション = 1 メッセージ**。複数の主張を混ぜない
- **ベネフィット > 機能**。「何ができるか」ではなく「何が変わるか」
- **具体 > 抽象**。「業務効率化」→「週4時間の報告作業を15分に」
- **能動態 > 受動態**。「レポートが生成されます」→「ワンクリックでレポートを生成」

### 5d. CTA（行動喚起）

弱い CTA を避け、具体的なアクション + 得られるものを示す:

| 弱い CTA（避ける） | 強い CTA（使う） |
|:---|:---|
| 送信 | 無料トライアルを始める |
| 詳細はこちら | 導入事例を見る |
| お問い合わせ | 30日間無料で試す |
| 登録 | チェックリストをダウンロード |

**CTA の公式**: [動詞] + [得られるもの] + [補足条件]

### 5e. コピーのスタイルルール

1. **シンプル > 複雑** — 「活用する」ではなく「使う」、「促進する」ではなく「助ける」
2. **具体 > 曖昧** — 「最適化」「革新的」「シームレス」を避ける
3. **能動 > 受動** — 主語を明確に、動作を前面に
4. **自信 > 逡巡** — 「ほぼ」「だいたい」「おそらく」を削除
5. **見せる > 語る** — 副詞で説明せず、結果で示す
6. **誠実 > 誇張** — 架空の統計や推薦文は絶対に作らない

**チェックリスト**:
- [ ] 見出しを 3 案以上作成し、それぞれに根拠を添えた
- [ ] リードが読者の注意を最初の 1〜2 行で引いている
- [ ] 本文が 1 セクション 1 メッセージの原則に従っている
- [ ] CTA が「動詞 + 得られるもの」の公式に従っている
- [ ] スタイルルール 6 項目をすべて遵守している

---

## Step 6: 7 Sweeps 編集

完成したドラフトを 7 つの観点で順次チェックし、品質を体系的に磨き上げる。
各 Sweep の詳細手順は [seven-sweeps.md](references/seven-sweeps.md) を参照。

### Sweep 1: Clarity（明瞭性）

読者がすぐに理解できるか?

- 1 文で 1 つの情報だけを伝えているか
- 専門用語を説明なしに使っていないか
- 代名詞の指示先が曖昧でないか
- 前提知識を過剰に仮定していないか

### Sweep 2: Specificity（具体性）

曖昧な表現を具体的な数字・事実に置換する:

| 曖昧（NG） | 具体（OK） |
|:---|:---|
| 時間を節約 | 週4時間を削減 |
| 多くの企業 | 2,847社 |
| すぐに結果 | 14日で効果を実感 |
| サポート充実 | 2時間以内に返信 |

### Sweep 3: Conversational（会話調）

声に出して読んで自然か? 企業語や硬い表現を人間の言葉に変換する。

### Sweep 4: Sensory（感覚的表現）

読者が場面を「見て・聞いて・感じられる」か? 抽象を具象に変える:

- NG: 「ストレスが軽減されます」
- OK: 「月曜の朝、受信トレイを開いても胃がキリキリしなくなる」

### Sweep 5: Flow（流れ）

セクション間の論理的なつながりは自然か? 読者が迷わず最後まで読み進められるか?

### Sweep 6: Cut（削減）

不要な単語・文・セクションを容赦なく削る:

- 「非常に」「本当に」「まさに」 → 削除
- 「〜するために」→「〜に」
- 「〜することが可能です」→「〜できます」
- 同じことを別の言い方で繰り返していないか

### Sweep 7: Power（説得力）

最終チェック。感情を動かし、行動を促す力があるか?

- 社会的証明は CTA の近くに配置されているか
- リスクリバーサル（保証・無料期間など）が明示されているか
- 読者が行動しない理由をすべて潰しているか

### Copy Slop 防止チェック（Distributional Convergence 対策）

7 Sweeps の後、最終的に「AI が書いた感」が残っていないかを確認する。LLM がコピーを生成する際にも distributional convergence が起きる。全案件で「同じ構造・同じトーン・同じ CTA」に収束することを防ぐため、以下のアンチパターンを確認する。

| # | Copy Slop パターン | 検出方法 | 対策 |
|:---|:---|:---|:---|
| CP-1 | **ボイスの均質化** | ブランドボイスが異なるはずの案件で同じトーンのコピー | marketing-context.md のブランドパーソナリティを毎行で参照。「この1文はブランドの何を体現しているか」を自問 |
| CP-2 | **フレームワークの機械的適用** | PAS/AIDA を機械的に適用し、全コピーが同じ構造 | フレームワークは骨格。肉付けはペルソナの VOC から。同じフレームワークでも表現の角度を変える |
| CP-3 | **見出しフォーミュラの固定化** | 同じ公式から同じパターンの見出しが繰り返される | 公式は出発点。最終形は顧客の言葉と業界の文脈で書き換える。「フォーミュラを外した1案」を必ず含める |
| CP-4 | **社会的証明のテンプレ化** | 「N社が導入」「満足度X%」のパターンが全案件で同じ | 証明の形式を変える（数値/ストーリー/推薦の声/ビフォーアフター）。業界固有の信頼シグナルを使う |
| CP-5 | **CTA の収束** | 全CTAが「無料で始める」「今すぐ試す」に集約 | CTAはペルソナの認知段階とチャネルで決まる。「動詞+得られるもの」の公式は守りつつ得られるものを具体的に |
| CP-6 | **7 Sweeps のチェックボックス化** | 7 Sweep を形式的に通過させるだけで実質的改善がない | 各 Sweep で「変更した箇所と理由」を記録する。Sweep 後に「AI が書いた感」が残っていないか最終チェック |
| CP-7 | **メールシーケンスの均一構造** | Welcome→教育→価値→オファー→フォローが全案件で同じ | シーケンスの長さ・間隔・トーン変化をプロダクトの複雑さと購買サイクルに合わせる |
| CP-8 | **VOC の捏造** | 「顧客の声」が実は AI が推測した汎用的な表現 | VOC は実データからの引用。データがなければ「仮説」と明記し、検証計画を添える |

> **Why**: コピーは顧客と直接触れるタッチポイントであり、均質化の影響が最も目に見える形で現れる。
> 「AI が書いた感」のあるコピーは、読者に「自分向けではない」という印象を与え、
> どれだけ正しいフレームワークを使っても共感も行動も生まれない。
>
> **核心原則**: **フレームワークは骨格、VOC は肉体、ブランドボイスは魂** — 構造はフレームワークに従いつつ、
> 表現の一つひとつは顧客の生の言葉とブランドの人格で満たす。「別の企業のコピーに差し替えても違和感がないか？」→ 違和感がないなら Copy Slop である。

**チェックリスト**:
- [ ] 7 Sweep すべてを順番に実施した
- [ ] 各 Sweep で「変更した箇所と理由」を記録した（CP-6）
- [ ] 各 Sweep 後に前の Sweep を再確認した（修正が新たな問題を生んでいないか）
- [ ] 削除した単語数 / 修正箇所を記録した
- [ ] 声に出して読んで自然に聞こえることを確認した
- [ ] Copy Slop チェック（CP-1〜CP-8）に 2 つ以上該当しない
- [ ] コピーのトーンが marketing-context.md のブランドボイスと一致している（CP-1）
- [ ] 見出し候補に「フォーミュラを外した1案」が含まれている（CP-3）
- [ ] CTA が認知段階とチャネルに合わせた具体的な表現になっている（CP-5）
- [ ] VOC が実データに基づいている、または「仮説」と明記されている（CP-8）

## Step 7: チャネル別最適化

LP / メール / SNS / ブログそれぞれの特性に合わせてコピーを調整する。
詳細は [channel-guidelines.md](references/channel-guidelines.md) を参照。

### 7a. LP（ランディングページ）

```
ヒーロー（見出し + サブ見出し + CTA + ビジュアル）
↓ ソーシャルプルーフバー（ロゴ / 数値 / 推薦の声）
↓ 課題セクション（読者の痛みを代弁）
↓ ソリューション / ベネフィット（3〜5 個）
↓ 使い方（3〜4 ステップ）
↓ 詳細な推薦の声 / 事例
↓ FAQ（反論への回答）
↓ 最終 CTA + リスクリバーサル
```

### 7b. メール

- **件名**: 2〜4 語、内部メール風、小文字推奨
- **プレビューテキスト**: 件名を補完（90〜140 文字）
- **本文**: Hook → Context → Value → CTA → Sign-off
- **長さ**: トランザクション 50〜125 語 / 教育 150〜300 語 / ストーリー 300〜500 語

メールシーケンスのテンプレートは [email-templates.md](references/email-templates.md) を参照。

### 7c. SNS

- **LinkedIn**: フック（1行目で「もっと見る」を押させる）、1,200〜1,500 文字、リンクはコメント欄に
- **Twitter/X**: 100 文字以下、スレッドは教育型に、ビジュアル活用
- **Instagram**: リール重視、カルーセルで教育コンテンツ、ストーリーで親密さ

### 7d. ブログ

- **構成**: 導入（問題提起）→ 本文（H2/H3で構造化）→ まとめ（CTA）
- **スキャン性**: 短い段落、箇条書き、太字で要点強調
- **SEO**: ターゲットキーワードを見出しと冒頭段落に自然に配置

**チェックリスト**:
- [ ] 選択したチャネルのガイドラインに準拠している
- [ ] モバイル表示を考慮したフォーマットになっている
- [ ] メールの場合、件名とプレビューテキストを作成した
- [ ] SNS の場合、プラットフォーム固有の文字数制限を守っている

## Step 8: テストと改善提案

コピーの効果を測定し、継続的に改善するための計画を策定する。

### 8a. A/Bテスト案の設計

テストする要素を優先順位付けする:

| 優先度 | テスト対象 | テスト方法 | 測定指標 |
|:---|:---|:---|:---|
| 高 | 見出し | 2案のうち勝者を判定 | クリック率 / 滞在時間 |
| 高 | CTA 文言 | ボタンテキストの比較 | コンバージョン率 |
| 中 | メール件名 | 開封率で比較 | 開封率 |
| 中 | リード文 | 離脱率で比較 | 直帰率 |
| 低 | 本文の長さ | 長短 2 バージョン | 完読率 / CV率 |

### 8b. KPI測定ポイント

チャネル別の成功指標:

| チャネル | 主要KPI | ベンチマーク |
|:---|:---|:---|
| LP | コンバージョン率 | 2〜5%（業界平均） |
| メール（ナーチャー） | 開封率 / クリック率 | 20〜40% / 2〜5% |
| メール（コールド） | 返信率 | 5〜10%（良好） |
| SNS | エンゲージメント率 | 1〜3%（LinkedIn） |
| ブログ | 平均滞在時間 | 2〜4分 |

### 8c. 改善サイクル

1. テスト実施 → 2. データ収集 → 3. 勝者判定 → 4. 学びを記録 → 5. 次のテスト設計

**チェックリスト**:
- [ ] A/Bテスト案を優先度順に 2〜3 個提案した
- [ ] 各テストの測定指標と成功基準を定義した
- [ ] チャネル別のKPIベンチマークを提示した
- [ ] テスト結果の記録方法を提案した

## Troubleshooting

| 問題 | 原因 | 解決策 |
|:---|:---|:---|
| コピーが企業語・お役所言葉になる | 社内で使う表現をそのまま転用している | 「友人に説明するなら?」と自問し、会話調に書き直す |
| 見出しがクリックされない | 抽象的・汎用的すぎる | 具体的な数字・期間・対象者を入れ、成果型フォーミュラを使う |
| CTA のコンバージョン率が低い | 「送信」「登録」など曖昧な文言 | 「動詞 + 得られるもの」公式に変更し、リスクリバーサルを添える |
| メールの開封率が低い | 件名が長い / 売り込みに見える | 2〜4語の内部メール風件名に変更。小文字、売り込み語を排除 |
| コピーが長すぎて読まれない | すべてを1回で伝えようとしている | 1 セクション 1 メッセージの原則を徹底。Cut Sweep で削減 |
| ターゲットに刺さらない | 顧客の言葉ではなく自社の言葉で書いている | VOC 分析をやり直し、顧客のレビュー・SNS から表現を借りる |
| 7 Sweeps の途中で迷う | 複数の観点を同時にチェックしようとしている | 1 Sweep = 1 観点を厳守。各 Sweep 後に前の Sweep を再確認 |
| SNS 投稿のエンゲージメントが低い | フック（1行目）が弱い | Curiosity / Story / Contrarian フック公式を使って書き直す |
| コールドメールの返信率が低い | パーソナライゼーションが問題と無関係 | 「パーソナライゼーションを削除しても意味が通るか」テストする。問題に直結させる |
| メールシーケンスで離脱が多い | 各メールが同じ角度で同じことを言っている | アングルローテーション: 各メールで異なる価値提案・証拠・視点を提供する |
| ブログ記事がスキャンしにくい | 段落が長く、視覚的な区切りがない | 2〜4文の短い段落、箇条書き、太字の要点、H2/H3の構造化を適用 |
| A/Bテストで有意差が出ない | テスト対象が微差 / サンプル不足 | 大胆に異なるバリエーションをテストし、十分なサンプルサイズを確保する |
| 全案件のコピーが同じトーン・構造になる | Copy Slop（distributional convergence） | CP-1〜CP-8 を確認。特にボイス（CP-1）・フレームワーク適用（CP-2）・CTA（CP-5）が案件固有の表現になっているか検証する |
| 7 Sweeps を実施しても「AI が書いた感」が残る | Sweep を形式的に通過させているだけ（CP-6） | 各 Sweep で具体的に「何を変えたか・なぜ変えたか」を記録する。最終的に「人間のマーケターが書いたように読めるか」を第三者に確認する |
| 顧客の声（VOC）が実在感に欠ける | LLM が汎用的な VOC を生成している（CP-8） | VOC は必ず実データ（レビュー、インタビュー、SNS）から引用する。データがなければ「仮説 VOC」と明記し、検証方法を併記する |
| 不正なフォーマットのコピーが出力される | 入力テンプレートの構造エラーや不備 | バリデーションチェックと想定外入力のエラーハンドリング |

## アンチパターン検出

このスキルの出力品質を検証し、AI Decay（品質劣化）を防止するためのアンチパターン検出リスト。各項目は品質劣化パターンの検出条件を定義し、不合格時は対応する対策（修正ガイドライン）に従って改善する。

- [ ] コピーがブランドボイスの具体的なトーン指示に従っているか（汎用的な「プロフェッショナル」トーンに収束していないか）
- [ ] CTA が具体的なアクションを示しているか（「詳しくはこちら」等の曖昧表現を排除）
- [ ] 数値・実績データがコピーに含まれているか（「多くのお客様に選ばれています」ではなく具体的な数字）
- [ ] 7 Sweeps 編集フレームワークの全 Sweep が実行されているか
- [ ] ターゲットペルソナの「顧客の言葉」がコピーに反映されているか（マーケター目線の言葉遣いになっていないか）
- [ ] チャネル別（LP/メール/SNS/ブログ）の文字数・構成ガイドラインに準拠しているか
