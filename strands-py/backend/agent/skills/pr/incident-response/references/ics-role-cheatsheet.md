# ICS Role Cheatsheet

> incident-response スキル補足資料 — ICS（Incident Command System）役割早見表

## 1. 役割一覧と責務

### Incident Commander (IC)

| 項目 | 内容 |
|:---|:---|
| **主な責務** | 全体統括、役割割当、障害排除、意思決定 |
| **委任ルール** | 委任していない役割はすべて IC が担う |
| **最初のアクション** | インシデント宣言、Slack チャンネル開設、役割割当 |
| **禁止事項** | 技術的な調査に没頭すること（Ops に委任） |
| **ハンドオフ** | 「あなたが IC です、OK?」と口頭確認 + チーム通知 |

**IC の判断フレームワーク**:
1. 影響範囲は広がっているか？ → 人を増やす
2. 現在のアプローチは機能しているか？ → 代替手段を検討
3. チームの疲労は蓄積しているか？ → ハンドオフを計画
4. ステークホルダーは最新情報を持っているか？ → 通知更新

### Operations Lead (Ops)

| 項目 | 内容 |
|:---|:---|
| **主な責務** | システム操作、運用ツール適用、技術的調査 |
| **権限** | インシデント中にシステム変更できる**唯一の**チーム |
| **最初のアクション** | ダッシュボード確認、影響範囲の特定、緩和策の実行 |
| **報告先** | IC に定期的にステータス更新 |
| **注意点** | 「直す」前に「止血」。根本原因追究は後回し |

### Communications Lead (Comms)

| 項目 | 内容 |
|:---|:---|
| **主な責務** | ステータスページ更新、ステークホルダー通知 |
| **更新頻度** | 最低 **4 時間ごと**（重大インシデントは 30 分ごと） |
| **最初のアクション** | ステータスページの初報投稿 |
| **通知先** | 社内（経営、CS、BD）+ 社外（ステータスページ） |
| **注意点** | 技術用語を避け、ユーザー影響を明確に伝える |

**ステータス更新テンプレート**:
```
[YYYY-MM-DD HH:MM UTC] [Investigating/Identified/Monitoring/Resolved]
影響: <ユーザーへの影響の説明>
状況: <現在の対応状況>
次回更新: <次回更新予定時刻>
```

### Planning Lead

| 項目 | 内容 |
|:---|:---|
| **主な責務** | バグ起票、ハンドオフ調整、長期課題管理 |
| **最初のアクション** | インシデントドキュメントの初期化 |
| **追加責務** | 食事手配、交代要員の調整、会議室確保 |
| **長期インシデント** | シフト計画の策定、引き継ぎドキュメント作成 |
| **注意点** | 人的ケアも含む後方支援が本質 |

## 3. 重大度（Severity）レベル

| レベル | 基準 | 対応体制 | 通知範囲 |
|:---|:---|:---|:---|
| **SEV-1** | 全ユーザー影響、収益停止 | 全チーム動員、IC 必須 | 経営層 + 全社 |
| **SEV-2** | 一部ユーザー影響、機能劣化 | 関連チーム、IC 推奨 | VP + 関連部門 |
| **SEV-3** | 軽微な影響、回避策あり | オンコール対応 | チーム内 |
| **SEV-4** | ユーザー影響なし、予防的対応 | 通常営業時間で対応 | チケットのみ |

## 4. コミュニケーション規律

### チャットプロトコル

| ルール | 理由 |
|:---|:---|
| #incident-XXX に全通信を集約 | ログが残る、後のポストモーテムに活用 |
| DM での技術議論禁止 | 情報の散逸を防止 |
| 重要な決定は明文化 | 口頭合意は消える |
| @mention で指名、曖昧な依頼禁止 | 「誰かやって」は誰もやらない |
| タイムスタンプ付きで更新 | タイムラインの再構成に必須 |

### ハンドオフチェックリスト

- [ ] 現在の状況サマリを口頭 + 文書で共有
- [ ] 未完了のアクションアイテムを引き継ぎ
- [ ] 後任が IC/Ops/Comms/Planning の**どの役割を引き継ぐか**明示
- [ ] チームに「@新担当 が IC を引き継ぎました」と通知
- [ ] 引き継ぎ時点のタイムラインをドキュメントに記録
