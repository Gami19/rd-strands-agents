# 広告コピーフレームワーク集

> 実績のある広告コピーの構成パターン。各フレームワークの構造、適用シーン、具体例を網羅する。

---

## 1. AIDA（Attention → Interest → Desire → Action）

最も古典的な広告構成フレームワーク。認知から行動までの全段階をカバーする。

### 構造

```
[A] 注目を引く（Attention）
[I] 興味を持たせる（Interest）
[D] 欲求を喚起する（Desire）
[A] 行動を促す（Action）
```

### 適用シーン

- Meta Ads のプライマリテキスト（長文可能な配置）
- LinkedIn Ads のイントロテキスト
- ランディングページ連動の広告

### 広告コピー例

**プライマリテキスト（Meta Ads）**:

```
レポート作成に毎週5時間かけていませんか？ [A]
DataFlowなら、10以上のデータソースを自動で統合。 [I]
導入企業は平均75%の工数削減を実現しています。 [D]
14日間の無料トライアルを今すぐ開始 → [A]
```

### 要素別の長さの目安

| 段階 | Google RSA | Meta プライマリ | LinkedIn イントロ |
|:---|:---|:---|:---|
| Attention | ヘッドライン1本 | 1行（30文字） | 1行（40文字） |
| Interest | ヘッドライン1本 | 1-2行（50文字） | 1行（40文字） |
| Desire | ディスクリプション | 1-2行（50文字） | 1行（40文字） |
| Action | CTA ヘッドライン | 1行（CTA） | CTA ボタン |

## 2. PAS（Problem → Agitate → Solve）

課題認知層に最も効果的なフレームワーク。痛みを増幅してから解決策を提示する。

### 構造

```
[P] 問題を提示する（Problem）
[A] 痛みを増幅する（Agitate）
[S] 解決策を提示する（Solve）+ CTA
```

### 適用シーン

- 課題認知段階のオーディエンス
- Google Ads の検索広告（課題系キーワード）
- Meta Ads のリターゲティング

### 広告コピー例

**Google RSA**:

```
ヘッドライン: 「手作業のレポートにうんざり？」[P]
ヘッドライン: 「毎週5時間の無駄を解消」[A]
ヘッドライン: 「DataFlowで自動化を始める」[S]
ディスクリプション: 「手動でデータを集めてExcelに貼る日々は終わり。
DataFlowが10+ソースを自動統合。無料で試す →」[PAS全体]
```

**Meta Ads**:

```
まだ毎週レポートを手作業で作っていますか？ [P]
あなたがExcelと格闘している間に、競合はデータドリブンな意思決定をしています。 [A]
DataFlowなら3クリックでレポート完成。14日間無料 → [S]
```

## 3. BAB（Before → After → Bridge）

変化のストーリーを描くフレームワーク。ビフォーアフターの対比で説得力を高める。

### 構造

```
[B] 現在の痛みある状態（Before）
[A] 理想の未来の状態（After）
[B] プロダクトが橋渡し（Bridge）
```

### 適用シーン

- 解決策探索段階のオーディエンス
- ビジュアル広告（画像や動画と組み合わせ）
- テスティモニアル形式の広告

### 広告コピー例

```
Before: 承認プロセスがメール・Slack・スプレッドシートに散乱。 [B]
After: すべての承認が一元管理され、期限通りに完了。 [A]
TaskFlowがあなたのツールをつなぎ、プロジェクトを前に進めます。 [Bridge]
```

## 4. 4Us（Urgent, Unique, Ultra-specific, Useful）

ヘッドラインの品質を評価・改善するための4軸フレームワーク。

### 4つの軸

| 軸 | 説明 | 弱い例 | 強い例 |
|:---|:---|:---|:---|
| **Urgent（緊急性）** | 今すぐ行動する理由 | 「ツールをお試しください」 | 「今月末まで初月無料」 |
| **Unique（独自性）** | 他にない価値 | 「便利なツール」 | 「AIが自動でレポート生成」 |
| **Ultra-specific（超具体性）** | 数値・詳細の具体性 | 「時間を節約」 | 「レポート作成を週5時間→30分に」 |
| **Useful（有用性）** | 読者にとっての利益 | 「高機能ダッシュボード」 | 「経営判断に必要な数字が1画面で」 |

### 使い方

既存のヘッドラインを4Usで採点（各1-5点）し、弱い軸を強化する:

```
「データ分析ツール」
  U(rgent): 1 — 緊急性なし
  U(nique): 1 — 汎用的すぎる
  U(ltra-specific): 1 — 何をするか不明
  U(seful): 2 — 漠然とした有用性
  合計: 5/20 → 改善が必要

「来月から報告書を自動化。先着50社は初期費用無料」
  U(rgent): 5 — 先着・期限あり
  U(nique): 3 — 自動化という具体性
  U(ltra-specific): 4 — 50社、初期費用
  U(seful): 4 — 明確な利益
  合計: 16/20 → 強いヘッドライン
```

## 5. FAB（Feature → Advantage → Benefit）

機能訴求を顧客視点のベネフィットに変換するフレームワーク。

### 構造

```
[F] 機能（Feature）— プロダクトが持つもの
[A] 利点（Advantage）— 機能がもたらす技術的優位性
[B] 便益（Benefit）— ユーザーの生活がどう変わるか
```

### FAB 変換表の例

| Feature | Advantage | Benefit |
|:---|:---|:---|
| リアルタイムダッシュボード | データが常に最新 | 古い数字で誤った判断をしなくなる |
| AI 自動分類 | 手動仕分け不要 | 毎日30分の単純作業から解放される |
| Slack 連携 | 通知がSlackに届く | 別画面を開かずにチームで状況を共有 |
| SOC2 認証取得 | セキュリティ基準適合 | 情シスの審査を最短で通過できる |

### 広告への適用

```
ヘッドライン（Feature）: 「AI自動レポート生成」
ディスクリプション（Benefit）: 「毎週のデータ集計から解放。
AIが10+ソースを統合し、経営陣向けレポートを自動生成。」
```

**重要**: 広告では常に Benefit（便益）を前面に出す。Feature だけでは響かない。

## 6. 社会的証明リード

信頼構築を最優先するフレームワーク。競争が激しい市場で差別化に有効。

### 構造

```
[証拠] 印象的な数値・テスティモニアル
[説明] プロダクトの概要
[CTA] 行動喚起
```

### バリエーション

| タイプ | 例 |
|:---|:---|
| **顧客数** | 「10,000+チームが利用中」 |
| **実績数値** | 「導入企業の平均CTR改善率42%」 |
| **テスティモニアル** | 「"レポート作成が75%短縮された" — マーケ部長 田中氏」 |
| **受賞歴** | 「G2 2024年 Best Software 受賞」 |
| **メディア掲載** | 「日経クロステック掲載」 |

## 7. 直接レスポンス

明確な主張と即座のアクションを求めるフレームワーク。コンバージョン直近のオーディエンスに最適。

### 構造

```
[主張] 大胆な成果・約束
[証拠] 裏付けとなる数値・事実
[CTA] 緊急性を含むアクション
```

### 広告コピー例

```
ヘッドライン: 「レポート作成時間を80%削減」
ディスクリプション: 「5,000+のマーケティングチームが実証済み。
14日間無料 — 今月末まで初期費用0円。」
```

## フレームワーク選択ガイド

| 認知段階 | 推奨フレームワーク | 理由 |
|:---|:---|:---|
| **未認知** | PAS, 好奇心フック | 課題を顕在化させる |
| **課題認知** | PAS, AIDA | 痛みを増幅し解決策を提示 |
| **情報収集** | BAB, FAB, 社会的証明 | 比較材料と信頼性を提供 |
| **比較検討** | 4Us, 直接レスポンス, 社会的証明 | 具体性と緊急性で決断を促す |
| **購買直前** | 直接レスポンス | オファーとCTAで背中を押す |
