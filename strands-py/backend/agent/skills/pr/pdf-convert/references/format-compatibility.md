# 入力フォーマット互換性マトリクス

> Docling の対応フォーマット一覧、機能別互換性、変換精度、制限事項、エンコーディング問題のトラブルシューティング

---

## 1. 対応フォーマットと機能マトリクス

### フォーマット別機能対応

| 機能 | PDF (Text) | PDF (Scan) | DOCX | PPTX | XLSX | HTML | PNG/JPG/TIFF |
|:---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **テキスト抽出** | A | B* | A | B | B | A | C* |
| **見出し階層** | A | C* | A | B | -- | A | -- |
| **テーブル認識** | A | B* | A | B | A | A | C* |
| **箇条書き** | A | B* | A | B | -- | A | -- |
| **数式** | B | D* | B | D | -- | B | -- |
| **画像抽出** | B | -- | B | B | -- | B | -- |
| **コードブロック** | B | D* | B | -- | -- | A | -- |
| **ページ構造** | A | B* | A | A | -- | B | -- |
| **メタデータ** | A | B | B | B | B | B | D |

**凡例**: A = 高精度 / B = 実用的 / C = 限定的 / D = 非対応に近い / -- = 非該当 / * = OCR 依存

### 入力フォーマット別の推奨用途

| フォーマット | 推奨用途 | 非推奨用途 |
|:---|:---|:---|
| **PDF（テキスト）** | RFP、仕様書、報告書、契約書 | — |
| **PDF（スキャン）** | 紙書類のデジタル化、古い文書 | 大量の図面（CAD 等） |
| **DOCX** | 議事録、提案書、仕様書 | 複雑なレイアウト（段組み多数） |
| **PPTX** | プレゼン資料、研修資料 | データ分析資料（Excel 推奨） |
| **XLSX** | データ定義書、項目一覧 | 複雑なマクロ付きシート |
| **HTML** | Web ドキュメント、API ドキュメント | SPA（JavaScript 動的生成） |
| **画像** | ホワイトボード写真、スクリーンショット | 低解像度の写真 |

## 2. 各フォーマットの制限事項

### PDF 固有の制限

| 制限 | 詳細 | 回避策 |
|:---|:---|:---|
| **パスワード保護** | 暗号化された PDF は変換不可 | パスワード解除後に変換 |
| **フォーム入力フィールド** | PDF フォームの入力値が取得できない場合がある | フォームを「フラット化」してから変換 |
| **透かし（Watermark）** | テキストとして抽出される場合がある | 変換後に手動除去 |
| **段組みレイアウト** | 2 段組み以上の場合、テキスト順序が乱れる場合がある | VLM パイプラインで改善可能 |
| **縦書き** | 日本語縦書きの認識精度が低い | VLM + 手動補正 |
| **デジタル署名** | 署名画像がテキストに混入する場合がある | 変換後に手動除去 |

### DOCX 固有の制限

| 制限 | 詳細 | 回避策 |
|:---|:---|:---|
| **変更履歴** | Track Changes の中間状態は反映されない | 「すべての変更を反映」後に変換 |
| **コメント** | DOCX 内のコメントは抽出されない | コメントを本文に変換してから処理 |
| **SmartArt** | テキストは抽出されるが構造は失われる | 手動で構造を再構成 |
| **埋め込みオブジェクト** | Excel グラフ等の埋め込みは画像化 | 元データを個別に変換 |
| **マクロ (.docm)** | マクロは無視される | テキスト・構造のみ変換 |

### PPTX 固有の制限

| 制限 | 詳細 | 回避策 |
|:---|:---|:---|
| **アニメーション** | 全アニメーション情報が失われる | 最終表示状態のみ変換 |
| **スライドマスター** | マスターのテキストは抽出されない場合がある | 個別スライドの内容のみ確認 |
| **グラフ** | 画像として処理される | 元データの Excel を別途変換 |
| **ノート** | スライドノートは変換される | — |
| **図形内テキスト** | テキストボックス・図形内の文字は精度が落ちる | 手動補正 |

## 3. 画像・数式・表の変換精度

### 画像の扱い

| 入力形式 | 画像の変換方法 | 出力先 | 精度 |
|:---|:---|:---|:---|
| PDF 内の画像 | 参照リンクとして出力 | `![](image_N.png)` | 高 |
| DOCX 内の画像 | 抽出して参照 | `![](image_N.png)` | 高 |
| PPTX 内の画像 | 抽出して参照 | `![](image_N.png)` | 中 |
| 図中のテキスト | OCR/VLM で抽出試行 | 本文テキスト | 低-中 |

### 数式の変換精度

| 数式タイプ | 精度 | 出力形式 | 補足 |
|:---|:---|:---|:---|
| LaTeX 埋め込み | 高 | `$...$` / `$$...$$` | 元が LaTeX なら忠実に変換 |
| MathType / Equation Editor | 中 | LaTeX 近似 | 複雑な数式は手動補正が必要 |
| 画像化された数式 | 低 | テキスト近似 or 未変換 | VLM で精度向上可能 |
| 上付き/下付き文字 | 中 | HTML タグ or 近似 | Unicode で表現可能な範囲は高精度 |

### テーブルの変換精度

| テーブルタイプ | 精度 | 注意点 |
|:---|:---|:---|
| 単純テーブル（罫線あり） | A（高） | ほぼ完璧に変換 |
| セル結合テーブル | B（実用的） | 結合が崩れる場合あり。ACCURATE モード推奨 |
| 罫線なしテーブル | C（限定的） | 列の区切りが誤認識される場合あり |
| 入れ子テーブル | C（限定的） | 外側テーブルのみ認識される場合あり |
| カラーハイライト付きテーブル | B（実用的） | 色情報は失われるがテキストは保持 |
| ページをまたぐテーブル | B（実用的） | 自動結合されるが手動確認推奨 |

## 4. 文字化けの原因と対策

### 文字化けパターン別対策

| パターン | 症状 | 原因 | 対策 |
|:---|:---|:---|:---|
| **豆腐化（□□□）** | 文字が四角に置換 | フォント埋め込みなし + システムにフォントなし | 元 PDF のフォント情報を確認。フォントをインストールするか OCR で再変換 |
| **文字抜け** | 一部の文字が消失 | CID フォントのマッピング失敗 | VLM パイプラインで再変換。または OCR を有効化 |
| **文字化け（意味不明文字列）** | `{~*#` 等のゴミ文字 | エンコーディング不一致 | 入力ファイルのエンコーディングを確認し UTF-8 に変換 |
| **半角/全角の混在** | 数字やカッコの幅が不統一 | Unicode 正規化の不一致 | 変換後に NFC 正規化を適用 |
| **結合文字の分離** | 「が」が「か」+「゛」に | Unicode 結合文字の処理 | NFC 正規化で結合 |
| **旧字体/異体字** | 意図しない字形 | CJK 互換漢字の処理 | 手動で現代字体に修正 |

### エンコーディング問題の診断フロー

```
文字化け発生
  ├── 全体が文字化け?
  │     → 入力ファイルのエンコーディングを確認
  │       ├── Shift_JIS → UTF-8 に変換してから docling 実行
  │       ├── EUC-JP → UTF-8 に変換してから docling 実行
  │       └── UTF-8 BOM → BOM を除去
  │
  ├── 一部の文字のみ?
  │     → フォント埋め込みの問題
  │       ├── フォント埋め込みあり → docling のバージョンを最新に
  │       └── フォント埋め込みなし → OCR で再変換
  │
  └── 特定の記号のみ?
        → Unicode 特殊文字の処理
          → 変換後に手動置換（例: ① → (1)）
```

## 5. フォーマット間の変換品質比較

### 総合品質スコア（5 点満点）

| 評価項目 | PDF (Text) | PDF (Scan) | DOCX | PPTX | HTML |
|:---|:---:|:---:|:---:|:---:|:---:|
| テキスト忠実度 | 5 | 3 | 5 | 4 | 5 |
| 構造の保持 | 4 | 2 | 5 | 3 | 4 |
| テーブル精度 | 4 | 3 | 5 | 3 | 5 |
| 処理速度 | 5 | 2 | 5 | 4 | 5 |
| 手動補正の必要性 | 低 | 高 | 最低 | 中 | 低 |
| **総合推奨度** | **4.5** | **2.5** | **5.0** | **3.5** | **4.5** |

> DOCX が最も高品質な変換結果を得られる。顧客に選択肢がある場合は DOCX での資料提供を推奨する。
