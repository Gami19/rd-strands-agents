## 1. LP（ランディングページ）最適化

### 分析フレームワーク（影響度順）

#### 1-1. バリュープロポジション（最高インパクト）

**診断チェック**:
- 訪問者が 5 秒以内に「何のサービスか」「なぜ自分に関係あるか」を理解できるか
- 機能ではなくベネフィットで語っているか
- 顧客の言葉（社内用語ではなく）で書かれているか

**改善パターン**:

| 改善前 | 改善後 | 原則 |
|:---|:---|:---|
| 「AIプラットフォーム」 | 「レポート作成を75%自動化」 | 具体性 |
| 「高機能な分析ツール」 | 「経営判断に必要なデータを1画面に」 | ベネフィット |
| 「DX推進ソリューション」 | 「月100時間の手作業をなくす」 | 顧客の言葉 |

#### 1-2. ヘッドライン

**強いヘッドラインのパターン**:
- **成果提示型**: 「[望む成果] を [痛みなく] 実現する」
- **数値訴求型**: 「[具体的数値] の [成果]」
- **社会的証明型**: 「[N社/N人] が選んだ [カテゴリ]」
- **問題提起型**: 「まだ [非効率な方法] で [作業] していませんか？」

**テスト優先度**: ヘッドライン変更は最も ROI の高い CRO 施策の 1 つ。

#### 1-3. CTA の配置・コピー・階層

**CTA コピーの強度**:

| レベル | 例 | 転換率への影響 |
|:---|:---|:---|
| 弱 | 「送信」「登録」「詳しくはこちら」 | 低い |
| 中 | 「無料で始める」「デモを見る」 | 中程度 |
| 強 | 「14日間無料で試す」「ROI を計算する」 | 高い |

**配置ルール**:
- ファーストビュー内にプライマリ CTA を 1 つ
- スクロール後の決断ポイントごとに CTA を繰り返す
- フッター手前に最終 CTA を配置

#### 1-4. 社会的証明と信頼シグナル

**効果的な配置**:
- **CTA の直下**: クリック直前の不安を解消
- **ベネフィット主張の直後**: 主張の裏付け
- **ファーストビュー内**: 初見の信頼構築

**信頼シグナルの種類と優先度**:

| 種類 | 例 | 説得力 |
|:---|:---|:---|
| 具体的数値 | 「顧客満足度 98%」「NPS +72」 | 最高 |
| 顧客ロゴ | 大手企業のロゴ一覧 | 高 |
| テスティモニアル | 写真+名前+役職+具体的成果 | 高 |
| メディア掲載 | 「日経新聞に掲載」等 | 中 |
| セキュリティバッジ | SSL、SOC2、ISO27001 | 中（特に BtoB） |

## 2. フォーム最適化

### コアルール: 全フィールドにはコストがある

| フィールド数 | 転換率への影響（目安） |
|:---|:---|
| 1-3 フィールド | ベースライン |
| 4-6 フィールド | 10-25% 低下 |
| 7+ フィールド | 25-50% 低下 |

### フィールド別最適化

| フィールド | 最適化ポイント |
|:---|:---|
| **メールアドレス** | 単一フィールド、タイプミス検出、確認フィールド不要 |
| **名前** | 「氏名」1 フィールド vs 「姓」「名」分割をテスト |
| **電話番号** | 可能な限り任意に。必須の場合は理由を明記 |
| **会社名** | メールドメインから自動推定可能か検討 |
| **フリーテキスト** | 任意にする。文字数ガイダンスを提供 |

### マルチステップフォーム

**採用基準**: 6 フィールド以上の場合に検討する。

**設計原則**:
1. プログレスバーを表示（ステップ X / Y）
2. 簡単な項目から開始、機密項目は後半に
3. 1 ステップ 1 テーマ
4. 戻る操作を許可する
5. リフレッシュ時にデータを保持する

### 段階的コミットメントパターン

```
Step 1: メールアドレスのみ（低摩擦）
  ↓
Step 2: 名前、会社名（コミットメント構築）
  ↓
Step 3: 詳細質問（既に投資したので完了する動機が強い）
```

### エラーハンドリング

| NG パターン | OK パターン |
|:---|:---|
| 「入力エラーです」 | 「有効なメールアドレスを入力してください（例: name@company.com）」 |
| エラー時に全フィールドクリア | エラー時に入力済みデータを保持 |
| ページ上部にエラー表示 | エラーフィールドの直下に表示 |

## 3. 価格ページ改善

### 3 プラン構成の設計

| 要素 | 入門プラン | 推奨プラン | 上位プラン |
|:---|:---|:---|:---|
| **目的** | 参入障壁の除去 | 大多数のランディング先 | 高価値顧客の取り込み |
| **機能** | コア機能、制限付き | フル機能、妥当な上限 | 全機能、上位上限 |
| **価格** | アンカーの底 | 中央値 | 推奨の 2-3 倍 |
| **CTA** | 「無料で始める」 | 「おすすめ」バッジ | 「お問い合わせ」 |

### 価格心理学テクニック

| テクニック | 説明 | 実装例 |
|:---|:---|:---|
| **アンカリング** | 高価格を先に見せて基準を上げる | 上位プランを左に配置 |
| **チャームプライシング** | 端数価格は安く感じる | ¥4,980 vs ¥5,000 |
| **デコイ効果** | 劣った選択肢で推奨を引き立てる | 機能少+高価格の中間プラン |
| **フレーミング** | 「1日あたり○○円」表記 | 月額¥9,800 → 1日あたり約¥327 |
| **年額割引強調** | 月額比での節約額を明示 | 「年額プランで2ヶ月分お得」 |

### 価格ページの必須要素

- [ ] プラン比較テーブル（機能の有無を視覚的に表示）
- [ ] 推奨プランの明示（「最も人気」「おすすめ」バッジ）
- [ ] 年額/月額の切り替えトグル（節約額を強調）
- [ ] FAQ（価格に関する反論を処理）
- [ ] 信頼シグナル（CTA 付近にテスティモニアル）
- [ ] 保証（「30日間返金保証」等）
- [ ] 無料トライアル/フリーミアムの明示

## 4. CRO 実験テンプレート集

### ヒーローセクション実験

| テスト | 仮説 | 測定指標 |
|:---|:---|:---|
| ヘッドライン A/B | 具体的数値入りが抽象表現より CVR が高い | サインアップ率 |
| ヒーロー画像形式 | スクリーンショット > イラスト > 動画 | スクロール率、CVR |
| CTA ボタンカラー | コントラスト色が視認性を高め CTR 向上 | CTA クリック率 |
| ファーストビュー構成 | 社会的証明追加で信頼が向上 | サインアップ率 |

### フォーム実験

| テスト | 仮説 | 測定指標 |
|:---|:---|:---|
| フィールド数削減 | 5→3 フィールドで完了率 +20% | フォーム完了率 |
| マルチステップ化 | プログレスバー表示で離脱率低下 | 各ステップの離脱率 |
| ソーシャルログイン | Google ログイン追加で摩擦低減 | サインアップ率 |
| CTA コピー | 「送信」→「無料で始める」で CVR 向上 | CTA クリック率 |

### 価格ページ実験

| テスト | 仮説 | 測定指標 |
|:---|:---|:---|
| プラン表示順 | 高額→低額で推奨プラン選択率向上 | プラン別選択率 |
| 年額デフォルト | 年額表示デフォルトで年額契約率向上 | 年額/月額比率 |
| FAQ 追加 | 価格 FAQ で反論処理し CVR 向上 | 価格ページ→サインアップ率 |
