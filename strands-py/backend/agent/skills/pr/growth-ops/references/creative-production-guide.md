# クリエイティブ大量制作ガイド

> Figma 変数、Canva テンプレート、クリエイティブ自動化の実践ガイド。

---

## 1. テンプレート設計の基本原則

### 変数化すべき要素

クリエイティブの差し替え要素をすべて変数として定義する:

| 変数カテゴリ | 具体的な要素 | 変数化の方法 |
|:---|:---|:---|
| **テキスト** | ヘッドライン、サブコピー、CTA | テキスト変数 |
| **画像** | メインビジュアル、背景、アイコン | 画像変数 |
| **カラー** | アクセントカラー、背景色 | カラー変数 |
| **ロゴ** | 自社ロゴ、パートナーロゴ | 画像変数 |
| **数値** | 割引率、実績数値 | テキスト変数 |

### サイズ展開マスター

主要な広告サイズの一覧:

| プラットフォーム | フォーマット | サイズ (px) | アスペクト比 |
|:---|:---|:---|:---|
| Meta Feed | 画像広告 | 1200x628 | 1.91:1 |
| Meta/Instagram | スクエア | 1080x1080 | 1:1 |
| Meta Stories | ストーリーズ | 1080x1920 | 9:16 |
| Google Display | レクタングル | 300x250 | — |
| Google Display | リーダーボード | 728x90 | — |
| LinkedIn | スポンサードコンテンツ | 1200x627 | 1.91:1 |
| LinkedIn | スクエア | 1080x1080 | 1:1 |
| X (Twitter) | カード | 1200x628 | 1.91:1 |

## 2. Figma を使った大量制作

### Step 1: マスターコンポーネントの設計

1. **ベースフレーム作成**: 最も使用頻度の高いサイズ（例: 1200x628）で設計
2. **レイヤー構成の標準化**:
   ```
   Frame: ad-banner-master
   ├── Background（背景）
   ├── Image-main（メインビジュアル）
   ├── Text-headline（ヘッドライン）
   ├── Text-subline（サブコピー）
   ├── CTA-button（CTAボタン）
   ├── Logo（ロゴ）
   └── Badge（任意: キャンペーンバッジ）
   ```
3. **コンポーネント化**: マスターデザインをコンポーネントに変換

### Step 2: 変数の設定

Figma Variables でテキスト（headline, subline, cta-text）、カラー（accent-color, bg-color）を変数として定義する。

### Step 3: バリアント生成

コンポーネントプロパティで訴求軸（課題解決/社会的証明/緊急性/成果訴求）、サイズ（feed/square/stories）、テーマ（light/dark）を切り替え可能にする。

### Step 4: 一括書き出し

**推奨プラグイン**:
- **Export Kit**: 全バリアントを一括で PNG/JPG/SVG に書き出し
- **Figma to Sheets**: 変数データをスプレッドシートと連携
- **Batch Styler**: スタイルの一括変更

**命名規則**:
```
[訴求軸]_[サイズ]_[バージョン].[拡張子]
例: benefit_feed_v1.png
    social-proof_square_v2.png
```

## 3. Canva を使った大量制作

### ブランドキットの設定

Canva のブランドキット機能で以下を登録:

| 要素 | 登録内容 |
|:---|:---|
| **ロゴ** | メインロゴ、アイコン、白抜きバージョン |
| **カラーパレット** | プライマリ、セカンダリ、アクセント、背景色 |
| **フォント** | 見出し用、本文用、CTA用 |
| **テンプレート** | 承認済みのマスターデザイン |

### Bulk Create（一括作成）ワークフロー

1. **マスターテンプレート作成**: 差し替え箇所を `{{変数名}}` で定義
2. **CSV データの準備**:

```csv
headline,subline,cta,image_url
作業時間を75%削減,14日間無料トライアル,今すぐ始める,img001.png
1万社が導入済み,業界シェアNo.1の実績,導入事例を見る,img002.png
月100時間の作業を自動化,初月50%OFF,詳しく見る,img003.png
```

3. **Bulk Create 実行**: CSV をアップロードし、自動でバリエーション生成
4. **確認・微調整**: 文字のはみ出し、画像の切れ等を目視チェック

### Magic Resize（サイズ展開）

1 つのデザインから複数サイズを自動生成:
- 元デザイン（1200x628）→ 1080x1080、1080x1920、728x90 等
- 自動調整後、テキスト位置・画像クロップの微調整が必要

---

## 4. クリエイティブマトリクスの構築

訴求軸（課題解決・社会的証明・成果訴求・緊急性等）とサイズ（Feed・Square・Stories等）の掛け合わせでバリエーションを体系的に管理する。各セルに ID（例: A1-S1）を振り、制作進捗を追跡する。

---

## 5. 品質チェックと承認フロー

### 制作前チェック

- [ ] ブランドガイドラインとの整合性を確認した
- [ ] 訴求軸×サイズのマトリクスが承認されている
- [ ] テキストの文字数がプラットフォーム制約内である
- [ ] 画像の権利関係をクリアした

### 制作後チェック

- [ ] テキストのはみ出し・改行崩れがない
- [ ] 画像の切れ・ぼやけがない
- [ ] CTA が十分な視認性を持つ
- [ ] モバイルでの表示確認を実施した
- [ ] ファイル名が命名規則に準拠している
- [ ] 書き出し解像度が適切である（72dpi 以上）

### 承認フロー

```
制作者 → セルフチェック → レビュアー承認 → 最終確認 → 入稿
```

**Figma の場合**: コメント機能で指摘 → 修正 → Resolve で管理
**Canva の場合**: 共有リンクでレビュー → コメント → 修正
