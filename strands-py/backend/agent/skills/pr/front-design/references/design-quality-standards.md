# デザイン品質基準と AI slop 回避ガイド

> front-design スキルが生成するデザインの品質を担保するための基準。
> 全成果物はこのドキュメントのチェックリストを通過してから納品する。

---

## Distributional Convergence（分布的収束）

LLM のフロントエンドデザイン出力における根本問題。Anthropic が命名した概念。

> Without direction, Claude samples from this high-probability center.
> （方向性を与えなければ、Claude はこの高確率の中心からサンプリングする）
>
> --- Anthropic, "Improving Frontend Design through Skills" (2025-11-12)

**LLM は「知らない」のではなく「許可がない」。** デザインの知識は持っているが、指示がなければ最も安全で汎用的な選択肢（= 訓練データの最頻出パターン）に収束する。これが「どのサイトも同じに見える」AI slop の正体。

### 4次元ステアリング

Anthropic Applied AI チームが開発した ~400 トークンの「蒸留美学プロンプト」は、以下の4次元を個別に制御する。

| 次元 | 避けるべきデフォルト | 推奨する代替 |
|:---|:---|:---|
| **Typography** | Inter, Roboto, Open Sans, Lato, Arial, デフォルトシステムフォント | 独自フォントを Google Fonts から選定。ウェイト 100 vs 900 の極端コントラスト。サイズジャンプ 3x 以上 |
| **Color & Theme** | 白背景に紫グラデーション。均等に分散した控えめなパレット | 一貫した美学にコミット。CSS 変数で統一。**支配色 + シャープアクセント**。IDE テーマ・文化的美学からインスピレーション |
| **Motion** | 静的なページ、トランジションなし。散発的なマイクロインタラクション | CSS-only アニメーション優先。**ページロード時の段階的表示**（`animation-delay`）が最も効果的 |
| **Backgrounds** | `background: #ffffff` のべた塗り | CSS グラデーション重ね、幾何学パターン、ノイズテクスチャ、コンテキスト効果 |

### 蒸留美学プロンプトの核心

Anthropic の ~400 トークンプロンプトの本質は「**自覚させる + 具体的な禁止 + 代替の提示**」のセット。

1. **自覚させる**: 「あなたは一般的な出力に収束する傾向がある」と明示
2. **禁止を具体化する**: 「Inter を使うな」「紫グラデーションを避けよ」と名指しで制限
3. **代替を示す**: 「JetBrains Mono, Playfair Display, IBM Plex を使え」と具体的な選択肢を提供
4. **再収束を防ぐ**: 特定の代替に寄り過ぎることも警告（「Space Grotesk に収束しがち」）

> Interpret creatively and make unexpected choices that feel genuinely designed for the context.
> Vary between light and dark themes, different fonts, different aesthetics.
> It is critical that you think outside the box!
>
> --- Anthropic, Distilled Aesthetics Prompt

### 分離プロンプティング（Isolated Prompting）

全次元を一括制御せず、特定次元のみを指示する手法。

- **Typography Only**: フォント選定のみに集中 → 「フォント指示だけで他の次元も改善される」（Anthropic の発見）
- **Theme Constraint**: 特定の美学テーマ（Solarpunk, RPG, IDE theme 等）にロック → 一貫性が飛躍的に向上

この手法は front-design の Step 2d（テーマ駆動デザイン）で活用する。

## 品質チェックリスト

### a) アクセシビリティ

| チェック項目 | 基準 |
|:---|:---|
| テキストのコントラスト比 | WCAG AA 4.5:1 以上（通常テキスト） |
| 大テキストのコントラスト比 | WCAG AA 3:1 以上（18px 以上 or 14px bold 以上） |
| フォーカスインジケーター | 全インタラクティブ要素に `:focus-visible` を設定、`outline` を消さない |
| 色だけの情報伝達禁止 | エラーや状態を色だけで伝えない。アイコン・テキスト・パターンを併用 |
| 画像の alt テキスト | 装飾画像は `alt=""`、意味のある画像は内容を説明する alt を付与 |
| キーボードナビゲーション | Tab 順序が論理的。CTA やリンクは全てキーボードで到達・操作可能 |
| 動きの抑制 | `prefers-reduced-motion: reduce` でアニメーションを無効化する |
| タッチターゲット | ボタン・リンクは最低 44x44px のタッチ領域を確保 |

### b) パフォーマンス

| チェック項目 | 基準 |
|:---|:---|
| 重い `box-shadow` の回避 | ぼかし半径 50px 超の `box-shadow` を複数重ねない。1要素あたり最大2層 |
| `filter` の使用制限 | `blur()` は必要な要素のみ。ページ全体やスクロール中の要素に適用しない |
| フォント最適化 | `font-display: swap` を指定。ウェイトは必要最小限（通常2-3ウェイト） |
| 画像最適化 | WebP/AVIF を優先、`loading="lazy"` を実装、明示的な `width`/`height` を指定 |
| CSS の最小化 | 未使用のスタイルを含めない。CSS 変数で値を集約し重複を避ける |
| アニメーション | `transform` と `opacity` のみをアニメーションする。`left`/`top`/`width` のアニメーション禁止 |
| `will-change` | アニメーション対象に `will-change: transform` を付けてGPU合成を促す。乱用しない |

### c) ブラウザ互換性

| チェック項目 | 基準 |
|:---|:---|
| 対象ブラウザ | Chrome / Firefox / Safari / Edge の最新2バージョン |
| CSS Grid / Flexbox | フォールバック不要（対象ブラウザ全てで完全サポート済み） |
| `backdrop-filter` | Safari は `-webkit-` プレフィックス必須。Firefox 103+ で対応済み |
| `clamp()` | 対象ブラウザ全てで対応済み。フォールバック不要 |
| `color-mix()` | Chrome 111+ / Firefox 113+ / Safari 16.2+。古いブラウザ向けにはフォールバック色を検討 |
| コンテナクエリ | Chrome 105+ / Firefox 110+ / Safari 16+。レイアウトの代替手段をメディアクエリで用意 |
| `@layer` | Chrome 99+ / Firefox 97+ / Safari 15.4+。対象ブラウザで対応済み |
| ベンダープレフィックス | `backdrop-filter` の `-webkit-` のみ。他は不要 |

## AI slop 回避チェックリスト

「AI が作った感」の正体は、コンセプトとの接続がない汎用的な装飾の集合体。以下のアンチパターンを具体的に避ける。

| # | AI slop パターン | なぜダメか | 代替案 |
|:--|:---|:---|:---|
| 1 | 紫→青グラデーション背景 | AI ツールのデフォルト出力として最も認知されている。見た瞬間に「AI製」と判断される | コンセプトの業種・トーンからカラーを導出する。暖色系、アースカラー、モノクロなど選択肢は広い |
| 2 | Inter / Roboto / Arial をそのまま使用 | AI のデフォルトフォント。個性がなく、どのサイトも同じ印象になる | トーンに合ったフォントを選定する。高級感なら Playfair Display、テックなら Space Grotesk、和風なら Shippori Mincho など |
| 3 | 全て同じ角丸のカード | 均一な `border-radius: 12px` の繰り返しは思考停止のシグナル | 要素の役割で角丸を変える。小ボタンは 4-6px、カードは 8-12px、モーダルは 16px。あるいは角丸を使わない選択肢もある |
| 4 | 意味なくセンター寄せ | AI は「中央揃え」をデフォルトにしがち。全てセンターだとビジュアルヒエラルキーが消失する | コンテンツの種類に応じて配置を変える。ヒーローは左寄せ+右に画像、テキストブロックは左寄せ、CTAのみセンター等 |
| 5 | コンセプトと無関係な装飾 | 「きれいだから」で追加されたグラデーション、パーティクル、ウェーブは意図が説明できない | 装飾は「なぜこの装飾か」を1文で説明できるものだけ使う。建築系なら直線、自然系なら有機的曲線など |
| 6 | 全要素にドロップシャドウ | シャドウの乱用は画面を重くし、階層関係が不明瞭になる | シャドウは「浮いている」要素にだけ使う。カードホバー、モーダル、ドロップダウンなど。静的要素には不要 |
| 7 | 汎用ストック写真 | 「握手するビジネスマン」「ノートPC前の笑顔」は AI 生成とストック写真の区別がつかない | 実際のプロダクト写真、抽象テクスチャ、イラスト、パターン背景など、業種固有のビジュアルを選ぶ |
| 8 | 「Get Started」「Learn More」等の無個性 CTA 文言 | AI のテンプレート文言。ユーザーの行動と結びつかず、クリック率も低い | 具体的なアクションに言い換える。「無料で試す」「プランを見る」「30秒で登録」「デモを予約」 |
| 9 | 過剰な余白（クリックできる面積が少ない） | AI は余白を「おしゃれ」と学習しがちで、実用密度が下がる。情報にたどり着けない | 余白はリズムで設計する。見出し上は大きく、段落間は中程度、関連要素間は小さく。目的なく空けない |
| 10 | `background-clip: text` のグラデーション文字乱用 | 見出しを全てグラデーション文字にすると安っぽく見える。可読性も下がる | グラデーション文字は1ページ1箇所以下。使うなら最も強調したい見出しだけ。他は単色で十分 |
| 11 | 同じトーンの配色で全セクション統一 | 変化がなくページが単調になる。スクロールしても景色が変わらない | セクションごとに明度を切り替える。白背景→ダーク背景→白背景のリズムを作る |
| 12 | アイコンの乱用（意味が被るアイコン群） | AI は「視覚的に寂しいから」でアイコンを追加する。同じようなアイコンが並ぶと逆効果 | アイコンはテキストを補完する場合のみ。テキストだけで伝わるなら不要。使うなら同一アイコンセットで統一 |

## コンセプト駆動デザインの原則

AI slop を避ける根本的な解決策は、**全てのデザイン判断にコンセプトからの根拠を持たせる**こと。

### 1. 装飾の根拠を説明できるか

全ての装飾要素に対して「なぜこの装飾をこの場所に使うのか」を1文で説明できなければならない。

```
OK:  医療系サイトなので、清潔感を示す白基調 + 信頼感のブルーアクセント
OK:  カフェのLPなので、コーヒー豆のブラウン系グラデーション + 手書き風フォント
NG:  きれいだから紫→青グラデーション
NG:  流行っているからガラスモーフィズム
```

### 2. カラーはブランド/業種から論理的に導出する

| 業種/トーン | 導出されるカラー方向性 |
|:---|:---|
| テック / 先進性 | ダーク背景 + ネオンアクセント、またはクリーンなホワイト + 彩度高めのブルー系 |
| 金融 / 信頼感 | ネイビー + ゴールド、またはダークグリーン + ホワイト |
| 飲食 / 温かみ | アースカラー（テラコッタ、オリーブ、クリーム）+ 食材写真の色を反映 |
| ヘルスケア / 清潔感 | ホワイト基調 + ペールブルーまたはティール + ライトグレー |
| クリエイティブ / 遊び心 | 高彩度の補色配色、または黒白 + 1色アクセント |
| 高級 / エレガンス | ブラック + ゴールド or シャンパン、深いバーガンディ、ダークネイビー |

### 3. フォントはトーンとの整合性で選ぶ

| トーン | フォント方向性 | 具体例 |
|:---|:---|:---|
| 権威・信頼 | セリフ体 | Playfair Display, Cormorant Garamond, Shippori Mincho |
| モダン・テック | ジオメトリックサンセリフ | Space Grotesk, Outfit, Sora |
| 親しみ・カジュアル | ラウンデッドサンセリフ | Nunito, Quicksand, Zen Maru Gothic |
| 洗練・ミニマル | ネオグロテスク | Satoshi, General Sans, Noto Sans JP |
| クラフト・手作り | 手書き風/スラブセリフ | Caveat (手書き), Zilla Slab, Klee One |

### 4. 「削る」判断を恐れない

良いデザインは足し算ではなく引き算。以下を自問する:

- この要素を削除してもコンセプトは伝わるか？ → 伝わるなら削除
- この装飾がなくても情報は伝わるか？ → 伝わるなら削除
- 2つの似た装飾があるとき、どちらか1つで十分か？ → 十分なら片方を削除
