# カラーパレット設計ガイド

> サイトの印象を決定づけるカラー選定の原則・テクニック・テンプレートを集約。

| ヘルスケア / 医療 | ティール・ライトブルー | `#0D9488` `#38BDF8` | 清潔感・安心感 |
| EC / リテール | コーラル・マゼンタ | `#F43F5E` `#EC4899` | 購買意欲・華やかさ |
| 教育 / EdTech | イエロー・グリーン | `#EAB308` `#22C55E` | 成長・親しみやすさ |

## トーン別カラー傾向

| 伝えたいトーン | 推奨色相 | 彩度・明度の傾向 |
|:---|:---|:---|
| 信頼感・安定 | ブルー / ネイビー | 中〜低彩度、中明度 |
| 先進性・テック | パープル / ティール | 中〜高彩度、中明度 |
| 温かみ・親しみ | オレンジ / イエロー | 中彩度、高明度 |
| 高級感・洗練 | ブラック / ゴールド | 低彩度、低明度（ダーク基調） |
| 清潔感・ミニマル | ホワイト / ライトグレー | 極低彩度、極高明度 |
| エネルギー・情熱 | レッド / マゼンタ | 高彩度、中明度 |
| 自然・安心 | グリーン / アースカラー | 中彩度、中〜高明度 |

## WCAG AA コントラスト比 早見表

最小コントラスト比: **通常テキスト 4.5:1 / 大テキスト(18px以上bold, 24px以上) 3:1**

| テキスト色 | 背景色 | コントラスト比 | AA 通常 | AA 大 |
|:---|:---|:---|:---|:---|
| `#18181B`（gray-900） | `#FFFFFF` | 17.9:1 | PASS | PASS |
| `#18181B` | `#F4F4F5`（gray-50） | 15.4:1 | PASS | PASS |
| `#3B82F6`（blue-500） | `#FFFFFF` | 3.4:1 | FAIL | PASS |
| `#2563EB`（blue-600） | `#FFFFFF` | 4.6:1 | PASS | PASS |
| `#FFFFFF` | `#2563EB` | 4.6:1 | PASS | PASS |
| `#FFFFFF` | `#18181B` | 17.9:1 | PASS | PASS |
| `#6B7280`（gray-500） | `#FFFFFF` | 5.0:1 | PASS | PASS |
| `#9CA3AF`（gray-400） | `#FFFFFF` | 2.9:1 | FAIL | FAIL |
| `#DC2626`（red-600） | `#FFFFFF` | 4.6:1 | PASS | PASS |

> **要点**: blue-500 は白背景の通常テキストに使えない。ボタンやリンクには blue-600 以上を使う。gray-400 はプレースホルダー等にも使用不可。

## よくあるカラーの落とし穴

| 落とし穴 | 問題 | 改善策 |
|:---|:---|:---|
| 原色をそのまま使う | `#0000FF` `#FF0000` は安っぽく見える | HSL で彩度を下げ、明度を調整する |
| 彩度が全色均一 | のっぺりした印象、視線誘導が効かない | プライマリだけ高彩度、サブは彩度を落とす |
| ニュートラルに色味がない | 純粋なグレーは冷たく無機質に見える | プライマリの色相をわずかに混ぜる（上記テンプレート参照） |
| ダークモードで明度を反転しただけ | 暗い背景に高彩度色は眩しく浮く | ダーク用に彩度を10-15%下げた別パレットを用意 |
| カラー数が多すぎる | 統一感がなくなり、ブランドが弱まる | プライマリ + アクセント + ニュートラルの3系統に絞る |
| テキストに純黒を使う | 白背景にコントラストが高すぎて目が疲れる | `gray-800`〜`gray-900` を使う |

## HSL を使ったパレット生成テクニック

ベースカラー1色から10段階のスケールを生成する手順:

### ステップ 1: ベースHueを決める

```
例: ブランドカラー #2563EB → HSL(217, 91%, 53%)
ベース Hue = 217
```

### ステップ 2: 明度スケールを作る

色相を固定し、明度を 96% → 10% へ段階的に下げる。
同時に彩度も微調整する（明るい色は彩度を下げ、暗い色はやや上げる）。

```css
/* ベース Hue: 217 から10段階を生成 */
--color-50:  hsl(217, 80%, 96%);   /* ほぼ白。背景のアクセントに */
--color-100: hsl(217, 78%, 90%);   /* 薄いティント。ホバー背景に */
--color-200: hsl(217, 76%, 80%);   /* バッジ・タグの背景に */
--color-300: hsl(217, 74%, 68%);   /* ボーダー・無効状態に */
--color-400: hsl(217, 72%, 56%);   /* セカンダリボタンに */
--color-500: hsl(217, 70%, 48%);   /* ★ ベースカラー（ボタン・リンク） */
--color-600: hsl(217, 72%, 40%);   /* ホバー時のボタンに */
--color-700: hsl(217, 74%, 32%);   /* 見出し・強調テキストに */
--color-800: hsl(217, 76%, 24%);   /* ダークUI要素に */
--color-900: hsl(217, 78%, 16%);   /* 最も暗い。ヘッダー背景に */
```

### ステップ 3: 用途マッピング

```css
:root {
  --btn-primary-bg:    var(--color-500);
  --btn-primary-hover: var(--color-600);
  --btn-primary-text:  #FFFFFF;
  --link-color:        var(--color-600);
  --link-hover:        var(--color-700);
  --bg-accent:         var(--color-50);
  --border-light:      var(--color-200);
}
```

> **ポイント**: 明度の差が小さすぎると隣接する段階の区別がつかない。50→100 は 6%、以降は 8-12% ずつ下げるのが目安。
