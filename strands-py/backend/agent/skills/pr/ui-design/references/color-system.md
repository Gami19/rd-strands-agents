# カラーシステム

> 出典: Refactoring UI — Working with Color / Color Palettes PDF（全 24 パレット収録）

---

## 核心原則

**色は「選ぶ」のではなく「システムとして設計する」。HSL ベースの 10 段階パレットで一貫性を保て。**

---

## 1. パレット構成の3層

| 層 | 役割 | 色数目安 |
|:---|:---|:---|
| **Primary** | ブランドカラー、主要アクション、リンク | 1 色系統 × 10 段階 |
| **Neutrals** | テキスト、背景、ボーダー（UI の 80%） | 1 色系統 × 10 段階 |
| **Supporting** | 成功・警告・エラー・情報 | 2〜4 色系統 × 各 10 段階 |

### 典型的なパレット構成

```
Primary:     Blue / Indigo / Purple / Cyan 等（1つ選択）
Neutrals:    Blue Grey / Cool Grey / Warm Grey / Grey
Supporting:  Red（エラー）/ Yellow（警告）/ Green（成功）/ Blue（情報）
```

---

## 2. HSL ベースの 10 段階設計

各色系統は **暗→明** の 10 段階で構成する。

### 段階と用途

| 段階 | 名称 | Lightness 目安 | 主な用途 |
|:---|:---|:---|:---|
| **50** | Darkest | 15-20% | テキスト、ダークモード背景 |
| **100** | Darker | 20-25% | ヘッダー背景 |
| **200** | Dark | 28-35% | ホバー状態 |
| **300** | Medium Dark | 35-43% | アクティブ状態 |
| **400** | Medium | 45-53% | ボーダー、セカンダリテキスト |
| **500** | Base | 55-60% | デフォルトのアイコン |
| **600** | Medium Light | 60-70% | プレースホルダーテキスト |
| **700** | Light | 74-80% | 薄いボーダー |
| **800** | Lighter | 85-90% | 薄い背景（hover） |
| **900** | Lightest | 92-97% | 最も薄い背景 |

### HSL 調整の要点

```
暗い段階: 彩度を高く（60-100%）、色相をわずかにシフト
明るい段階: 彩度をやや下げ（30-80%）、色相をわずかにシフト
```

- **色相の回転**: 暗い方に向かって青方向（hue -10〜-20）、明るい方に向かって黄方向（hue +10〜+20）にシフトすると自然なグラデーションになる
- **彩度のカーブ**: 中間が最も低く、両端に向かって高くする

## 4. Primary カラーの選び方

### 手順

1. **ベースカラー（中間）を決める**: ボタンの背景色にちょうどいい明るさ
2. **暗い端を決める**: テキストとして使える暗さ
3. **明るい端を決める**: 背景として使える明るさ
4. **中間を補完**: 均等ではなく知覚的に均一な間隔で

## 8. アクセシビリティ

### コントラスト比の基準（WCAG 2.1）

| レベル | 通常テキスト | 大きいテキスト（18px+ bold, 24px+） |
|:---|:---|:---|
| **AA** | 4.5:1 以上 | 3:1 以上 |
| **AAA** | 7:1 以上 | 4.5:1 以上 |

### 実践ルール

- テキスト色と背景色のコントラスト比を常に確認
- Primary カラーのテキスト使用時は **段階 50-300** のみ（暗い色）
- 背景使用時は **段階 800-900**（明るい色）
- **色だけで情報を伝えない**: アイコン・テキスト・パターンを併用

## チェックリスト

- [ ] パレットが Primary / Neutrals / Supporting の 3 層で構成されているか

- [ ] Neutral がブランドカラーと調和しているか（色温度の一致）

- [ ] セマンティックカラー（Red/Yellow/Green/Blue）が用意されているか
- [ ] 色だけでなくアイコン・テキストでも情報を伝えているか
