## 1. 深度とレイヤリング（Creating Depth）

### 1-1. 光源の模倣

UI における深度表現の基本原則は「光は上から来る」こと。

**Raised（浮き上がり）要素**:

```css
/* 上辺を明るく → 光を受けている */
box-shadow: inset 0 1px 0 hsl(224, 84%, 74%);
/* 下に影 → 光を遮っている */
box-shadow: 0 1px 3px hsla(0, 0%, 0%, 0.2);
```

- 上辺: 面の色より少し明るい色（手動で選択。半透明白は彩度を殺すため避ける）
- 下辺: 小さくシャープな影（blur は 2-3px 程度。壁のコンセントの影を参考に）

**Inset（凹み）要素**:

```css
/* 上辺に影 → 上のリップが光を遮る */
box-shadow: inset 0 2px 2px hsla(0, 0%, 0%, 0.1);
/* 下辺を明るく → 上方を向いているため光を受ける */
box-shadow: 0 2px 0 hsla(0, 0%, 100%, 0.15);
```

- テキスト入力、チェックボックス、ウェル（凹みパネル）に有効

**注意**: リアルさを追求しすぎない。視覚的手がかり程度に留める。

### 1-2. シャドウによるエレベーション表現

影の大きさ = 要素の高さ（z 軸上の位置）を表す。

| 影レベル | 用途 | box-shadow |
|:---|:---|:---|
| 最小 | ボタン、小さなインタラクティブ要素 | `0 1px 3px hsla(0,0%,0%,.2)` |
| 小 | カード、ドロップダウン | `0 4px 6px hsla(0,0%,0%,.1)` |
| 中 | ポップオーバー、ホバー時のカード | `0 5px 15px hsla(0,0%,0%,.1)` |
| 大 | モーダルダイアログ | `0 15px 35px hsla(0,0%,0%,.2)` |
| 最大 | 最前面の大きなモーダル | `0 20px 25px hsla(0,0%,0%,.15)` |

**インタラクションとの組み合わせ**:
- ホバー: 影を大きくする（浮き上がる感覚）
- クリック: 影を小さく or 除去する（押し込む感覚）
- ドラッグ: 影を大きくする（持ち上げた感覚）

### 1-3. 二重シャドウ（Two-Part Shadows）

質の高い影は 2 つのパートで構成される。

```css
box-shadow:
  0 4px 6px rgba(0, 0, 0, 0.07),   /* ① 直接光による影: 大きく柔らかい */
  0 5px 15px rgba(0, 0, 0, 0.1);   /* ② 環境光による影: タイトで暗い */
```

| パート | 特徴 | 役割 |
|:---|:---|:---|
| **① 直接光** | 大きな blur、やや強い offset | 遠くに伸びる本影 |
| **② 環境光** | 小さな blur、少ない offset、暗め | 要素直下の接地影 |

**エレベーションが高いほど**: ②の環境光影を薄く（or 除去）する。高い位置では接地影が消えるため。

### 1-4. フラットデザインでの深度表現

影を使わずに深度を表現する方法。

**色による深度**:
- 背景より**明るい**色 → 手前に浮く
- 背景より**暗い**色 → 奥に沈む

```css
/* Raised（浮き上がり） */
.card { background: hsl(210, 36%, 96%); }  /* 背景より明るい */

/* Inset（凹み） */
.well { background: hsl(210, 22%, 49%); }  /* 背景より暗い */
```

**ソリッドシャドウ**:

```css
/* blur なし、垂直 offset のみ */
box-shadow: 0 3px 0 hsl(220, 7%, 83%);
```

フラットな美学を維持しつつ、要素を浮かせる効果。

**要素の重畳（Overlapping）**:
- カードを背景の境界をまたいで配置（`margin-bottom: -60px;`）
- 要素を親より高くして上下にはみ出す（`margin: -60px 0;`）
- 画像の重畳時は「見えない枠線」（背景色と同色の border）でクラッシュを防ぐ

## 2. 画像の扱い（Working with Images）

### 2-1. テキストと画像のコントラスト

背景画像上のテキストは、明暗のダイナミクスが原因で読みにくくなる。

| 手法 | 実装 | 効果 |
|:---|:---|:---|
| **半透明オーバーレイ** | `background-color: hsla(0,0%,0%,.55)` | 全体の明暗差を均一化。最も簡単 |
| **画像コントラスト低減** | `brightness: +40%; contrast: -70%` | 画像の印象を保ちつつコントラストを均一化 |
| **画像カラー化** | ①コントラスト低減 → ②彩度除去 → ③multiply で塗りつぶし | ブランドカラーとの統一に有効 |
| **テキストシャドウ** | `text-shadow: 0 0 50px hsla(0,0%,0%,.4)` | テキスト周辺のみコントラスト強化。大きな blur、offset なし |

**組み合わせ推奨**: コントラスト低減 + テキストシャドウが最もバランスが良い。

### 2-2. 意図されたサイズ

すべてのグラフィック要素には「意図されたサイズ」がある。

| NG パターン | 問題 | 解決策 |
|:---|:---|:---|
| **アイコンの拡大** | 16-24px 用アイコンを 3-4 倍に → ディテール不足で chunky | アイコンを円形/角丸背景の中に配置。アイコン自体は元サイズ付近に |
| **スクリーンショットの縮小** | フルサイズ SS を 70% 縮小 → テキストが読めない | タブレットレイアウトで撮影 or 部分スクリーンショット or 簡略化イラスト |
| **アイコンの縮小** | 128px ロゴを favicon に → ディテールが潰れる | ターゲットサイズ用に簡略版を別途作成 |

### 2-3. ユーザーアップロードコンテンツ

UGC（ユーザー生成コンテンツ）の画像は、アスペクト比やトーンをコントロールできない。

**形状とサイズの制御**:

```css
.user-image {
  width: 200px;
  height: 200px;
  object-fit: cover;        /* はみ出し部分をクロップ */
  object-position: center;  /* 中央でクロップ */
}
```

**背景とのにじみ防止**:

```css
/* NG: border は画像の色とぶつかりやすい */
.avatar { border: 2px solid hsl(212, 12%, 72%); }

/* OK: inset box-shadow は目立たず自然 */
.avatar { box-shadow: inset 0 0 0 1px hsla(0, 0%, 0%, 0.1); }
```

## 3. 仕上げのテクニック（Finishing Touches）

### 3-1. デフォルトの強化（Supercharge the Defaults）

既存の UI 要素をアップグレードするだけで印象が変わる。

| 要素 | 標準 | 強化 |
|:---|:---|:---|
| **箇条書き** | `•` 記号 | チェックマーク、矢印、コンテンツに合ったアイコン |
| **引用文** | `blockquote` テキスト | 大きなクォーテーションマーク + ブランドカラー |
| **チェックボックス** | ブラウザデフォルト | ブランドカラーでカスタムスタイル |
| **ラジオボタン** | 丸 + ラベル | 選択可能なカード形式 |
| **テーブル** | テキストのみ | 画像・カラータグ・アイコンを追加 |
| **ドロップダウン** | リンクの縦並び | セクション分割、複数列、補足テキスト、アイコン付き |

### 3-2. アクセントボーダー

グラフィックデザインのスキルがなくても色付きの帯を加えるだけで「デザインされた」印象になる。

| 配置 | 用途 |
|:---|:---|
| カード上端 | カードに個性を追加 |
| ナビゲーションアイテム下/横 | アクティブ状態の強調 |
| アラート側面 | セマンティックカラーとの組み合わせ |
| 見出し下 | セクションの装飾（短い帯） |
| ページ上端 | サイト全体のブランディング |

```css
.card { border-top: 4px solid var(--primary-500); }
.alert-warning { border-left: 4px solid var(--yellow-500); }
```

### 3-3. 背景の装飾

| 手法 | 実装 | 注意 |
|:---|:---|:---|
| **背景色の変更** | セクションごとに異なる背景色 | コンテンツの区切りを強調 |
| **グラデーション** | 2 色（色相差 ≤30°） | エネルギッシュな印象に |
| **リピートパターン** | SVG / CSS パターン | コントラストは低めに保つ |
| **幾何学的シェイプ** | 角に配置する円、三角形 | コンテンツの邪魔にならない程度に |

### 3-4. 空の状態（Empty States）

コンテンツがまだない状態は**第一印象**である。

**必須要素**:
1. イラストまたはアイコン（注目を引く）
2. CTA（Call to Action）ボタン（次のアクションを促す）
3. 説明テキスト（何をすべきか案内）

**ルール**:
- コンテンツが存在しない間は、タブやフィルターなどの補助 UI を非表示にする
- プレースホルダーデータで設計しない — 空の状態を最初にデザインする

### 3-5. ボーダーの代替

ボーダーの多用はデザインをビジーにする。代替手段を検討する。

| 手法 | 効果 | 適用場面 |
|:---|:---|:---|
| **box-shadow** | ボーダーより柔らかい区切り | カード、パネル |
| **背景色の違い** | 隣接要素を自然に区別 | サイドバーとメイン、ストライプ行 |
| **余白の追加** | UI 要素を追加せずに分離 | リスト項目、セクション間 |

```css
/* NG: ボーダーが多すぎる */
.item { border-bottom: 1px solid #e5e7eb; }

/* OK: 余白で分離 */
.item + .item { margin-top: 16px; }

/* OK: 背景色で区別 */
.item:nth-child(even) { background: hsl(210, 36%, 96%); }
```

### 3-6. 固定観念を超える（Think Outside the Box）

コンポーネントの「あるべき姿」に縛られない。

- **ドロップダウン**: セクション分割、複数列、アイコン、説明テキスト
- **テーブル**: 関連カラムの統合、画像追加、カラータグ
- **ラジオボタン**: 選択可能なカード形式
- **データ表示**: 日付ピッカーの背景反転、ボタンを入力フィールド内に配置

## チェックリスト

### 深度
- [ ] 影が 5 段階のエレベーションシステムに従っている
- [ ] インタラクション時に影が変化する（ホバー↑、クリック↓）
- [ ] 光源の方向（上から）が統一されている
- [ ] フラットデザインの場合、色やソリッドシャドウで深度を表現している

### 画像
- [ ] 背景画像上のテキストに十分なコントラストがある
- [ ] アイコンが意図されたサイズで使用されている（拡大していない）
- [ ] UGC 画像が固定コンテナ内で `object-fit: cover` されている
- [ ] 画像の背景にじみが `inset box-shadow` で防止されている

### 仕上げ
- [ ] デフォルトの UI 要素が強化されている（箇条書き、チェックボックス等）
- [ ] アクセントボーダーが適切に配置されている
- [ ] 空の状態にイラスト + CTA がある
- [ ] ボーダーの代わりに余白・影・背景色を検討した
