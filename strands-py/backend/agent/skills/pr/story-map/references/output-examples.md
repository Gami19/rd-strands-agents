## P-001: 田中先生（教員・担任）

| 項目 | 内容 |
|:---|:---|
| **ID** | P-001 |
| **名前** | 田中 花子 |
| **年齢** | 35歳 |
| **役割** | 小学校教員（4年1組担任） |
| **目標** | 児童一人ひとりの学習状況を的確に把握し、個別最適な指導を行いたい |
| **課題** | 1. 複数システムに分散した児童情報を手作業で統合している（週2時間） 2. 指導要録の作成に膨大な時間がかかる（学期末に1人30分×30人） 3. 他学年・他校の指導事例を参照する手段がない |

| **出典** | inbox/Request.md §2.1, inbox/Material_list.md §3 |

```text
朝: 出欠確認を入力 → 授業中: 学習記録を記録 → 放課後: 成績データを分析 → 学期末: 指導要録を作成

### 重要なゴール

1. **効率化**: 児童情報の入力・参照を1つのシステムで完結させたい
2. **可視化**: 児童の学習推移をグラフで把握したい
3. **省力化**: 指導要録の自動生成で作業時間を半減させたい

## ビジョンステートメント（Vision Statement）

> 教育データ利活用基盤が実現する未来の姿

**ビジョン**: すべての教員が、児童一人ひとりの学習状況を的確に把握し、データに基づく個別最適な指導を実現できる環境を提供する。教育委員会は管内全校のデータを統合分析し、エビデンスに基づく教育施策を立案・評価できる。

### ビジョン達成のためのゴール

1. **教員の働き方改革**: 複数システムに分散した児童情報の統合により、週2時間の事務作業を削減する
2. **個別最適な指導の実現**: 児童の学習推移を可視化し、早期の支援介入を可能にする
3. **エビデンスに基づく施策立案**: 管内全校のデータ横断分析により、教育格差の実態把握と改善施策の効果測定を実現する

## ストーリー詳細定義の例

以下は個別ストーリーの詳細定義例である。ストーリーマップ内では概要のみ記載し、
必要に応じて本セクションのような詳細を別途記述する。

```markdown
## US-001-001-001: CSV から学籍情報を一括登録

| 項目 | 内容 |
|:---|:---|
| **ID** | US-001-001-001 |
| **フィーチャー** | FT-001: データを準備する |
| **エピック** | EP-001-001: 学籍情報を登録する |
| **ペルソナ** | P-001: 田中先生 |
| **ストーリー** | 田中先生として、CSV ファイルから児童の学籍情報を一括登録したい。なぜなら新学期に30人分を手入力するのは非効率だから。 |
| **規模** | M |
| **リリース** | R1 |
| **ビジネスルール参照** | BR-STU-001（学籍番号の一意性）, BR-STU-002（必須項目検証） |
| **ドメインモデル参照** | E-001: Student |
| **出典** | inbox/Request.md §2.1.3 |

### 受入基準

1. CSV ファイル（UTF-8, ヘッダー行あり）をアップロードできる
2. ヘッダー行からカラムが自動マッピングされる
3. 必須項目（学籍番号、姓、名、生年月日）が欠落している行はエラー行として報告される
4. 学籍番号が既存データと重複する場合、エラーとして報告される（BR-STU-001 準拠）
5. エラーのない行のみが登録され、登録件数が表示される
6. 30人分のデータが10秒以内に処理される（NFR-001 準拠）

### 画面イメージ（概要）

1. ファイル選択画面 → CSV ファイルをドラッグ&ドロップ
2. プレビュー画面 → 取込データのプレビュー、カラムマッピング確認
3. 結果画面 → 登録件数、エラー件数、エラー詳細ダウンロード
```
