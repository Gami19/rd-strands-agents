# ワークフロー品質チェックリスト

> ワークフロー（`.agent/workflows/*.md`）および CLAUDE.md テンプレート（`docs/templates/`）の品質を検証するチェックリスト。
> Anthropic 公式ガイドの原則とスキル品質基準を、ワークフロー文脈に適応した 5 領域 30 項目。

---

## 1. Anthropic ガイド原則の適用（8 項目）

| # | チェック項目 | 合格基準 |
|:---|:---|:---|
| W-A1 | **Sequential Workflow（Pattern 1）** | Phase が論理的な順序で定義されている |
| W-A2 | **Iterative Refinement（Pattern 3）** | 品質未達時の差し戻しサイクルが明示されている |
| W-A3 | **Context-aware Selection（Pattern 4）** | プロジェクト種別に応じたスキル/Phase の取捨選択ガイドがある |
| W-A4 | **Be Specific and Actionable** | 各 Phase の手順が具体的で、曖昧な表現（「適切に処理する」等）がない |
| W-A5 | **Error Handling** | Troubleshooting セクションが存在し、10 項目以上ある |
| W-A6 | **Quality Gates** | Phase 間の通過条件（品質ゲート）がサマリーテーブルとして冒頭にある |
| W-A7 | **Progressive Disclosure** | frontmatter → 全体像 → Phase 詳細の 3 段階で情報を開示している |
| W-A8 | **Composability** | 他のワークフローとの関係・併用方法が明示されている |

---

## 2. ワークフロー構造（6 項目）

| # | チェック項目 | 合格基準 |
|:---|:---|:---|
| W-S1 | **frontmatter** | `description` にワークフローの概要が記述されている |
| W-S2 | **全体像 ASCII 図** | ワークフロー全体を俯瞰する ASCII 図が冒頭にある |
| W-S3 | **Phase 構成** | 3-6 Phase で構成されている（多すぎず少なすぎない） |
| W-S4 | **Phase 間引き渡し** | 各 Phase 末に「次工程への引き渡し」トリガー例がある |
| W-S5 | **ADR 記録タイミング** | Phase ごとの ADR 記録タイミングがサマリーテーブルにある |
| W-S6 | **ディレクトリ構造** | 成果物の配置先を示すディレクトリテンプレートがある |

---

## 3. Phase 品質（6 項目）

各 Phase について以下を確認する。

| # | チェック項目 | 合格基準 |
|:---|:---|:---|
| W-P1 | **概要** | Phase の目的が 1-2 文で明記されている |
| W-P2 | **使用スキルテーブル** | スキル名 / 役割 / トリガー例の 3 列テーブルがある |
| W-P3 | **成果物テーブル** | 成果物 / 形式 / 配置先の 3 列テーブルがある |
| W-P4 | **チェックリスト** | Phase 通過条件が 5-8 項目のチェックリストで定義されている |
| W-P5 | **Agent Teams** | 並行実行パターンが定義されている（該当する Phase のみ） |
| W-P6 | **スキルの網羅性** | 関連する全スキルが使用スキルテーブルに記載されている（Phase 2 に Data Platform 等の漏れがないか） |

---

## 4. 付随セクション品質（6 項目）

| # | チェック項目 | 合格基準 |
|:---|:---|:---|
| W-C1 | **スキル発動マトリクス** | 全 Phase × 全スキルのマトリクステーブルがある |
| W-C2 | **スキル間連携図** | ASCII 図でスキル間の関係が可視化されている |
| W-C3 | **クイックスタート** | 3 つ以上の典型パターンが具体的なトリガー列挙で示されている |
| W-C4 | **Troubleshooting** | 問題 / 原因 / 解決策の 3 列テーブルが 10 項目以上ある |
| W-C5 | **関連ワークフロー** | 他ワークフローとの関係・併用指針がテーブルで示されている |
| W-C6 | **マトリクスの整合** | スキル発動マトリクスと各 Phase の使用スキルテーブルが一致している |

---

## 5. CLAUDE.md テンプレート品質（4 項目）

対応する `docs/templates/CLAUDE-*-workspace.md` について確認する。

| # | チェック項目 | 合格基準 |
|:---|:---|:---|
| W-T1 | **プレースホルダー** | `[プロジェクト名]` 等のプレースホルダーが適切に配置されている |
| W-T2 | **Skills Catalog** | ワークフローで使用する全スキルがカタログに記載されている |
| W-T3 | **スキル発動順序** | ワークフロー内でのスキル発動順序が番号付きリストで示されている |
| W-T4 | **カスタマイズガイド** | プロジェクト種別に応じたスキル取捨選択のマトリクスがある |

## チェック実施方法

### 全項目チェック

```
W-A1 ~ W-A8: □□□□□□□□  (8/8)
W-S1 ~ W-S6: □□□□□□      (6/6)
W-P1 ~ W-P6: □□□□□□      (6/6) ← 各 Phase で実施
W-C1 ~ W-C6: □□□□□□      (6/6)
W-T1 ~ W-T4: □□□□          (4/4)
合計: 30/30
```

### 不合格時の対応

| 不合格レベル | 対応 |
|:---|:---|
| W-A 領域（Anthropic 原則） | **必須修正** — ワークフロー設計の根幹 |
| W-S 領域（構造） | **必須修正** — ユーザビリティに直結 |
| W-P 領域（Phase 品質） | **必須修正** — 実行時の品質に直結 |
| W-C 領域（付随セクション） | **推奨修正** — 発見性・理解しやすさの向上 |
| W-T 領域（テンプレート） | **推奨修正** — 展開時の利便性向上 |

## Anthropic ガイドのワークフロー設計パターン対応表

| Anthropic パターン | ワークフローでの適用 |
|:---|:---|
| **Pattern 1: Sequential Workflow** | Phase 0 → 1 → 2 → 3 → 4 の順次フロー |
| **Pattern 2: Multi-MCP Coordination** | Agent Teams による並行実行パターン |
| **Pattern 3: Iterative Refinement** | Phase 2 ↔ Phase 3 の反復サイクル |
| **Pattern 4: Context-aware Selection** | プロジェクト種別に応じたスキル取捨選択ガイド |
| **Pattern 5: Domain-specific Intelligence** | 各スキルが持つドメイン知識の連携 |
