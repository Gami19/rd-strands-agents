## サンプルサイズ計算の基礎

### 必要な入力値

| 入力 | 説明 | 一般的な設定 |
|:---|:---|:---|
| 基準CVR | 現在のコンバージョン率 | 実測値を使用 |
| MDE（最小検出効果量） | 検出したい最小の改善幅 | ビジネスインパクトに基づく |
| 有意水準 | 偽陽性の許容率 | 95%（α = 0.05） |
| 検出力 | 真の効果を検出する確率 | 80%（β = 0.20） |

### MDE の決め方

- ビジネスインパクト: その改善幅で投資対効果があるか
- 実装コスト: 変更の労力に見合う効果か
- 過去のテスト結果: 現実的に達成可能な改善幅か

## テスト期間の計算

### 計算式

```
期間（日）= (群あたりサンプル数 x バリアント数) / (日次トラフィック x 露出率)
```

### 計算例

**例1: 高トラフィックページ**
- 必要: 10,000/群（2群 = 20,000）
- 日次: 5,000訪問
- 期間: 20,000 / 5,000 = **4日**（→ ただし最低1週間は実施）

**例2: 中トラフィックページ**
- 必要: 31,000/群（62,000合計）
- 日次: 2,000訪問
- 期間: 62,000 / 2,000 = **31日**

**例3: 低トラフィック + 部分露出**
- 必要: 15,000/群（30,000合計）
- 日次: 500訪問、50%露出
- 有効日次: 250
- 期間: 30,000 / 250 = **120日**（長すぎ → 対策が必要）

### 最低実施期間

| 条件 | 最低期間 | 理由 |
|:---|:---|:---|
| 全テスト | 1週間 | 曜日変動の吸収 |
| B2B サイト | 2営業サイクル | 平日/週末パターン |
| EC サイト | 給料日を含む期間 | 月初/月末の購買変動 |

### 最長実施期間

4-8週間を超えないこと:
- 新規性効果の減衰
- 外部要因の介入
- 他のテスト機会の喪失

## 複数バリアントの補正

3つ以上のバリアントでは多重比較の問題が生じる。

| バリアント数 | 必要サンプル倍率 | 比較組み合わせ数 |
|:---|:---|:---|
| 2（A/B） | 1x | 1 |
| 3（A/B/C） | 約1.5x | 3 |
| 4（A/B/C/D） | 約2x | 6 |
| 5+ | 削減を検討 | 10+ |

対策: Bonferroni 補正を適用するか、対応ツール（Optimizely 等）を使用する。

## 統計的検定の基礎

### 有意水準（α = 0.05 / 95%信頼水準）

- 観測された差が偶然である確率が5%未満であれば「統計的に有意」
- 保証ではなく閾値 — 20回テストすれば1回は偽陽性が出る

### 検出力（1 - β = 0.80）

- 真の効果がある場合に、それを正しく検出できる確率
- 80%: 真の効果があっても20%は見逃す

### 信頼区間

- 点推定値だけでなく区間推定を確認する
- 区間がゼロをまたいでいれば有意差なし

### 結果の解釈

| 結果 | 判断 | アクション |
|:---|:---|:---|
| 有意に勝利 | バリアントを採用 | 実装に進む |
| 有意に敗北 | コントロールを維持 | 敗因を分析して学ぶ |
| 有意差なし | 効果不明 | トラフィック追加 or より大胆な変更を検討 |
| シグナル混在 | 深掘り必要 | セグメント分析で確認 |

## サンプルサイズが確保できない場合の対処

トラフィック不足で必要サンプルに達しない場合の選択肢:

| 選択肢 | 方法 | トレードオフ |
|:---|:---|:---|
| MDE を大きくする | 20%以上のリフトのみ検出 | 小さな改善を見逃す |
| 信頼水準を下げる | 90%に変更 | 偽陽性のリスク増（文書化必須） |
| バリアントを減らす | 最有望な1案のみテスト | 学びが限定的 |
| トラフィックを合算 | 複数の類似ページで合算 | 均質性の前提が必要 |
| ファネル上流でテスト | トラフィックの多い地点に変更 | 元の仮説との乖離 |
| テストしない | 定性データで判断 | 統計的裏付けなし |
| 長期間実施 | 数週間〜数ヶ月 | 外部要因・新規性効果の問題 |

## よくある間違い

| 間違い | 説明 | 正しい対処 |
|:---|:---|:---|
| 検出力不足 | サンプル不足で効果を見逃す | 事前にサンプルサイズを計算 |
| 早期終了 | 途中結果を見て打ち切る | 事前に決めたサンプル数まで継続 |
| 基準CVRの誤り | サイト全体の平均CVRを使用 | 対象ページ・対象指標の実測値を使う |
| セグメント後出し | 全体で分析後にセグメントを切る | セグメント分析は事前に計画 |
| 多重テスト | 同時に多数のテストを実施 | トラフィック分散を考慮し優先順位をつける |
