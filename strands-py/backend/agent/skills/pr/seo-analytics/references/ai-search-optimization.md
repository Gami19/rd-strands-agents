## 全プラットフォーム共通の基本原則

AI検索で引用されるための3つの前提条件:

1. **インデックスに存在すること** — 各プラットフォームの検索バックエンドにインデックスされていなければ引用されない
2. **クロール可能であること** — robots.txt でAIボットが許可されていること
3. **抽出可能であること** — AIはページ単位ではなくパッセージ単位で引用する。明確な構造と自己完結パラグラフが必要

## Google AI Overviews

Google 検索の約45%に表示される。Google 自社インデックスを使用し、E-E-A-T シグナルを重視する。

### 特徴
- 従来のSEOシグナル（バックリンク・ページ権威性・トピック関連性）を基盤に、引用ソース・構造化データへの選好が加わる
- AI Overviews のソースの約15%のみが従来の検索上位10件と重複する
- 構造化データと明確な回答があれば、従来の順位が低くても引用される可能性がある

### 最適化施策

| 施策 | 効果 | 実装方法 |
|:---|:---|:---|
| スキーマ実装 | +30-40% 可視性 | Article, FAQPage, HowTo, Product |
| 引用ソース付きコンテンツ | +132% 可視性 | 出典付き統計データ・リンクを含める |
| 権威ある文体 | +89% 可視性 | 実証的・具体的・専門的な記述 |
| トピック権威性 | 高 | コンテンツクラスター + 内部リンク |
| 著者情報 | 高 | 著者バイオ・資格・実績を明記 |

### 最もトリガーしやすいクエリパターン
- 「〜とは」「〜の方法」（定義・ハウツー）
- 「おすすめ」「比較」（リスト・比較）

## ChatGPT

Bing ベースのインデックスを使用。ドメイン権威性の影響が最も大きいプラットフォーム。

### 特徴
- ドメイン権威性と信頼性が引用の約40%を決定する
- コンテンツ品質が約35%、プラットフォーム信頼が約25%
- 過去30日以内に更新されたコンテンツは約3.2倍引用されやすい
- コンテンツ-回答適合度（ChatGPTの回答形式との構造的類似性）が引用の約55%を占める

### 最適化施策

| 施策 | 優先度 | 実装方法 |
|:---|:---|:---|
| ドメイン権威性の強化 | 最高 | バックリンク獲得・権威あるサイトからのリンク |
| コンテンツの鮮度維持 | 最高 | 競合トピックは月次更新 |
| ChatGPTの回答形式に合わせる | 高 | 会話的・直接的・整理された構造 |
| 検証可能な統計データ | 高 | 出典付き数値データ |
| 見出し階層の整備 | 中 | H1 > H2 > H3 の論理的階層 |

### 引用元の傾向
- Wikipedia: 全引用の7.8%
- Reddit: 1.8%
- Forbes: 1.1%
- サードパーティの引用が自社サイト引用より影響大

## Perplexity

独自インデックス + Google を組み合わせ、常にソースリンク付きで回答する最も透明性の高いプラットフォーム。

### 特徴
- 多段階のリランキング（関連性 → ランキング要因 → ML品質評価）
- 権威あるドメインの優遇リスト（Amazon, GitHub, 主要学術サイト）
- 時間減衰アルゴリズムにより新しいコンテンツにチャンスがある
- FAQ スキーマと PDF コンテンツへの強い選好

### 最適化施策

| 施策 | 優先度 | 実装方法 |
|:---|:---|:---|
| FAQPage スキーマ | 最高 | Q&A コンテンツに JSON-LD で実装 |
| PDF の公開 | 高 | ホワイトペーパー・レポートを認証なしで公開 |
| Article スキーマ + 日付 | 高 | datePublished, dateModified を明記 |
| 自己完結パラグラフ | 高 | 単独で意味が通るパラグラフを書く |
| 公開頻度 | 中 | 定期的な新規コンテンツ公開 |

## Microsoft Copilot

Bing インデックスに完全依存。Microsoft エコシステム（Edge, Windows, 365）に組み込まれている。

### 最適化施策

| 施策 | 優先度 | 実装方法 |
|:---|:---|:---|
| Bing Webmaster Tools | 最高 | サイトを送信しインデックス状況を確認 |
| IndexNow プロトコル | 高 | 新規・更新コンテンツの即時インデックス |
| ページ速度 | 高 | 2秒以内のロード時間 |
| LinkedIn 活動 | 中 | 記事公開・企業ページ維持 |
| GitHub（該当する場合） | 中 | 技術コンテンツのリポジトリ公開 |

## Claude

Brave Search をバックエンドに使用。極めて選択的に引用する。

### 最適化施策

| 施策 | 優先度 | 実装方法 |
|:---|:---|:---|
| Brave Search での可視性 | 最高 | search.brave.com で自社ブランドの検索結果を確認 |
| 事実密度の最大化 | 最高 | 具体的数値・出典付きデータ・日付入り統計 |
| ClaudeBot 許可 | 必須 | robots.txt で ClaudeBot, anthropic-ai を Allow |
| 正確性の追求 | 高 | 当該トピックで最も正確な情報源を目指す |

## AIボットの robots.txt 設定

```
# AI検索ボット — 引用されたい場合は許可する
User-agent: GPTBot           # OpenAI（ChatGPT検索）
User-agent: ChatGPT-User     # ChatGPT ブラウジングモード
User-agent: PerplexityBot    # Perplexity AI検索
User-agent: ClaudeBot        # Anthropic Claude
User-agent: anthropic-ai     # Anthropic Claude（代替）
User-agent: Google-Extended   # Google Gemini / AI Overviews
User-agent: Bingbot          # Microsoft Copilot（Bing経由）
Allow: /

# トレーニング専用ボット — ブロックしても引用には影響なし
User-agent: CCBot             # Common Crawl（トレーニングデータ収集のみ）
Disallow: /
```

## AI引用を増やすコンテンツ最適化手法

Princeton GEO 研究（KDD 2024）による効果順:

| 手法 | 可視性向上 | 実装ガイドライン |
|:---|:---|:---|
| **引用ソースの追加** | +40% | 権威あるソースへのリンク付き参照 |
| **統計データの追加** | +37% | 出典・数値・時期を明記 |
| **専門家引用の追加** | +30% | 氏名・肩書き・所属付きの引用 |
| **権威ある文体** | +25% | 実証的で専門的なトーン |
| **明確さの向上** | +20% | 複雑な概念を平易に説明 |
| **専門用語の使用** | +18% | ドメイン固有の正確な用語 |
| **語彙の多様性** | +15% | ワードバリエーションの増加 |
| **流暢さの最適化** | +15-30% | 読みやすさ・流れの改善 |
| ~~キーワード詰め込み~~ | **-10%** | **AIの可視性を積極的に悪化させる** |

**最良の組み合わせ**: 流暢さ + 統計データ = 最大の効果。低ランクサイトは引用追加で最大115%の可視性向上。

## AI引用されやすいコンテンツの種類

| コンテンツ種別 | 引用シェア | 引用される理由 |
|:---|:---|:---|
| 比較記事 | 約33% | 構造化・バランス・高意図 |
| 決定版ガイド | 約15% | 包括的・権威的 |
| 独自リサーチ・データ | 約12% | ユニーク・引用可能な統計 |
| ベスト/リスト記事 | 約10% | 明確な構造・エンティティ豊富 |
| 商品ページ | 約10% | 具体的詳細をAIが抽出可能 |
| ハウツーガイド | 約8% | ステップバイステップ構造 |
| 意見/分析記事 | 約10% | 専門家の視点・引用可能 |

**AI引用されにくいコンテンツ**:
- 構造のない汎用的なブログ記事
- マーケティング文言ばかりの薄い商品ページ
- ゲートコンテンツ（AIがアクセスできない）
- 日付・著者情報のないコンテンツ
- PDF のみのコンテンツ（パースが難しい）

## AI可視性モニタリング

### 追跡すべき指標

| 指標 | 計測方法 |
|:---|:---|
| AI Overview 表示率 | Semrush / Ahrefs の AI Overview トラッキング |
| ブランド引用率 | Otterly AI, Peec AI, ZipTie |
| AIシェア・オブ・ボイス | 自社引用 vs 競合引用 |
| 引用センチメント | AIがブランドをどう説明しているか |

### DIYモニタリング（ツールなし）

月次の手動チェック:
1. 主要クエリ20個を選定
2. ChatGPT, Perplexity, Google で各クエリを実行
3. 自社が引用されているか、誰が引用されているかを記録
4. スプレッドシートで月次推移を追跡

## 優先順位の付け方

初めてAI検索最適化に取り組む場合の推奨順序:

1. **Google AI Overviews**（最優先） — 最大リーチ（Google 検索の45%以上）。既存のSEO基盤があるなら、スキーマ実装・引用ソース追加・E-E-A-T 強化から
2. **ChatGPT** — 最も利用されているAI検索ツール。鮮度維持（月次更新）・ドメイン権威性・回答形式適合
3. **Perplexity** — リサーチ層・アーリーアダプター向け。FAQスキーマ・PDF公開・自己完結パラグラフ
4. **Copilot / Claude** — エンタープライズ/開発者向け。基本対策（構造化コンテンツ・引用ソース・スキーマ）は全プラットフォームに効く
