# コスト最適化チェックリスト

> Databricks 環境のコストを最適化するための包括的なチェックリスト。6 大テクニック、ワークロード別の最適化戦略、コスト帰属の設計をまとめたリファレンス。

---

## 1. コスト構造の理解

### Databricks のコストモデル

```
総コスト = DBU コスト + クラウドインフラコスト
         = (DBU 消費量 x DBU 単価) + (VM + Storage + Network)
```

| コスト要素 | Classic Compute | Serverless Compute |
|:---|:---|:---|
| DBU | 低単価 | 高単価（インフラ込み） |
| VM | 別途発生 | DBU に含まれる |
| Storage | 別途発生 | 別途発生 |
| Network | 別途発生 | 別途発生 |

### コスト削減の優先順位

```
1. 不要リソースの停止・削除        -> 即効性: 高、労力: 低
2. オートスケーリングの設定         -> 即効性: 高、労力: 低
3. Spot インスタンスの導入          -> 即効性: 高、労力: 中
4. Serverless への移行              -> 即効性: 中、労力: 中
5. Photon の有効化                  -> 即効性: 中、労力: 低
6. クエリ・パイプラインの最適化     -> 即効性: 中、労力: 高
```

## 2. コスト最適化 6 大テクニック

### 2.1 オートスケーリング（期待効果: 40-60% 削減）

**チェックリスト**:
- [ ] 全クラスターに `min_workers` / `max_workers` を設定した
- [ ] ピーク時のワーカー数が `max_workers` に近すぎないか確認した
- [ ] オフピーク時にスケールダウンされているか確認した
- [ ] Enhanced Autoscaling（Serverless）の利用を検討した

| パラメータ | バッチ ETL | 対話的開発 | ストリーミング |
|:---|:---|:---|:---|
| `min_workers` | 1-2 | 1 | 2 |
| `max_workers` | ピークの 1.2 倍 | 4 | ピークの 1.5 倍 |

### 2.2 Spot インスタンス（期待効果: 最大 90% 削減）

**チェックリスト**:
- [ ] バッチ処理の Worker ノードに Spot を適用した
- [ ] Driver ノードは On-Demand を維持している
- [ ] Spot 回収時のリトライ設計を確認した
- [ ] Spot 非推奨のワークロードを特定した（ストリーミング、時間制約あり）

| 適用推奨度 | ワークロード | Spot 比率の目安 |
|:---|:---|:---|
| 強く推奨 | バッチ ETL（時間制約なし） | 80-90% |
| 推奨 | 定時バッチ（余裕あり） | 50-70% |
| 条件付き | ストリーミング（Worker のみ） | 30-50% |
| 非推奨 | 低レイテンシ要件、SLA 厳格 | 0% |

### 2.3 自動停止（期待効果: アイドルコスト排除）

**チェックリスト**:
- [ ] All-Purpose Cluster に `autotermination_minutes` を設定した（推奨: 30-60 分）
- [ ] SQL Warehouse に Auto Stop を設定した（推奨: 10-15 分）
- [ ] 夜間・週末に稼働しているクラスターがないか確認した
- [ ] Job Cluster は自動破棄されることを確認した

### 2.4 クラスターポリシー（期待効果: ガバナンス強化）

**チェックリスト**:
- [ ] チーム/部門ごとにクラスターポリシーを設定した
- [ ] 許可するインスタンスタイプを制限した
- [ ] 最大ワーカー数の上限を設定した
- [ ] GPU インスタンスの利用を許可制にした
- [ ] DBR バージョンを LTS のみに制限した

### 2.5 ライトサイジング（期待効果: 過剰プロビジョニング排除）

**チェックリスト**:
- [ ] クラスターのメトリクス（CPU 使用率、メモリ使用率）を確認した
- [ ] SQL Warehouse の Spill Rate を確認した
- [ ] 過剰なワーカー数を特定し、削減した
- [ ] 使用されていないクラスターを特定し、削除した

### 2.6 Photon Engine（期待効果: 3x-8x 高速化 = DBU 削減）

**チェックリスト**:
- [ ] SQL / DataFrame ベースのワークロードに Photon を有効化した
- [ ] UDF/RDD を使用しているワークロードは除外した
- [ ] Photon 有効化前後のジョブ実行時間を比較した
- [ ] DBU 消費量の削減を確認した

## 3. ワークロード別の最適化戦略

### ETL バッチ処理

| 最適化施策 | 期待効果 | 実施難易度 |
|:---|:---|:---|
| Serverless Jobs への移行 | TCO 30-50% 削減 | 低 |
| Spot インスタンスの導入 | VM コスト 60-90% 削減 | 低 |
| パイプラインの最適化 | 実行時間短縮 = DBU 削減 | 中-高 |
| `availableNow` トリガーの活用 | 不要な常時起動の排除 | 低 |

### SQL 分析

| 最適化施策 | 期待効果 | 実施難易度 |
|:---|:---|:---|
| Serverless SQL Warehouse | IWM による自動最適化 | 低 |
| Liquid Clustering の導入 | クエリ高速化 = DBU 削減 | 低 |
| Materialized View の活用 | 繰り返し集計の排除 | 中 |
| Gold テーブルの事前集計 | クエリ実行コスト削減 | 中 |

### ML ワークロード

| 最適化施策 | 期待効果 | 実施難易度 |
|:---|:---|:---|
| GPU Spot インスタンスの活用 | 学習コスト大幅削減 | 低 |
| 学習ジョブの Job Cluster 化 | 自動破棄でアイドル排除 | 低 |
| Feature Store の共有 | 重複計算の排除 | 中 |
| バッチ推論への切り替え | サービングコスト削減 | 中 |

## 4. コスト帰属（Cost Attribution）の設計

### カスタムタグ戦略

```
必須タグ:
  - team: チーム名（例: data-engineering, analytics）
  - project: プロジェクト名
  - environment: 環境（dev, staging, prod）
  - cost_center: コストセンター番号

推奨タグ:
  - owner: 責任者のメールアドレス
  - workload_type: ワークロード種別（etl, analytics, ml）
  - expiry_date: リソースの想定終了日
```

### タグの強制適用

クラスターポリシーでタグを必須にする:

```json
{
  "custom_tags.team": {
    "type": "fixed",
    "value": "data-engineering"
  },
  "custom_tags.environment": {
    "type": "allowlist",
    "values": ["dev", "staging", "prod"]
  },
  "custom_tags.project": {
    "type": "regex",
    "pattern": "^[a-z][a-z0-9-]{2,30}$"
  }
}
```

## 5. コスト監視・アラート

### 月次コストレビューチェックリスト

- [ ] DBU 消費量のトレンドを確認した（前月比、前年比）
- [ ] コストセンター別のコスト内訳を確認した
- [ ] 予算対比の達成状況を確認した
- [ ] 異常なコスト増加を特定した
- [ ] 未使用リソース（停止忘れのクラスター等）を棚卸しした

### アラート設定

| アラート条件 | 通知先 | 対応 |
|:---|:---|:---|
| 月額予算の 80% 到達 | チームリード | コスト状況の確認 |
| 月額予算の 100% 到達 | 管理者 | 追加予算の承認 or リソース制限 |
| 単一ジョブの異常コスト | ジョブオーナー | ジョブの調査・最適化 |
| クラスター稼働 24 時間超 | クラスターオーナー | 停止忘れの確認 |

## 6. TCO 比較テンプレート

### Serverless vs Classic の TCO 比較

| 項目 | Serverless | Classic | Classic + Spot |
|:---|:---|:---|:---|
| 月間 DBU 消費量 | X DBU | Y DBU | Y DBU |
| DBU 単価 | $A | $B | $B |
| DBU コスト | $X * A | $Y * B | $Y * B |
| VM コスト | $0 | $C | $C * 0.3 |
| 管理コスト（人件費） | 低 | 中 | 中 |
| **月額合計** | $X * A | $(Y*B + C) | $(Y*B + C*0.3) |

**注意**: 上記は概算テンプレート。実際の比較は Databricks の料金計算ツールと実際のワークロードメトリクスに基づいて行うこと。
