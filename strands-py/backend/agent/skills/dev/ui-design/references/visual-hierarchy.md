# 視覚的階層（Visual Hierarchy）

> 出典: Refactoring UI — Starting from Scratch / Hierarchy is Everything

---

## 核心原則

**すべての要素が重要を主張するとき、何も重要ではなくなる。**
視覚的階層とは、インターフェース内の要素の相対的重要度を制御する技術である。

---

## 1. サイズだけに頼らない

### アンチパターン

```
❌ 重要 → 大きく
❌ 非重要 → 小さく
```

h1 を巨大にし、副次情報を極小にすると、プライマリコンテンツが攻撃的になり、セカンダリコンテンツが判読不能になる。

### ベストプラクティス: 3つのレバーを使う

| レバー | 効果 | 例 |
|:---|:---|:---|
| **サイズ** | 注意を引く | 見出し 24px、本文 16px |
| **色 / コントラスト** | 優先度を示す | 濃い色=主要、薄い色=補助 |
| **フォントウェイト** | 強調度を制御 | bold=主要、normal/light=補助 |

---

## 2. テキスト階層の3レイヤー

| レイヤー | 役割 | スタイル指針 |
|:---|:---|:---|
| **Primary** | 主要情報（見出し、キー情報） | 濃い色（例: hsl(209, 61%, 16%)）、大きめサイズ、太字 |
| **Secondary** | 補足情報（説明文、メタデータ） | やや薄い色（例: hsl(209, 28%, 39%)）、普通サイズ、normal weight |
| **Tertiary** | 付随情報（日付、注釈、著作権） | 薄い色（例: hsl(209, 23%, 60%)）、小さめサイズ、light weight |

### 適用ルール

- Primary には **dark color + bold / semibold**
- Secondary には **medium grey + normal weight**
- Tertiary には **light grey + normal weight + smaller size**
- **色だけで差をつけるのが最も上品**（サイズ差を最小限に）

---

## 3. ラベルは支えであり主役ではない

### アンチパターン

```
❌ ラベルを太字にして値を普通に
   Name: John Doe
   Email: john@example.com
```

### ベストプラクティス

```
✅ 値を強調しラベルを抑える
   John Doe          ← 太字・濃い色
   john@example.com  ← 普通・薄い色

   （ラベルが必要なら小さく薄く上部に配置）
   Name
   John Doe
```

- ラベルなしで意味が通じるなら**ラベルを省略**する
- ラベルが必要なら**値より目立たせない**

---

## 4. 要素の重要度を下げるテクニック

| テクニック | 適用場面 | 効果 |
|:---|:---|:---|
| 色を薄くする | テキスト、アイコン | 視覚的重みを減らす |
| サイズを小さくする | メタデータ、注釈 | 注意を引かなくする |
| 余白を詰める | 関連情報のグループ | 独立した要素に見せない |

### アイコン付きアクションの階層

```
✅ プライマリ: ボタン（塗りつぶし背景 + 白テキスト）
✅ セカンダリ: ボタン（ボーダーのみ or 薄い背景）
✅ ターシャリ:  リンクスタイル（テキストのみ、アンダーライン）
```

---

## 5. セマンティック vs 視覚的階層

### アンチパターン: HTML 要素のデフォルトスタイルをそのまま使う

```html
❌ <h2>は常に同じスタイル</h2>
   → サイドバーの h2 もメインの h2 も同じサイズになってしまう
```

### ベストプラクティス

HTML のセマンティクス（h1, h2, h3）と視覚的スタイルは**分離**する。
サイドバーの h2 はメインの h3 より小さくてよい。

---

## 6. 強調と非強調のバランス

### ルール

- **すべてを強調しない**: 強調は相対的なもの
- **非強調を活用する**: 重要でないものを目立たなくすることで、重要なものが際立つ
- **max 2-3 レベルの階層**: プライマリ・セカンダリ（・ターシャリ）で十分
