# タイポグラフィガイド

> 出典: Refactoring UI — Typography / Choosing Fonts / Sizing and Spacing

---

## 核心原則

**タイポグラフィは UI の骨格である。フォント選びとサイズシステムが UI 全体の品質を決定する。**

---

## 1. フォント選択

### サンセリフ vs セリフ

| 種類 | 適用場面 | 推奨例 |
|:---|:---|:---|
| **サンセリフ** | UI テキスト、ダッシュボード、アプリ | IBM Plex Sans, Source Sans 3, Outfit, Satoshi, Cabinet Grotesk |
| **セリフ** | ブログ、マガジン、長文コンテンツ | Merriweather, Lora, Source Serif 4, Literata |
| **モノスペース** | コード、技術情報 | JetBrains Mono, Fira Code |

> **避けるべき**: Inter, Roboto, Open Sans, Lato（AI 生成デザインで最も過剰使用されるフォント群）。
> LLM は指示がなければこれらのフォントに収束する（Anthropic: "distributional convergence"）。
> ブランドの個性を出す場合は上記推奨フォントまたは Google Fonts からコンセプトに合ったフォントを選定すること。

### フォント選びの5原則

1. **x-height が高いフォントを選ぶ**: 小サイズでも読みやすい
2. **ウェイトのバリエーションが豊富**: 最低 5 ウェイト（Light〜Bold）
3. **数字のプロポーションを確認**: テーブルでは等幅数字（tabular figures）を使う
4. **文字の区別がつくか**: `Il1O0` が区別できるか
5. **2フォントまでに制限**: 見出し用 + 本文用（必要なら）

### システムフォントスタック

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto,
             'Helvetica Neue', Arial, sans-serif;
```

→ OS ネイティブフォントを使い、読み込み時間ゼロ・一貫した見た目を実現

> **注意**: システムフォントスタックは読み込み時間ゼロの利点があるが、AI slop 的な印象を与えるリスクがある。ブランドの個性を出す場合は Google Fonts からコンセプトに合ったフォントを選定すること。

### Google Fonts の読み込み

```html
<!-- Recommended: preconnect + link -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@400;500;600;700&family=Playfair+Display:wght@400;700;900&display=swap" rel="stylesheet">
```

```css
/* CSS での適用例 */
body {
  font-family: 'IBM Plex Sans', sans-serif;
}

h1, h2, h3 {
  font-family: 'Playfair Display', serif;
}
```

> **ポイント**: `@import` による読み込みはレンダリングブロッキングの原因になるため、HTML の `<link>` を推奨。`font-display: swap` を必ず指定して FOUT を許容する。

## 2. フォントサイズスケール

### アンチパターン

```
❌ 14px, 15px, 16px, 17px, 18px ...
   → 差が小さすぎて階層が作れない
```

### ベストプラクティス: モジュラースケール

**推奨スケール**:

```
12px — 14px — 16px — 18px — 20px — 24px — 30px — 36px — 48px — 60px — 72px
```

| サイズ | 用途 |
|:---|:---|
| **12px** | キャプション、ラベル、バッジ |
| **14px** | セカンダリテキスト、メタデータ |
| **16px** | 本文（デフォルト） |
| **18px** | 本文（大きめ）、サブヘッド |
| **20-24px** | h3 見出し、カードタイトル |
| **30-36px** | h2 見出し、セクションタイトル |
| **48-72px** | h1 見出し、ヒーローテキスト |

### rem を使う理由

```css
/* ユーザーのブラウザ設定を尊重 */
html { font-size: 100%; }  /* 16px が基準 */
h1 { font-size: 2.25rem; } /* 36px */
p  { font-size: 1rem; }    /* 16px */
```

## 3. 行間（line-height）

### 基本ルール

| テキストの種類 | line-height |
|:---|:---|
| **本文** | 1.5 〜 1.75 |
| **見出し** | 1.1 〜 1.3 |
| **大きな見出し（36px+）** | 1 〜 1.15 |

### サイズと line-height の反比例関係

```
フォントサイズが大きい → line-height は小さく
フォントサイズが小さい → line-height は大きく
```

---

## 4. 文字間隔（letter-spacing）

| 場面 | letter-spacing | 理由 |
|:---|:---|:---|
| **本文** | 0（デフォルト） | フォントのデフォルトが最適 |
| **大きな見出し** | -0.02em 〜 -0.05em | 大サイズでは間延びして見える |
| **小さいオールキャプス** | 0.05em 〜 0.1em | 大文字のみは詰まって見える |

## 5. テキストの配置

### 左揃えがデフォルト

```
✅ テキストは原則として左揃え（LTR 言語の場合）
❌ 長文の中央揃えは読みにくい（視線の戻り位置が不安定）
```

### 中央揃えが許容される場面

- ヒーローセクションの短い見出し
- カードのタイトル（1-2行）
- 呼び出しブロック（callout）
- フッターのコピーライト

### 右揃え

- 数値カラム（テーブルの金額列）
- 日付表示

## 6. フォントウェイトの使い分け

| ウェイト | 数値 | 用途 |
|:---|:---|:---|
| **Regular** | 400 | 本文テキスト |
| **Medium** | 500 | 強調テキスト、ナビ |
| **Semibold** | 600 | サブ見出し、ボタン |
| **Bold** | 700 | 見出し、重要情報 |

### ルール

- **Light (300) や Thin (100) は見出し専用**: 本文に使うと読みにくい
- **同じ段落内でのウェイト混在は 2 段階まで**: Regular + Bold のみ
- **色 + ウェイトで階層**: ウェイトだけに頼らない

## 7. フォントペアリング例

見出し用フォントと本文用フォントの組み合わせ。**コントラストが高いほど面白い**。Display + monospace、セリフ + ジオメトリックサンセリフ、ウェイト 100 vs 900 の極端なコントラストを意識する。

| # | 見出し | 本文 | トーン | 適合業種 |
|:---|:---|:---|:---|:---|
| 1 | **Playfair Display** (900) | Source Sans 3 (400) | エレガント・格調 | ホテル / ジュエリー / 高級ブランド |
| 2 | **Space Grotesk** (700) | IBM Plex Sans (400) | 先進的・テック | SaaS / 開発者向けツール |
| 3 | **Fraunces** (900) | Source Sans 3 (400) | 個性的・レトロモダン | カフェ / クラフト / D2C |
| 4 | **Outfit** (700) | DM Sans (400) | モダン・洗練 | デザインスタジオ / スタートアップ |
| 5 | **Bebas Neue** (400) | Barlow (400) | 大胆・力強い | スポーツ / メディア / イベント |
| 6 | **Cormorant** (700) | Lora (400) | 文学的・品格 | 出版 / 文化施設 / ギャラリー |
| 7 | **Cabinet Grotesk** (800) | Satoshi (400) | クリーン・現代的 | テクノロジー / フィンテック |
| 8 | **Bricolage Grotesque** (800) | Literata (400) | 独創的・知的 | メディア / ポートフォリオ |

### ペアリングの原則

1. **コントラストの原則**: 見出しと本文で異なるカテゴリ（サンセリフ + セリフ、ディスプレイ + テキスト用）を組み合わせる
2. **ウェイトの極端さ**: 見出しは 700-900、本文は 400。中間の 500-600 だけで構成しない
3. **サイズジャンプ**: 見出しと本文のサイズ差を 3x 以上にする（例: 本文 16px → 見出し 48px+）
4. **フォント数の制限**: 最大 2 フォント。3 つ以上は散漫になる
5. **Google Fonts で利用可能か確認**: 読み込み速度とライセンスを担保
