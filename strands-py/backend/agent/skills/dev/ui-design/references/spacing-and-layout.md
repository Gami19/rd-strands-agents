# スペーシング & レイアウト

> 出典: Refactoring UI — Layout and Spacing / Designing with Constraints

---

## 核心原則

**「目分量」で余白を決めない。制約ベースのスペーシングシステムを使え。**

---

## 1. スペーシングスケール

### アンチパターン

```
❌ margin-top: 13px;   ← 根拠のない数値
❌ padding: 17px 22px;  ← 毎回違う数値
```

### ベストプラクティス: 制約付きスケール

**基本スケール（推奨）**:

```
4px — 8px — 12px — 16px — 24px — 32px — 48px — 64px — 96px — 128px
```

| サイズ | 用途 |
|:---|:---|
| **4px** | インラインアイコンとテキストの間隔 |
| **8px** | 密接に関連する要素間（ラベルとフィールド） |
| **12px** | リスト項目間、フォーム要素間 |
| **16px** | 汎用的な間隔、段落間 |
| **24px** | セクション内の区切り |
| **32px** | カード内のパディング |
| **48px** | セクション間 |
| **64px** | 大きなセクション間 |
| **96px - 128px** | ページレベルの間隔 |

### 16px ベースのリニアスケール vs 非リニアスケール

- **小さい要素**: 差分は小さく（4, 8, 12, 16）
- **大きい要素**: 差分は大きく（32, 48, 64, 96, 128）
- **リニアスケール（8, 16, 24, 32...）は大きな値で差が小さすぎる**

---

## 2. 余白の原則

### 「密」からスタートする

```
✅ まず要素を近づけて配置し、必要に応じて広げる
❌ 広い余白からスタートすると、縮める判断が難しい
```

### 内側と外側のスペーシング

| 種類 | 役割 | 例 |
|:---|:---|:---|
| **padding** (内側) | 要素内のコンテンツ呼吸用空間 | カード内、ボタン内 |
| **margin** (外側) | 要素間の関係性を示す | セクション間、項目間 |

### グルーピングの法則

```
関連要素間 < 非関連要素間

例: フォームのラベルとフィールド間 (8px) < フォームフィールド間 (16px) < フォームグループ間 (32px)
```

---

## 3. レイアウトシステム

### ボックス内の余白を均等にしない

```
❌ padding: 24px;  ← 上下左右均等
✅ padding: 32px 24px;  ← 上下は広く、左右は狭く（テキスト読みやすさ）
```

### グリッド vs Flexbox

| 手法 | 適用場面 |
|:---|:---|
| **12カラムグリッド** | マーケティングページ、複雑なレイアウト |
| **Flexbox** | コンポーネント内、ナビゲーション |
| **Auto margins** | 単一要素の配置 |

### カラム幅の考え方

- テキストコンテンツ: **45〜75文字/行** が読みやすい最適幅
- 最大幅を `max-width` で制限する（`em` 単位推奨）
- ワイド画面で全幅に広がるテキストは読みにくい

---

## 4. レスポンシブデザインの原則

### モバイルファースト

1. 小さい画面で設計を始める
2. 要素を縦に積む
3. 画面が広がったら横に並べる

### ブレイクポイントのアプローチ

```
✅ コンテンツに基づくブレイクポイント
   → 「この要素がこの幅で窮屈になったら」変更する

❌ デバイスに基づくブレイクポイント
   → 「iPad は 768px だから」で決めない
```

### スペーシングもレスポンシブにする

```css
/* 小さい画面ではスペーシングも小さく */
.section { padding: 24px 16px; }

/* 大きい画面ではスペーシングも大きく */
@media (min-width: 768px) {
  .section { padding: 48px 32px; }
}
```

---

## 5. 要素のサイジング

### 固定幅 vs 相対幅

| アプローチ | 適用場面 |
|:---|:---|
| 固定幅（px） | アイコン、アバター、ロゴ |
| 相対幅（%） | コンテナ内のカラム |
| 最大幅（max-width） | テキストブロック、メインコンテンツ |
| 最小幅（min-width） | ボタン、入力フィールド |

### コンテンツに合わせたサイズ

```
✅ ボタンの幅はテキストの長さ + padding で決まる
❌ すべてのボタンを同じ幅にしない（short text と long text で見た目が変わる）
```

- [ ] テキストの行幅が 45〜75 文字に収まっているか

- [ ] padding と margin が一貫したスケールで使われているか
