# 競合分析フレームワーク

> 市場における自社のポジションを明確にし、差別化戦略を立案するためのテンプレート集。
> SKILL.md の Step 5 で使用する。

---

## 1. 競合の 3 分類

競合を正しく分類することが、的確な差別化の第一歩。

| 分類 | 定義 | 分析の焦点 | 例（プロジェクト管理ツールの場合） |
|:---|:---|:---|:---|
| **直接競合** | 同じ問題を、同じカテゴリの解決策で解決する | 機能比較、価格比較 | Asana、Monday.com |
| **間接競合** | 同じ問題を、別のアプローチで解決する | アプローチの優劣 | Excel、メール |
| **代替手段** | 問題を解決しない選択（現状維持を含む） | 乗り換え動機 | 「何もしない」「手作業のまま」 |

### 特定の手順

1. **顧客に聞く**: 「検討時に他にどんな選択肢がありましたか？」
2. **検索結果を分析する**: ターゲットキーワードで上位表示される競合を把握
3. **レビューサイトを確認する**: G2、ITreview 等でカテゴリ内の競合をリスト化
4. **代替手段を想像する**: 自社プロダクトがなければ顧客はどうするかを考える

## 3. ポジショニングマップ

### 3a. 軸の選び方

ポジショニングマップの軸は、以下の条件を満たすものを選ぶ:

1. **顧客が購買時に重視する要素**であること
2. **自社の差別化が際立つ要素**であること
3. **競合との違いが視覚的に明確**になること

### よく使われる軸の組み合わせ

| 軸 1 | 軸 2 | 適するシナリオ |
|:---|:---|:---|
| 価格（低-高） | 機能（シンプル-高機能） | 価格競争力がある場合 |
| 汎用性（特化-汎用） | 対象（SMB-Enterprise） | ニッチ戦略を取る場合 |
| 導入容易性（難-易） | カスタマイズ性（低-高） | 使いやすさが強みの場合 |
| 技術先進性（保守-先進） | サポート（セルフ-手厚い） | 技術力が強みの場合 |

### 3b. マップ作成テンプレート

```
         [軸2: 正の方向]
              │
         ┌────┼────┐
         │    │    │
         │    │    │
[軸1:負]──┼────┼────┼──[軸1:正]
         │    │    │
         │    │    │
         └────┼────┘
              │
         [軸2: 負の方向]

プロット:
  ★ 自社: (X, Y) — [一言コメント]
  ● 競合A: (X, Y) — [一言コメント]
  ● 競合B: (X, Y) — [一言コメント]
  ● 競合C: (X, Y) — [一言コメント]
```

### 3c. ブルーオーシャンの発見

マップ上の**空白地帯**は潜在的な機会を示す。ただし:
- 空白 = 需要がない可能性もある
- 顧客インタビューで需要を検証してから参入判断する

## 4. スイッチング・ダイナミクス（JTBD 四つの力）

### 4a. フレームワーク

```
  ┌─────────────────────────┐
  │   現在の解決策（現状維持）  │
  └────────┬────────────────┘
           │
    Push（不満）    Habit（慣性）
      ↓ 離脱を促す    ↑ 現状に留める
           │
  ─────────┼──────────
           │
    Pull（魅力）    Anxiety（不安）
      ↓ 引き寄せる    ↑ 躊躇させる
           │
  ┌────────┴────────────────┐
  │    新しい解決策（自社）     │
  └─────────────────────────┘
```

### 4b. 分析テンプレート

```markdown
## スイッチング・ダイナミクス

### Push（離脱を促す力）
現在のソリューションへの不満:
1. [不満1: 具体的なシーンと感情]
2. [不満2: 具体的なシーンと感情]
3. [不満3: 具体的なシーンと感情]

### Pull（引き寄せる力）
自社ソリューションの魅力:
1. [魅力1: 具体的なベネフィット]
2. [魅力2: 具体的なベネフィット]
3. [魅力3: 具体的なベネフィット]

### Habit（現状維持の力）
乗り換えを阻む慣性:
1. [障壁1: 既存のワークフロー/データ]
2. [障壁2: チームの慣れ/学習コスト]
3. [障壁3: 契約/コスト的なロックイン]

### Anxiety（不安の力）
乗り換えに対する懸念:
1. [不安1: データ移行のリスク]
2. [不安2: 新しいツールの学習コスト]
3. [不安3: サービス継続性/信頼性]
```

### 4c. 力のバランス評価

```
Push + Pull > Habit + Anxiety → スイッチ発生
Push + Pull < Habit + Anxiety → 現状維持
```

マーケティング施策は以下の方向で設計する:
- **Push を顕在化**: 現状の問題をコンテンツで啓発する
- **Pull を強化**: USP とベネフィットを明確に訴求する
- **Habit を弱める**: 移行ツール、インポーターを提供する
- **Anxiety を解消**: 無料トライアル、返金保証、事例を提供する

## 5. 差別化戦略の立案

### 5a. 差別化の 3 層

| 層 | 内容 | 持続性 | 例 |
|:---|:---|:---|:---|
| **機能的差別化** | 機能やスペックの違い | 低（模倣容易） | 「AIを搭載」 |
| **体験的差別化** | UX やサポートの違い | 中（模倣に時間） | 「5分で導入完了」 |
| **哲学的差別化** | ビジョンや価値観の違い | 高（模倣困難） | 「プライバシーファースト」 |

### 5b. 差別化ステートメント

```
[ターゲット顧客] にとって、
[自社プロダクト] は [カテゴリ] の中で、
[差別化ポイント] を提供する唯一の選択肢です。
なぜなら [根拠・証拠] だからです。
```

**例**:
```
中小企業のマーケティング担当者にとって、
[自社ツール] はマーケティング自動化ツールの中で、
専門知識なしで本格的なキャンペーンを実行できる唯一の選択肢です。
なぜなら AIが最適なコンテンツとタイミングを自動で提案するからです。
```

## 6. 競合モニタリング

### 6a. 定期チェック項目

| 頻度 | チェック項目 | 情報源 |
|:---|:---|:---|
| **週次** | 競合の SNS 投稿、ブログ更新 | SNS、RSS |
| **月次** | 新機能リリース、価格変更 | 製品ブログ、リリースノート |
| **四半期** | 市場シェアの変動、新規参入 | 業界レポート、レビューサイト |
| **半期** | ポジショニングマップの更新 | 総合分析 |

### 6b. アラートトリガー

以下の変化があった場合、即座にポジショニングを見直す:

- 競合の大型資金調達
- 競合の価格改定（特に値下げ）
- 大手企業の市場参入
- 競合の重大な障害・スキャンダル
- 自社顧客の大量解約
