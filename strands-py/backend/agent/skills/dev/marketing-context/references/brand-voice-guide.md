# ブランドボイス設計フレームワーク

> ブランドの人格を言語化し、すべてのコミュニケーションに一貫性を持たせるためのガイド。
> SKILL.md の Step 4 で使用する。

---

## 1. ブランドボイスとは

ブランドボイスは「何を言うか」ではなく「どう言うか」。
プロダクトの人格を反映した、一貫したコミュニケーションスタイルである。

| 要素 | 定義 | 例 |
|:---|:---|:---|
| **ボイス** | ブランドの人格（不変） | 「誠実で、専門的で、親しみやすい」 |
| **トーン** | 場面に応じた声のニュアンス（可変） | LP では力強く、エラー時は共感的に |
| **スタイル** | 文法・表記のルール | 「です/ます調」「英数字は半角」 |

## 2. ブランドパーソナリティ設計

### 2a. 形容詞マトリクス法

4 つの軸でブランドの立ち位置を決める:

```
フォーマル ─────── カジュアル
  ↑                   ↑
「信頼できる」       「気軽に話せる」

真面目 ──────────── ユーモラス
  ↑                   ↑
「正確さ重視」       「楽しさ重視」

控えめ ──────────── 大胆
  ↑                   ↑
「堅実な印象」       「挑戦的な印象」

実用的 ──────────── 情緒的
  ↑                   ↑
「機能を語る」       「感情に訴える」
```

各軸で 1-10 のスコアをつけ、ブランドの位置を可視化する。

### 2b. 「する/しない」定義法

各パーソナリティ特性を「する/しない」で具体化する:

```markdown
## ブランドパーソナリティ: [形容詞]

### する
- [具体的な表現や行動]
- [具体的な表現や行動]

### しない
- [避けるべき表現や行動]
- [避けるべき表現や行動]
```

**例: 「プロフェッショナル」**

| する | しない |
|:---|:---|
| データや事例に基づいて主張する | 根拠なく「すごい」「最高」と言う |
| 正確な数値を提示する | 曖昧な表現で煙に巻く |
| 専門用語を使う場合は平易な説明を添える | 専門用語を説明なしに多用する |
| 間違いを認めて迅速に訂正する | 問題を隠蔽する |

## 3. トーンの場面別設計

### 3a. トーンスペクトラム

同じブランドボイスでも、場面によってトーンは変化する:

```
楽しい ◆━━━━━━━━━━━━━━━━━━━━━━━━━━◆ 真剣
       SNS    LP    ブログ  ドキュメント  障害通知
```

### 3b. 場面別ガイド

| 場面 | トーンの方向性 | 具体例 |
|:---|:---|:---|
| **LP・広告** | 力強く、明確に | 「チームの生産性を3倍にする、たった1つの習慣」 |
| **ブログ・コンテンツ** | 知識的で、親しみやすく | 「実際に試してみた結果をお伝えします」 |
| **製品ドキュメント** | 正確で、簡潔に | 「以下の手順でAPIキーを取得します」 |
| **オンボーディング** | 励ましつつ、具体的に | 「あと1ステップで設定完了です!」 |
| **エラーメッセージ** | 共感的で、解決志向 | 「処理中に問題が発生しました。以下をお試しください」 |
| **障害通知** | 誠実で、責任を持って | 「サービスに影響が出ています。原因の調査を進めています」 |
| **SNS** | カジュアルだが品位を保つ | 「お待たせしました。新機能をリリースしました」 |
| **CS 対応** | 温かく、迅速に | 「ご不便をおかけして申し訳ございません。担当が確認いたします」 |

### 3c. トーン変調の原則

1. **喜びの場面**: テンションを上げてよい（新機能リリース、目標達成）
2. **困りごとの場面**: テンションを下げ、共感から入る（エラー、障害）
3. **学習の場面**: 中立的で、ステップバイステップ（ドキュメント）
4. **説得の場面**: 具体的な根拠とベネフィットで（LP、セールス）

## 4. スタイルガイド

### 4a. 基本ルール

| 項目 | ルール | 例 |
|:---|:---|:---|
| **文体** | です/ます調（LP・ドキュメント）/ だ・である調（技術ブログ） | 場面に応じて統一 |
| **主語** | 「私たち」または省略 | 「弊社」は堅すぎる場合が多い |
| **英数字** | 半角 | ○ API、× ＡＰＩ |
| **数値表記** | 3桁区切りカンマ | ○ 1,000、× 1000 |
| **括弧** | 日本語は「」、英語は "" | 「設定」→ "Settings" |
| **箇条書き** | 体言止めまたは用言止めで統一 | 混在させない |

### 4b. 文章の長さ

| 媒体 | 1文の目安 | 1段落の目安 |
|:---|:---|:---|
| LP ヘッドライン | 20文字以内 | 1文 |
| LP ボディ | 40文字以内 | 2-3文 |
| ブログ | 60文字以内 | 3-5文 |
| ドキュメント | 50文字以内 | 2-4文 |
| SNS | 70文字以内 | 1-2文 |

### 4c. 能動態の原則

受動態より能動態を優先する:

| NG（受動態） | OK（能動態） |
|:---|:---|
| 「設定が完了されます」 | 「設定が完了します」 |
| 「レポートが生成されました」 | 「レポートを生成しました」 |
| 「お客様に送信されます」 | 「お客様に送信します」 |

## 5. NG 表現カタログ

### 5a. 過剰表現

| NG 表現 | 理由 | 代替表現 |
|:---|:---|:---|
| 「最高の」「究極の」 | 根拠なき最上級 | 「業界トップクラスの」「高水準の」 |
| 「革命的な」「画期的な」 | 大言壮語 | 「新しいアプローチの」 |
| 「完全な」「完璧な」 | 非現実的な期待 | 「包括的な」「充実した」 |
| 「たった○分で」 | 期待値の過剰設定 | 「シンプルな操作で」 |
| 「誰でも簡単に」 | 個人差の無視 | 「直感的な操作で」 |

### 5b. 回避表現

| NG 表現 | 理由 | 代替表現 |
|:---|:---|:---|
| 「問題ありません」 | 問題を矮小化 | 「承知いたしました」 |
| 「普通は」「当然」 | 相手を否定する暗示 | 省略するか具体的に説明 |
| 「ご存知の通り」 | 知らない人を排除 | 省略するか簡潔に説明 |
| 「お忙しいところ」 | 形式的すぎる | 「お手数ですが」 |

### 5c. 競合言及ルール

| する | しない |
|:---|:---|
| 一般的なカテゴリで言及（「従来のツール」） | 競合の固有名詞で否定する |
| 客観的な比較データを示す | 主観的な優劣判断を述べる |
| 自社の強みを語る | 他社の弱みを攻撃する |

## 6. ボイス設計ワークシート

以下のテンプレートを埋めることで、ブランドボイスを定義できる:

```markdown
## ブランドボイス定義

### パーソナリティ（3-5 個の形容詞）
1. [形容詞] — [する/しないの具体例]
2. [形容詞] — [する/しないの具体例]
3. [形容詞] — [する/しないの具体例]

### トーンスペクトラム
楽しい ◆━━━[自社の位置]━━━━━◆ 真剣

### 場面別トーン
```

## 7. ブランドボイスの検証方法

### 7a. 入れ替えテスト

書いた文章のブランド名を競合のブランド名に入れ替える。
違和感なく成立してしまうなら、ボイスに独自性がない。

### 7b. ペルソナ音読テスト

ターゲットペルソナになりきって文章を読む。
「自分に向けて書かれている」と感じられるか確認する。

### 7c. チーム横断テスト

マーケ・営業・CS の各チームメンバーに同じ質問への回答を書いてもらう。
ボイスガイドの浸透度を測定できる。
