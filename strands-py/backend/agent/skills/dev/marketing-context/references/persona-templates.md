# ペルソナテンプレート集

> 3 つのビジネスモデル（BtoB / BtoC / SaaS）に対応するペルソナテンプレート。
> SKILL.md の Step 3 で使用する。

## 1. BtoB ペルソナテンプレート

企業間取引のペルソナは、個人ではなく**購買関与者の集合**として設計する。

### 購買関与者マップ

| 役割 | 典型的な肩書き | 関心事 | 重視する指標 |
|:---|:---|:---|:---|
| **ユーザー** | 現場担当者、エンジニア | 日常業務の効率化 | 操作性、学習コスト |
| **チャンピオン** | マネージャー、リーダー | チーム課題の解決 | ROI、導入事例 |
| **意思決定者** | 部長、VP | 事業目標への貢献 | 戦略的価値、スケーラビリティ |
| **経済的決裁者** | CFO、購買部門 | コスト最適化 | TCO、契約条件 |
| **技術検証者** | CTO、情シス | 技術的適合性 | セキュリティ、API、統合性 |

### BtoB ペルソナシート

```markdown
## ペルソナ: [名前]（[Primary/Secondary/Negative]）

### 基本情報
- **役職**: [例: マーケティング部マネージャー]
- **企業**: [業種・規模・ステージ]
- **チーム規模**: [直属の部下数]
- **IT リテラシー**: [高/中/低]
- **購買における役割**: [ユーザー/チャンピオン/決裁者]

### 課題と痛み
1. [日常的に最も時間を取られていること]
2. [上司から求められているが達成できていないこと]
3. [現在のツール・プロセスへの不満]

### ゴール
1. [業務上の短期目標（3か月）]
2. [キャリア上の中期目標（1年）]
3. [チーム・組織に対する貢献目標]

### JTBD（Jobs to Be Done）
- **機能的ジョブ**: [達成したいタスク]
- **感情的ジョブ**: [どう感じたいか]
- **社会的ジョブ**: [周囲からどう見られたいか]

### 反論と対処法
| 反論 | 対処法 |
|:---|:---|
| 「既存ツールで十分」 | [乗り換えの具体的メリットを数値で示す] |
| 「導入が面倒」 | [段階的導入プランを提案] |
| 「予算がない」 | [ROI シミュレーションを提供] |

### 情報収集チャネル
- [業界メディア・ブログ]
- [SNS（LinkedIn, X）]
- [展示会・カンファレンス]
- [同僚・知人からの推薦]
```

## 2. BtoC ペルソナテンプレート

個人消費者のペルソナは**ライフスタイルと感情**に焦点を当てる。

### BtoC ペルソナシート

```markdown
## ペルソナ: [名前]（[Primary/Secondary/Negative]）

### 基本情報
- **年齢・性別**: [例: 32歳・女性]
- **職業**: [例: IT企業のプロジェクトマネージャー]
- **居住地**: [都市部/郊外/地方]
- **世帯構成**: [例: 夫と2歳の子供]
- **年収レンジ**: [例: 600-800万円]
- **デジタルリテラシー**: [高/中/低]

### 一日のタイムライン
| 時間帯 | 行動 | 接触メディア |
|:---|:---|:---|
| 朝（6-9時） | [通勤準備、ニュースチェック] | [SNS、ニュースアプリ] |
| 日中（9-18時） | [仕事] | [PC、Slack] |
| 夜（18-23時） | [育児、自分の時間] | [YouTube、Instagram] |

### 課題と痛み
1. [時間がないこと（具体的なシーン）]
2. [情報が多すぎて選べないこと]
3. [やりたいけど続かないこと]

### 動機と欲求
- **表面的欲求**: [例: 運動不足を解消したい]
- **深層的欲求**: [例: 子供と元気に遊べる体力がほしい]
- **社会的欲求**: [例: 「頑張ってるね」と言われたい]

### 購買行動パターン
1. **きっかけ**: [何がトリガーで検索を始めるか]
2. **検索**: [どこで情報を探すか（Google、SNS、口コミ）]
3. **比較**: [何を基準に比較するか（価格、口コミ、デザイン）]
4. **決定**: [最終的に何が決め手になるか]
5. **共有**: [購入後に口コミを書くか、SNS でシェアするか]

### 使うべき言葉 / 避けるべき言葉
| 使うべき | 避けるべき |
|:---|:---|
| [顧客が実際に使う言葉] | [業界用語・専門用語] |
```

## 3. SaaS ペルソナテンプレート

SaaS は BtoB と BtoC の中間的な性質を持つ。**セルフサーブ型**と**セールス型**で設計が異なる。

### SaaS ペルソナの特徴

| 観点 | セルフサーブ型 | セールス型 |
|:---|:---|:---|
| 購買プロセス | 個人の判断で即決 | 組織の承認が必要 |
| 重視する情報 | 無料プラン、UI、ドキュメント | ROI、セキュリティ、SLA |
| 初期行動 | 即サインアップ | 資料請求、デモ依頼 |
| ペルソナの粒度 | BtoC 寄り（個人中心） | BtoB 寄り（関与者複数） |

### SaaS ペルソナシート

```markdown
## ペルソナ: [名前]（[Primary/Secondary/Negative]）

### 基本情報
- **役職**: [例: スタートアップのCTO]
- **企業規模**: [従業員数、ARR]
- **技術スタック**: [現在使っているツール群]
- **購買パターン**: [セルフサーブ/セールス]

### プロダクト利用シナリオ
1. **導入きっかけ**: [何が起きて検索を始めたか]
2. **初期評価**: [無料プランで何を試すか]
3. **Aha モーメント**: [「これは使える」と感じる瞬間]
4. **有料転換トリガー**: [何がきっかけで課金するか]
5. **チーム展開**: [個人利用からチーム利用への拡大パターン]

### SaaS 固有の指標
| 指標 | 定義 | このペルソナでの傾向 |
|:---|:---|:---|
| Time to Value | サインアップから初期価値体験まで | [例: 5分以内が望ましい] |
| Feature Adoption | 主要機能の利用率 | [例: 3機能以上で定着] |
| Expansion Revenue | アップセル・クロスセルの可能性 | [例: チーム拡大で Pro へ] |

### JTBD とスイッチング
- **Push（現状への不満）**: [現在のツールの何が不満か]
- **Pull（自社への引力）**: [何に魅力を感じるか]
- **Habit（乗り換え障壁）**: [何が乗り換えを阻むか]
- **Anxiety（不安）**: [データ移行、学習コスト等]
```

## ペルソナ設計のベストプラクティス

### やるべきこと

1. **顧客の言葉をそのまま使う** — 「効率化したい」ではなく「毎日30分のコピペ作業をなくしたい」
2. **ネガティブペルソナを定義する** — 誰に売らないかを明確にすることで、メッセージが研ぎ澄まされる
3. **ペルソナに名前を付ける** — チーム内の共通言語になる（「田中さんならどう思う？」）
4. **定期的に検証する** — 四半期に 1 回は実際の顧客データと照合する
5. **3 層に限定する** — ペルソナが多すぎるとメッセージが散漫になる

### やってはいけないこと

1. **デモグラフィックだけで定義する** — 年齢・性別だけでは行動を予測できない
2. **全員を対象にする** — 「20-60代の男女」はペルソナではない
3. **社内の希望を反映する** — 「こんな顧客がいてほしい」ではなく「実際にいる顧客」を描く
4. **一度作ったら放置する** — 市場も顧客も変化する
5. **ペルソナを 5 つ以上作る** — 管理不能になる
