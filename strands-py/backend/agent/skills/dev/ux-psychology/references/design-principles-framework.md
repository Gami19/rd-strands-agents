# デザイン原則フレームワーク

> チームのデザイン原則を心理学の法則と接続する 3 層フレームワーク。

---

## フレームワーク概要

```
Design Principle（チームの目標・価値観）
  ↓
Psychological Law（心理学的観察・根拠）
  ↓
Rules（具体的・測定可能なルール）
```

この 3 層構造により:
- **Principle**: チームが何を優先するか（ゴール）
- **Law**: なぜそれが重要か（心理学的裏付け）
- **Rules**: 具体的に何をすべきか（実行可能な指針）

---

## 良いデザイン原則の条件

| 条件 | 説明 | 良い例 | 悪い例 |
|:---|:---|:---|:---|
| **自明でない** | 明白すぎる真理ではない | 「親しみやすさ > 新規性」 | 「デザインは直感的であるべき」 |
| **実際の問いを解決** | チームの具体的な判断を助ける | 「選択肢の明快さ > 豊富さ」 | 「良いデザインを心がける」 |
| **意見がある** | 優先順位と焦点が明確 | 「速度 > 完全性」 | 「バランスを取る」 |
| **記憶しやすい** | チームが日常的に参照できる | 短く、焦点が明確 | 長く、抽象的 |

---

## Principle → Law の対応例

| Design Principle | 対応する Law | 根拠 |
|:---|:---|:---|
| 選択肢の明快さ > 豊富さ | Hick's Law | 選択肢の数と意思決定時間は対数的に比例 |
| 親しみやすさ > 新規性 | Jakob's Law | ユーザーは他サイトの経験から期待を転移する |
| 即時フィードバック > 完全な結果 | Doherty Threshold | 400ms 以内の応答で生産性が飛躍的に向上 |
| ユーザーの負担 < システムの負担 | Tesler's Law | 核心的複雑性はシステム側が負担すべき |
| ピーク体験 > 平均体験 | Peak-End Rule | 体験はピークと終了で判断される |
| 美しさ = 機能 | Aesthetic-Usability | 美的品質がユーザビリティの知覚を向上 |
| 到達しやすさ > 見た目の整列 | Fitts's Law | ターゲット獲得時間はサイズと距離に依存 |
| 構造化 > 情報の網羅 | Miller's Law | 作業記憶のチャンク限界に配慮 |
| 柔軟な入力 > 厳格な形式 | Postel's Law | 入力は寛容に、出力は保守的に |
| 際立つ重要性 > 均質な配置 | Von Restorff Effect | 異なるものが最も記憶に残る |

## ルール策定の手順

### 1. Principle を選択

チームの目標・価値観に基づいて選定。例: 「選択肢の明快さ > 豊富さ」

### 2. 対応する Law を特定

Principle の目標に最も関連する心理学的法則を選択。例: Hick's Law

### 3. 具体的なルールを導出

Law に基づいて、プロダクト固有の測定可能なルールを策定。

```
Principle: 選択肢の明快さ > 豊富さ
Law: Hick's Law（意思決定時間は選択肢の数に比例）
Rules:
  - 1 画面の主要選択肢は 3 個以下にする
  - 補足説明は 80 文字以内で簡潔に
  - 推奨オプションには視覚的な強調を付ける
```

```
Principle: 親しみやすさ > 新規性
Law: Jakob's Law（ユーザーは他サイトと同じ動作を期待する）
Rules:
  - デザインシステムの共通パターンを優先使用する
  - 派手な UI や独特なアニメーションでユーザーの気を散らさない
  - 独自パターンを導入する場合は必ずユーザビリティテストで検証する
```

## チームへの導入手順

| ステップ | 内容 |
|:---|:---|
| **1. チーム特定** | ワークショップ参加者を決定。デザインチーム + ステークホルダー |
| **2. 合意形成** | デザイン原則の目的と成功基準を定義 |
| **3. 発散** | 各メンバーが 10-15 分で原則案をブレインストーミング |
| **4. 収束** | アイデアをテーマ別に整理し、ドット投票で優先順位付け |
| **5. 洗練・適用** | テーマを統合し、Law と Rules を接続 |
| **6. 共有・推進** | ポスター、共有ドキュメント、デスクトップ壁紙で可視化。チーム内外で推進 |

## 認知度向上の戦術

| 戦術 | 説明 |
|:---|:---|
| **可視化** | オフィス壁にポスター掲示、デスクトップ壁紙 |
| **Show & Tell** | 定期的なナレッジシェアリングセッション |
| **デザインレビュー統合** | レビュー時に原則との整合性を確認 |
| **新メンバーオンボーディング** | 原則を新人教育に組み込む |
