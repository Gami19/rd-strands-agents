# 倫理デザインチェックリスト

> 心理学をデザインに応用する際の倫理的配慮チェックリスト。

---

## 行動操作テクニックの検証

### 1. 可変報酬（Intermittent Variable Rewards）

| チェック項目 | 合格基準 |
|:---|:---|
| Pull-to-refresh の使用目的 | コンテンツ更新が本当に必要な場面でのみ使用 |
| 通知の頻度と内容 | ユーザーにとって価値のある情報のみ通知 |
| ランダム報酬パターン | 中毒的な反復行動を意図的に誘発していない |
| フィードバックループ | ユーザーが自発的に離脱できる設計になっている |

### 2. 無限ループ（Infinite Loops）

| チェック項目 | 合格基準 |
|:---|:---|
| 自動再生 | ユーザーが明示的にオフにできる設定がある |
| 無限スクロール | 自然な区切り・一時停止ポイントがある |
| 連続視聴カウントダウン | キャンセルが容易で、一定回数後に停止 |
| コンテンツの終端 | ユーザーに「十分見た」と感じさせるデザイン |

### 3. ソーシャルアファメーション（Social Affirmation）

| チェック項目 | 合格基準 |
|:---|:---|
| いいね・コメント数の表示 | 自己価値の唯一の指標になっていない |
| 比較の促進 | 他者との過度な比較を煽っていない |
| 通知のタイミング | ソーシャル通知が注意を過度に引きつけない |
| 未成年への配慮 | 若年ユーザーの精神的健康への影響を考慮 |

### 4. パーソナライゼーション（Personalization）

| チェック項目 | 合格基準 |
|:---|:---|
| フィルターバブル | 多様な視点・コンテンツも表示される設計 |
| データ収集の透明性 | 何のデータをどう使っているか明示 |
| 推薦の多様性 | アルゴリズムが極端なコンテンツに誘導しない |
| オプトアウト | パーソナライゼーションを無効にできる |

### 5. デフォルト設定（Defaults）

| チェック項目 | 合格基準 |
|:---|:---|
| プライバシーデフォルト | ユーザーのプライバシーを最大限保護する設定がデフォルト |
| 共有範囲 | コンテンツの公開範囲がユーザーの期待と一致 |
| 通知設定 | 過剰な通知がデフォルトで有効になっていない |
| データ収集 | 必要最小限のデータ収集がデフォルト |

### 6. フリクション除去（Lack of Friction）

| チェック項目 | 合格基準 |
|:---|:---|
| ワンクリック購入 | 衝動買いを過度に促進していないか |
| 簡略化された同意 | ユーザーが何に同意しているか理解できるか |
| 取り消し可能性 | 重要な操作に確認ステップがあるか |
| 適切なフリクション | セキュリティ、プライバシー、重要な決定には適度なフリクションがあるか |

### 7. 互恵性（Reciprocity）

| チェック項目 | 合格基準 |
|:---|:---|
| 社会的義務感 | プラットフォームが不要な社会的義務を生んでいないか |
| 招待・通知 | ユーザーの意図しない招待が自動送信されていないか |
| 返報の圧力 | アクションへの返報を過度に求めていないか |

## ダークパターンチェック

| ダークパターン | 説明 | チェック |
|:---|:---|:---|
| **強制アクション** | サービス利用に不要なデータ提供を強制 | 必須項目は本当に必要な情報のみか |
| **ゴキブリホイホイ** | 入りやすいが出にくい設計 | 退会・解約が登録と同程度に容易か |
| **ミスディレクション** | 注意を意図的に逸らして別のアクションを促す | 主要な選択肢が明確に区別されているか |
| **確認シェイミング** | 拒否オプションを恥ずかしい表現にする | 拒否も尊重される中立的な表現か |
| **隠れたコスト** | 最終段階で予期しない費用を追加 | 全コストが早い段階で明示されているか |
| **偽りの緊急性** | 不必要な時間制限やカウントダウンで焦らせる | 時間制限に正当な理由があるか |
| **偽りの社会的証拠** | 虚偽の「○人が閲覧中」等の表示 | ソーシャルプルーフが正確な情報か |

## 倫理的デザインの原則

| 原則 | 実践 |
|:---|:---|
| **ハッピーパスを超えて考える** | エッジケース、脆弱なユーザー、長期的影響を考慮 |
| **チームを多様化する** | 異なる性別・年齢・背景のメンバーで盲点を減らす |
| **データの先を見る** | 定量データだけでなく、ユーザーとの対話から「なぜ」を理解 |
| **フリクションを味方にする** | エラー防止、セキュリティ強化、熟考促進にフリクションを活用 |
| **自問する** | 「この機能はユーザーの最善の利益に合致しているか？」 |
