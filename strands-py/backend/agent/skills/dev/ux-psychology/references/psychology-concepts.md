# 心理学概念の詳細解説

> Laws of UX の背景にある心理学概念。各法則が「なぜ」効くのかを理解するための参照資料。

---

## メンタルモデル（Mental Models）

システムがどう動くかについて人が持つ内的表象。過去の経験に基づいて形成される。

| 要素 | 説明 |
|:---|:---|
| **形成過程** | 累積的な経験から構築（他のウェブサイト、物理的コントロール、日常の操作） |
| **デザインへの影響** | メンタルモデルとデザインが一致すると良い UX、不一致は混乱と離脱を招く |
| **調査手法** | ユーザーインタビュー、ペルソナ、ジャーニーマップ、エンパシーマップ |
| **関連法則** | Jakob's Law |

---

## 認知負荷（Cognitive Load）

インターフェースを理解・操作するために必要な精神的リソースの量。John Sweller が 1988 年に理論化。

| 類型 | 説明 | デザインでの対策 |
|:---|:---|:---|
| **内在的負荷** | タスク自体の本質的な複雑さ | タスク分解、段階的開示 |
| **外在的負荷** | 情報の提示方法による不必要な負荷 | チャンキング、視覚的階層、一貫性 |
| **関連負荷** | 学習や理解に直接寄与する負荷 | 適切な課題設定、達成可能な目標 |

作業記憶バッファのスロット数は有限。超過すると情報が失われ、タスク困難化・圧倒感・タスク放棄に至る。

## System 1 / System 2（二重過程理論）

Daniel Kahneman『Thinking, Fast and Slow』(2013) に基づく認知処理の二重モデル。

| | System 1 | System 2 |
|:---|:---|:---|
| **速度** | 速い | 遅い |
| **意識** | 自動的・無意識 | 意図的・意識的 |
| **努力** | 低い | 高い |
| **機能** | パターン認識、直感、第一印象 | 分析、計算、複雑な判断 |
| **デザインとの関連** | 第一印象の形成（50ms）、美的評価、親しみやすさの判断 | フォーム入力、設定変更、複雑な選択 |

**デザインへの示唆**: System 1 で良い第一印象を形成し、System 2 の負荷を最小化する。確認ダイアログ等のフリクションで System 2 を意図的に活性化することもある。

## 認知バイアス（Cognitive Biases）

判断における系統的な思考の歪み。Amos Tversky と Daniel Kahneman が 1972 年に導入。

| バイアス | 説明 | デザインとの関連 |
|:---|:---|:---|
| **確証バイアス** | 既存の信念を確認する情報を選好 | フィルターバブル、パーソナライゼーションの偏り |
| **ネガティビティバイアス** | ネガティブな出来事をより鮮明に記憶 | エラー処理の重要性、ネガティブピークの軽減 |
| **新近性効果** | 最後に見た情報を最もよく記憶 | 終了体験のデザイン、最後のステップの品質 |
| **複雑性バイアス** | 複雑な解決策を好む傾向 | シンプルな解決策の価値を過小評価するリスク |
| **記憶バイアス** | ピークと終了で体験全体を評価 | Peak-End Rule の活用 |

## 選択的注意（Selective Attention）

関連情報に焦点を当て、無関係な情報をフィルタリングする能力。注意は容量と持続時間の両面で有限。

| 関連現象 | 説明 | デザインへの示唆 |
|:---|:---|:---|
| **バナーブラインドネス** | 広告に見える要素を無意識に無視 | コンテンツが広告に似ていないか確認 |
| **変化盲** | 注意が他に向いている時に変化に気づかない | 重要な変化には明確な視覚的手がかりを付ける |
| **注意と作業記憶** | 注意力は作業記憶と密接に関連 | 認知負荷を下げることで注意力を確保 |

---

## フロー状態（Flow）

Mihály Csíkszentmihályi が 1970 年に提唱。タスクに完全に没入し、集中力とコントロール感が最適化された精神状態。

| 条件 | 説明 |
|:---|:---|
| **スキルと課題のバランス** | 難しすぎるとフラストレーション、簡単すぎると退屈 |
| **明確なフィードバック** | 行動の結果が即座に分かる |
| **没入感** | 内なる批判から解放され、タスクに完全集中 |
| **生産性への効果** | フロー状態では最大 5 倍の生産性向上 |

**デザインへの示唆**: 適切なフィードバック（<400ms）、不要なフリクションの除去、段階的な課題設定でフロー状態を促進する。

---

## オペラント条件付け（Operant Conditioning）

B. F. Skinner の実験に基づく行動学習のメカニズム。行動と結果の関連付けによって行動が強化・弱化される。

| 強化スケジュール | 説明 | デジタルでの例 |
|:---|:---|:---|
| **固定比率** | 毎回報酬 → 空腹時のみ行動 | ポイントカード（10回で報酬） |
| **固定間隔** | 定期的報酬 → 報酬時期付近のみ行動 | 定期メール配信 |
| **可変比率** | ランダム報酬 → 衝動的・反復的行動 | Pull-to-refresh、通知、スロットマシン |
| **過少報酬** | 報酬が少なすぎる → 行動停止 | レスポンスのないインターフェース |

**倫理的示唆**: 可変報酬は最も中毒性が高い。デザイナーはこのメカニズムがユーザーの利益に合致しているか常に確認すべき。
