# メールシーケンステンプレート集

> 目的別のメールシーケンス設計テンプレート。各メールの件名例・本文構成・CTA を具体的に示す。

## 1. Welcome シーケンス（5 通）

**目的**: 新規登録ユーザーを歓迎し、製品の価値を体験させ、有料化へ導く。
**送信スケジュール**: 登録直後 → 1 日後 → 3 日後 → 5 日後 → 7 日後

### メール 1: 歓迎と最初の一歩（登録直後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「ようこそ! まずはこれだけやってみてください」 |
| 本文構成 | 歓迎の挨拶 → 製品の核心価値を 1 文で → 最初にやるべきアクション 1 つ → CTA |
| CTA | 「初期設定を始める（3 分で完了）」 |

### メール 2: クイックウィンの提供（1 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「最初の 5 分で実感できること」 |
| 本文構成 | 「まだ触っていなくても大丈夫」の安心感 → 最も簡単な成功体験の手順 → 成果の具体例 → CTA |
| CTA | 「チュートリアルを試す」 |

### メール 3: 社会的証明（3 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「A 社が 2 週間で変わった話」 |
| 本文構成 | 導入企業のミニストーリー（課題→導入→成果）→ 読者への共感 → CTA |
| CTA | 「同じ成果を体験する」 |

### メール 4: 機能深掘り（5 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「まだ使っていない機能がありますよ」 |
| 本文構成 | ユーザーの行動データに基づく未使用機能の紹介 → その機能で得られるベネフィット → CTA |
| CTA | 「この機能を試す」 |

### メール 5: 有料化への誘導（7 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「無料期間、あと 7 日です」 |
| 本文構成 | これまでの利用実績の要約 → 有料プランで追加される価値 → 料金の正当化 → リスクリバーサル → CTA |
| CTA | 「プランを選んで続ける」 |

## 2. Onboarding シーケンス（4 通）

**送信スケジュール**: 購入直後 → 3 日後 → 7 日後 → 14 日後

### メール 1: セットアップガイド（購入直後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「3 ステップで準備完了です」 |
| 本文構成 | 感謝 → セットアップ手順（3 ステップ以内）→ サポート連絡先 → CTA |
| CTA | 「セットアップを完了する」 |

### メール 2: コア機能の活用（3 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「この機能だけで元が取れます」 |
| 本文構成 | 最も ROI が高い機能の紹介 → 具体的な使い方（スクリーンショット付き）→ 成果の数字 → CTA |
| CTA | 「今すぐ試す」 |

### メール 3: Tips とベストプラクティス（7 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「上位 10% のユーザーがやっていること」 |

| CTA | 「ベストプラクティスガイドをダウンロード」 |

### メール 4: マイルストーン祝福と次のステップ（14 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「2 週間の成果をまとめました」 |
| 本文構成 | 利用データの要約（成果の可視化）→ 次に挑戦すべき機能 → アップセル機会の提示 → CTA |
| CTA | 「次のレベルに進む」 |

## 3. Re-engagement シーケンス（3 通）

**目的**: 休眠ユーザーを再活性化し、製品に呼び戻す。
**送信スケジュール**: 休眠 14 日後 → 21 日後 → 30 日後

### メール 1: 気づきと新情報（休眠 14 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「お久しぶりです。新機能をお知らせします」 |
| 本文構成 | 不在中に追加された新機能・改善点 → ユーザーの課題に関連する価値 → CTA |
| CTA | 「新機能を確認する」 |

### メール 2: 損失回避と社会的証明（休眠 21 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「他のユーザーはこんな成果を出しています」 |
| 本文構成 | 同業他社の成功事例（読者が見逃している価値）→ 「あなたも同じ成果を」→ 限定オファー → CTA |
| CTA | 「アカウントに戻る」 |

### メール 3: 最終確認（休眠 30 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「続けますか? それとも...」 |
| 本文構成 | 率直な問いかけ → 離脱理由のアンケートリンク → 特別オファー（最終）→ CTA |
| CTA | 「もう一度試す」または「フィードバックを送る」 |

## 4. Cold Outreach シーケンス（4 通）

**目的**: 面識のない見込み客にアプローチし、商談の機会を獲得する。
**送信スケジュール**: 初回 → 3 日後 → 7 日後 → 14 日後

### メール 1: 初回アプローチ（初回）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「{会社名} の {課題領域} について」 |
| 本文構成 | パーソナライズされた観察（採用情報・プレスリリース等）→ 課題の仮説 → 自社の価値（1 文）→ CTA |
| CTA | 「来週 15 分お時間いただけますか?」 |

**構成**: `{相手の状況への言及}` → `{類似企業の成果}` → `{相手にとっての価値 1 文}` → `{15 分の提案}`

### メール 2: 価値の追加提供（3 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「re: {前回の件名}」（スレッド返信形式） |
| 本文構成 | 前回メールへの軽い言及 → 有益なコンテンツ（レポート / 事例 / インサイト）の無償提供 → CTA |
| CTA | 「レポートをお送りしましょうか?」 |

### メール 3: 社会的証明（7 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「{同業他社} の事例」 |
| 本文構成 | 同業種の具体的な成功事例（Before → After の数字）→ 「御社でも同様の効果が見込めます」→ CTA |
| CTA | 「15 分のデモで御社での効果をお見せします」 |

### メール 4: ブレークアップ（14 日後）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「タイミングが違いましたか?」 |
| 本文構成 | 率直で簡潔に → 3 つの選択肢を提示（今話したい / 3 ヶ月後にリマインド / 興味なし）→ CTA |
| CTA | 「1 / 2 / 3 の番号で返信してください」 |

## 5. Promotional シーケンス（3 通）

**目的**: キャンペーンやセール情報を告知し、期間内の購入・申込を促進する。
**送信スケジュール**: キャンペーン開始日 → 中間リマインド → 最終日

### メール 1: キャンペーン告知（開始日）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「【3 日間限定】年に一度の特別オファー」 |
| 本文構成 | オファーの概要（何が・いくらで・いつまで）→ 対象者にとっての価値 → 社会的証明 → CTA |
| CTA | 「特別価格で申し込む」 |

### メール 2: 中間リマインドと追加価値（中間日）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「残り 48 時間。よくある質問にお答えします」 |
| 本文構成 | 残り時間の明示 → よくある質問への回答（反論処理）→ 追加の社会的証明 → CTA |
| CTA | 「今すぐ申し込む（残り 48 時間）」 |

### メール 3: 最終通知と緊急性（最終日）

| 項目 | 内容 |
|:---|:---|
| 件名例 | 「本日 23:59 で終了します」 |
| 本文構成 | 締切の明示 → オファーの価値を再要約（1〜2 文）→ 見逃した場合の損失 → 最終 CTA |
| CTA | 「最後のチャンス: 特別価格で申し込む」 |

## メールシーケンス設計の共通原則

| 原則 | 説明 |
|:---|:---|
| 1 メール 1 ゴール | 各メールに求めるアクションは 1 つだけ |
| アングルローテーション | 毎回異なる角度（証拠、ストーリー、データ、FAQ）でアプローチ |
| 件名は短く | 2〜6 語。内部メール風のカジュアルさが開封率を上げる |
| プレビューテキスト | 件名を補完する情報を 90〜140 文字で。件名の繰り返しは NG |
| モバイルファースト | 本文は 50〜300 語。スクロールなしで CTA が見える構成 |
| 送信タイミング | BtoB は火〜木の 9:00-11:00。BtoC は曜日より時間帯（昼休み・夜）が重要 |
