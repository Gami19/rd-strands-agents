# 7 Sweeps 編集フレームワーク

> 7 つの観点で順次チェックし、コピーの品質を体系的に磨き上げる編集手法。
> 1 Sweep = 1 観点。各 Sweep 後に前の Sweep を壊していないか確認する。

```
Clarity → Specificity → Conversational → Sensory → Flow → Cut → Power
  └──────────── 各 Sweep 後に前の Sweep を再確認 ──────────────────┘
```

---

## Sweep 1: Clarity（明瞭性）

**目的**: 読者が 1 回読んだだけで意味を正確に理解できる状態にする。

### チェックポイント

- [ ] 各文が 1 つの情報だけを伝えている（1 文 1 情報の原則）
- [ ] 専門用語に説明が添えられている、または平易な言葉に置換済み
- [ ] 代名詞（「それ」「これ」）の指示先が文脈から一意に特定できる
- [ ] 読者の前提知識を過剰に仮定していない
- [ ] 曖昧な主語（「〜されます」）が能動態に変換されている
- [ ] 否定の二重使用（「〜しないわけではない」）がない

### Before / After 例文

| Before（NG） | After（OK） |
|:---|:---|
| 当社は業界をリードする技術力で革新的なソリューションを提供し、お客様のビジネスの成長を加速させ、競争優位性を確立します | 当社の技術がビジネス成長を加速します。競合に先んじる力を手に入れてください |
| 最適化されたワークフローが提供されます | 当社がワークフローを最適化します |
| API 連携で業務を自動化 | 今お使いのツールとワンクリックで連携し、手作業を自動化 |

---

## Sweep 2: Specificity（具体性）

**目的**: 曖昧な表現をすべて具体的な数字・事実・固有名に置き換える。

### チェックポイント

- [ ] 「改善」「向上」「最適化」などの曖昧動詞が具体表現に置換されている
- [ ] 数字・期間・割合が少なくとも 1 箇所に含まれている
- [ ] 「多くの」「さまざまな」が具体名や数値に置換されている
- [ ] 削除しても意味が変わらないフィラー表現が除去されている
- [ ] 主張に裏付けとなるデータソースがある（なければ削除を検討）

### Before / After 例文

| Before（NG） | After（OK） |
|:---|:---|
| 時間を節約できます | 週 4 時間の作業を 15 分に短縮します |
| 多くの企業が導入しています | 2,847 社が導入しています（2025 年 12 月時点） |
| すぐに効果を実感できます | 導入 14 日目から効果が数値に現れます |
| サポートが充実しています | 営業時間内は平均 2 時間以内に返信します |

---

## Sweep 3: Conversational（会話調）

**目的**: 企業語やお役所言葉を排除し、読者と 1 対 1 で話しているように書く。

### チェックポイント

- [ ] 企業語（「ご活用いただけます」「促進いたします」）が会話調に変換されている
- [ ] 敬語のレベル（です/ます調 vs だ/である調）がコピー全体で統一されている
- [ ] ブランドの人格（友人的 / 専門家的 / 先輩的）が一貫している
- [ ] 「私たち/当社」と「御社/あなた」の使い分けが自然
- [ ] 声に出して読んでも違和感がない

### Before / After 例文

| Before（NG） | After（OK） |
|:---|:---|
| 本サービスをご活用いただくことで生産性の促進が見込まれます | このツールを使えば、もっと早く仕事が終わります |
| 弊社ソリューションのシームレスなインテグレーションにより | 今のツールとスムーズにつながるので |
| イノベーティブなアプローチで課題をレバレッジ | 新しいやり方で課題を解決します |

---

## Sweep 4: Sensory（感覚的表現）

**目的**: 抽象的なベネフィットを、読者が「見て・聞いて・感じられる」具体的な場面に変換する。

### チェックポイント

- [ ] 抽象的なベネフィットが具体的な場面描写に変換されている
- [ ] 読者が自分の状況に重ね合わせられるシーンが最低 1 箇所ある
- [ ] 感情（安堵・達成感・解放感）に訴える表現がある
- [ ] 過剰な感情表現で操作的に感じられない
- [ ] 五感（視覚・聴覚・触覚・時間感覚）のいずれかに訴えている

### Before / After 例文

| Before（NG） | After（OK） |
|:---|:---|
| ストレスが軽減されます | 月曜の朝、受信トレイを開いても胃がキリキリしなくなります |
| 業務効率が向上します | 金曜 17 時に「今週もやりきった」と思える余裕が生まれます |
| チーム連携が改善します | 「あの件どうなった?」と Slack で追いかける回数がゼロになります |
| データに基づく意思決定ができます | 会議で「たぶん」「おそらく」という言葉が消えます |

---

## Sweep 5: Flow（流れ）

**目的**: 読者が引っかからず、自然に最初から最後まで読み通せる論理構造をつくる。

### チェックポイント

- [ ] セクション間の論理的なつながりが自然である
- [ ] 各セクションの冒頭が前セクションからスムーズに接続している
- [ ] 情報の順序が読者の思考プロセスに沿っている（課題→解決→証拠→行動）
- [ ] 飛躍がない（「なぜ急にこの話?」と感じさせない）
- [ ] 文の長さにリズムがある（短文と中文を交互に配置）

### Before / After 例文

| Before（NG） | After（OK） |
|:---|:---|
| 当社は 2015 年に創業しました。AI 機能を搭載しています。無料トライアルがあります | 毎月の集計作業に疲れていませんか? 当社の AI がワンクリックで集計します。まずは 14 日間、無料でお試しください |
| 機能 A があります。機能 B もあります。お客様の声はこちら。料金プランはこちら | 課題を特定 → 解決策を提示 → 導入企業の成果 → あなたも同じ成果を。まずは無料診断から |

---

## Sweep 6: Cut（削減）

**目的**: 説得力に貢献しない単語・文・セクションを容赦なく削り、密度を上げる。

### チェックポイント

- [ ] 弱い強調語（「非常に」「本当に」「まさに」「かなり」）を削除した
- [ ] 冗長な表現を短縮した（「〜することが可能です」→「〜できます」）
- [ ] 同じ内容の別表現での繰り返しを統合した
- [ ] 価値を提供しないフィラーセクションを削除した
- [ ] 各文に「この文を削除したら説得力は落ちるか?」テストを適用した

### Before / After 例文

| Before（NG） | After（OK） |
|:---|:---|
| 基本的に、このツールは非常に簡単に使うことが可能です | このツールは簡単に使えます |
| さまざまな業界の多種多様な企業様にご利用いただいております | 製造・金融・小売の 120 社が利用中です |
| 実際のところ、正直なところを申し上げますと | （削除） |
| 〜するためにご利用いただくことが可能となっております | 〜に使えます |

---

## Sweep 7: Power（説得力）

**目的**: 最終チェック。読者の感情を動かし、行動を促す力がコピーに備わっているか確認する。

### チェックポイント

- [ ] すべての主張に証拠（数字、事例、引用）が添えられている
- [ ] 社会的証明が CTA の近くに配置されている
- [ ] リスクリバーサル（保証、無料期間、返金ポリシー）が明示されている
- [ ] 読者が行動しない理由（反論）をすべて潰している
- [ ] CTA が「動詞 + 得られるもの」の公式に従っている
- [ ] 「だから何?（So What?）」テストにすべての主張が合格している

### Before / After 例文

| Before（NG） | After（OK） |
|:---|:---|
| AI 搭載の分析機能があります | AI が見逃していたインサイトを自動発見。手作業の半分の時間で意思決定できます |
| 業界 No.1 の導入実績です | G2 で 4.8/5.0 の評価。3,000 社以上が継続利用中です |
| ぜひお試しください | 14 日間無料。クレジットカード不要。3 分で利用開始できます |

---

## Sweep 間の戻り確認ルール

| 完了した Sweep | 戻り確認対象 |
|:---|:---|
| Sweep 2 完了後 | Sweep 1（Clarity）を再確認 |
| Sweep 3 完了後 | Sweep 1-2 を再確認 |
| Sweep 4 完了後 | Sweep 1-3 を再確認 |
| Sweep 5 完了後 | Sweep 1-4 を再確認 |
| Sweep 6 完了後 | Sweep 1-5 を再確認（削減で明瞭性が損なわれていないか特に注意） |
| Sweep 7 完了後 | Sweep 1-6 を全て最終確認 |
