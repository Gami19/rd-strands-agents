# コピーフレームワーク集

> マーケティングコピーの構造を決定する 8 つの主要フレームワーク。状況に応じて最適なものを選択する。

---

## 1. AIDA（Attention-Interest-Desire-Action）

**構造**: 注意 → 興味 → 欲求 → 行動

**最適なシーン**: データや証拠が豊富な BtoB 商材、高額商品の LP、論理的に説得を積み上げたい場合。

**具体例**:
```
[Attention] 営業チームの 68% が「CRM のデータ入力」を最大のストレスと回答しています。
[Interest]  毎日 45 分の手入力。月に換算すると 15 時間が消えています。
[Desire]    SalesBot 導入企業では入力作業が 90% 自動化され、
            営業担当 1 人あたり月 12 時間を商談に充てられるようになりました。
[Action]    14 日間の無料トライアルで、御社のデータで効果を確認してください。
```

---

## 2. PAS（Problem-Agitate-Solution）

**構造**: 課題を特定 → 痛みを増幅 → 解決策を提示

**最適なシーン**: 読者が課題を認識しているが行動していない場合。最も汎用性が高く、LP・メール・SNS すべてに使える。

**具体例**:
```
[Problem]  毎週 4 時間をレポート作成に費やしていませんか?
[Agitate]  年間 200 時間。新規顧客の開拓に使えたはずの時間です。
           しかも手作業のレポートはミスが入り込み、経営判断を誤らせるリスクも。
[Solution] ReportAI はワンクリックで正確なレポートを生成します。
           導入企業はレポート作成時間を 80% 削減しています。
```

---

## 3. BAB（Before-After-Bridge）

**構造**: 現在の痛み → 理想の未来 → 製品が橋渡し

**最適なシーン**: ビフォー・アフターの落差が大きい変革型サービス。感情に訴えたい判断者向け。

**具体例**:
```
[Before] 今のチームは毎月 100 通のメールを手動で書いています。
         返信率は 2%。大半は開封すらされていません。
[After]  パーソナライズされたメールが毎日自動で送信され、
         返信率が 10% に跳ね上がったらどうなりますか?
[Bridge] それがまさに MailFlow の導入企業で起きていることです。
         Acme 社は導入 90 日で商談数を 3 倍にしました。
```

---

## 4. 4Ps（Promise-Picture-Proof-Push）

**構造**: 約束 → 情景描写 → 証拠 → 後押し

**最適なシーン**: 短いコピーで説得を完結させたい場合。SNS 投稿、広告、短いメール向け。

**具体例**:
```
[Promise] 採用コストを半分にできます。
[Picture] 来月から求人媒体への出稿をやめ、
          社員紹介と自社メディアだけで優秀な人材が集まる状態を想像してください。
[Proof]   HireWell 導入企業 120 社の平均採用コスト削減率は 53%。
[Push]    今月中のお申し込みで初期費用ゼロ。まずは無料診断から。
```

---

## 5. StoryBrand

**構造**: ヒーロー（顧客）→ 課題 → ガイド（自社）→ 計画 → 行動喚起 → 成功 / 失敗回避

**最適なシーン**: ブランドサイト、About ページ、長期的なブランディング。顧客を物語の主人公にする。

**7 つの要素**:

| 要素 | 役割 | 例 |
|:---|:---|:---|
| ヒーロー | 顧客が何を求めているか | 「もっと売上を伸ばしたい中小企業の経営者」 |
| 外的課題 | 機能的な問題 | 「マーケティングに時間を割けない」 |
| 内的課題 | 感情的フラストレーション | 「大手には太刀打ちできないという焦り」 |
| ガイド | 共感 + 権威 | 「私たちも 5 人のチームから始めました。今は 3,000 社を支援」 |
| 計画 | 3 ステップ | 「1. 無料診断 → 2. 戦略提案 → 3. 実行サポート」 |
| 成功 | 理想の未来 | 「広告費を抑えながら月 30 件の問い合わせ」 |
| 失敗回避 | 行動しないリスク | 「競合に顧客を奪われ続ける状態」 |

---

## 6. QVC（Question-Value-Call）

**構造**: 課題を問う質問 → 簡潔な価値提案 → 直接的な次のステップ

**最適なシーン**: C-suite 向けコールドメール、超短文コピー。興味のフィルタリングに最適。

**具体例**:
```
[Question] SDR の時間の大半がリサーチに消えていませんか?
[Value]    当社はプロスペクトリサーチを自動化し、
           営業担当が商談に集中できるようにします。
           導入企業では 1 人あたり週 3 倍のミーティングを実現。
[Call]     10 分のデモ、ご興味ありますか?
```

---

## 7. FAB（Feature-Advantage-Benefit）

**構造**: 機能 → 優位性 → 利益

**最適なシーン**: 製品比較ページ、機能紹介セクション、テクニカルバイヤー向けのコピー。機能をベネフィットに翻訳する。

**変換テーブル例**:

| Feature（機能） | Advantage（優位性） | Benefit（利益） |
|:---|:---|:---|
| リアルタイムダッシュボード | 手作業の集計が不要 | 毎朝 30 分の集計作業がゼロに。その時間を戦略立案に使える |
| SOC 2 Type II 認証取得 | 業界最高水準のセキュリティ | 情シスの監査対応工数を年間 120 時間削減 |
| Slack / Teams 連携 | 既存ツールから離れずに操作 | ツール切り替えのストレスなく、チーム全員が即日利用開始 |

**注意**: Feature だけで止めない。必ず Benefit まで到達させる。

---

## 8. PASTOR（Problem-Amplify-Story-Testimony-Offer-Response）

**構造**: 課題 → 増幅 → ストーリー → 証言 → 提案 → 反応喚起

**最適なシーン**: 長文セールスレター、メールシーケンス、コンサルティング・教育系サービス。各要素を個別メールに分割可能。

**具体例（メールシーケンスへの分割）**:

| メール | 要素 | 内容例 |
|:---|:---|:---|
| 1 通目 | Problem | 「月次レポートに 3 日かかっていませんか?」 |
| 2 通目 | Amplify | 「年間 36 日。つまり 1 ヶ月以上をレポート作成だけに費やしています」 |
| 3 通目 | Story | 「同じ悩みを抱えていた A 社の経理部長の話をさせてください」 |
| 4 通目 | Testimony | 「A 社はレポート時間を 85% 削減し、経営会議の質が劇的に向上しました」 |
| 5 通目 | Offer + Response | 「30 日間無料で試せます。カレンダーからデモ予約をどうぞ」 |

## フレームワーク選択マトリクス

| 状況 | 第 1 候補 | 第 2 候補 | 理由 |
|:---|:---|:---|:---|
| LP（課題認識あり） | PAS | AIDA | 痛みの増幅で行動を促す |
| LP（変革型サービス） | BAB | StoryBrand | ビフォー・アフターの落差で感情を動かす |
| 製品比較・機能紹介 | FAB | AIDA | 機能をベネフィットに翻訳する |
| ブランドサイト | StoryBrand | BAB | 顧客中心の物語でブランド構築 |
| コールドメール（C-suite） | QVC | PAS | 簡潔さが最優先 |
| メールシーケンス | PASTOR | PAS | 要素を複数メールに分散できる |
| SNS 投稿 | 4Ps | PAS | 短文で説得を完結 |
| ブログ記事 | AIDA | PAS | 長文で論理的に積み上げ |

## 見出しフォーミュラ集

| パターン | フォーミュラ | 例 |
|:---|:---|:---|
| 成果型 | 「{成果} を {痛みなしに} 実現する」 | 「売上 2 倍を広告費ゼロで実現する」 |
| 課題型 | 「もう {嫌なこと} に悩まない」 | 「もうレポート作成に週末を潰さない」 |
| 対象型 | 「{対象} のための {カテゴリ}」 | 「ひとりマーケターのための分析ツール」 |
| 差別化型 | 「{従来と正反対の方法} で {成果} を」 | 「コードを書かずにアプリを開発する」 |
| 証拠型 | 「{数字} の {人々} が {製品} で {成果}」 | 「2,847 社が導入し平均 32% のコスト削減」 |
